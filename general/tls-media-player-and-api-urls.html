소개

TLS (전송 계층 보안)는 인터넷을 통해 전송되는 데이터를 암호화하여 보호합니다. (TLS는 SSL이라고하는 이전 암호화 프로토콜을 대체했습니다.) 이 프로세스는 대부분 사용자에게 투명합니다. 웹을 통해 전달되는 데이터는 암호화 유무에 관계없이 전달될 수 있으므로 클라이언트 (예: 브로커) 에게 데이터를 암호화해야 함을 서버에 알립니다. 이는 일반적으로 HTTP 대신 URL에 HTTPS 프로토콜을 사용하여 수행됩니다.

Video Cloud는 다음 조건에 따라 모든 미디어 전달에 대해 TLS를 지원합니다.

  • Brightcove CDN을 사용하는 사용자는 Brightcove 지원에 문의둘 중 하나에 대해 계정을 구성할 수 있습니다. HTTPS + HTTP데이터 전달 또는 HTTPS 전용배달.
  • 다른 CDN (BYO CDN이라고도 함)을 사용하는 사용자는 CDN (지원되는 경우)에서 이러한 옵션을 요청할 수 있습니다.

TLS가 필요한지 여부는 주로 동영상을 배포하는 사이트 / 페이지가 HTTPS를 통해로드되는지 여부에 따라 달라집니다.로드되는 경우 동영상 및 동영상 자산, 플레이어 및 API 요청도 요청해야합니다. HTTPS; 그렇지 않으면 뷰어의 브라우저에서로드되지 않거나 경고를 생성 할 수 있습니다.

TLS 지원 변경

인터넷 보안 전문가의 권장 사항에 따라 Brightcove는 아래에 설명 된대로 TLS 버전에 대한 지원을 변경합니다.

변경 사항은 무엇입니까?

Video Cloud는 TLS 버전 1.0 또는 1.1에 대한 지원을 점진적으로 종료합니다. 이러한 버전을 사용하는 고객은 가능한 한 빨리 마이그레이션해야합니다.

다음에 대한 TLS 1.0 및 1.1에 대한 지원이 해제되었습니다.

  • 갤러리 API
  • Delivery System API
  • Player Management API

Brightcove가 이전 TLS 버전을 더 이상 사용하지 않는 이유는 무엇입니까?

고객 데이터의 보안은 Brightcove의 최우선 과제입니다. 이러한 변화는 업계 전반에 걸쳐 진행되고 있으며 많은 기술 파트너 및 동료가 이미 초기 TLS를 비활성화하는 조치를 취했습니다.

이 변경에 대한 준비가되었는지 어떻게 알 수 있습니까?

TLS 1.0 / 1.1은 더 이상 사용되지 않으므로 Video Cloud 플랫폼과의 연결은 TLS 1.2 암호화 프로토콜 (또는 그 이상)을 사용해야합니다. 영향이 최소화 될 것으로 예상하지만 두 가지 위험 영역이 있습니다.

  1. 비디오 클라우드 API에 대한 모든 연결 (예: 파이썬 스크립트에서 CMS API로의 호출) 은 TLS 1.2 이상을 지원하는 플랫폼에서 이루어져야 합니다. 이는 자동화 된 프로세스가 호환되는 플랫폼에서 호스팅되어야 함을 의미합니다.
  2. 사용자가 지원되지 않는 브라우저를 사용하거나 관련 암호화 프로토콜을 비활성화했거나 지원되지 않는 장치를 사용하는 경우 영향을받을 수 있습니다.

일반적으로 영향은 사용자의 일부를 차지하는 구형 장치 및 브라우저로 제한되어야합니다. 지원되지 않는 클라이언트 및 기기의 샘플은 아래를 참조하세요.

지원되지 않는 플랫폼 (플레이어 / SDK)

앞서 언급했듯이 "일반적인"HTTP 연결은 변경 사항의 영향을받지 않습니다. 다음은 TLS 1.2를 사용한 HTTPS 연결이 지원되지 않는 사용자 에이전트 목록입니다.

OS / 브라우저 조합

  • Windows 7 / Chrome 29 (또는 이전 Chrome 버전)
  • Windows 7 / Firefox 10.10.12, Firefox 17.0.7, Firefox 24.2.0, Firefox 22, Firefox 24
  • Windows 8 / Firefox 26
  • Windows XP / IE 6
  • Windows Vista / IE 7
  • Windows Phone 8.0 / IE 10
  • Windows 7 / Opera 12.15, Opera 15, Opera 16
  • OS X 10.6.8 / Safari 5.1.9
  • OS X 10.8.4 / Safari 6.0.4
  • Android 4.3 이하

기타 지원되지 않는 플랫폼 및 장치

플레이어 또는 SDK를 제공하지 않거나 API와 사용자 정의 통합을 작성한 플랫폼 및 장치에서 통합을 확인해야합니다. 아래 목록은 완전한 것은 아니지만 TLS 1.2를 지원하지 않는 여러 플랫폼 및 OEM 장치를 포함합니다.

TLS 1.2 활성화

경우에 따라 TLS 1.2가 지원되지만 비활성화 될 수 있습니다. Windows 웹 브라우저에서 TLS 1.2 프로토콜을 활성화하려면 아래를 참조하십시오.

  • 마이크로 소프트 인터넷 익스플로러
  • 구글 크롬
  • 모질라 파이어 폭스
  • 오페라

마이크로 소프트 인터넷 익스플로러

  1. Internet Explorer를 엽니 다.
  2. 메뉴 표시 줄에서 도구> 인터넷 옵션> 고급 탭을 클릭하십시오.
  3. 보안 범주까지 아래로 스크롤하고 TLS 1.2 사용 옵션 상자를 수동으로 선택합니다.
  4. 확인을 클릭하십시오.
  5. 브라우저를 닫고 Internet Explorer를 다시 시작하십시오.

구글 크롬

  1. Google 크롬을 엽니 다.
  2. Alt-F를 클릭하고 설정을 선택합니다.
  3. 아래로 스크롤하여 고급 설정 표시를 선택합니다.
  4. 네트워크 섹션까지 아래로 스크롤하고 프록시 설정 변경을 클릭합니다.
  5. 고급 탭을 선택하십시오.
  6. 보안 범주까지 아래로 스크롤하고 TLS 1.1 사용 및 TLS 1.2 사용 옵션 상자를 수동으로 선택합니다.
  7. 확인을 클릭하십시오.
  8. 브라우저를 닫고 Chrome을 다시 시작하십시오.

모질라 파이어 폭스

  1. Firefox를 엽니 다.
  2. 주소 표시줄에 about:config를 입력하고 Enter 키를 누릅니다.
  3. 검색 필드에 을 입력합니다tls . 에 대한 항목을 찾아 두 번 클릭합니다security.tls.version.max .
  4. TLS 1.2의 프로토콜을 적용하려면 정수 값을 3으로 설정하십시오.
  5. 확인을 클릭하십시오.
  6. 브라우저를 닫고 Mozilla Firefox를 다시 시작하십시오.

오페라

  2. Ctrl + F12를 클릭합니다.
  3. 네트워크 섹션까지 아래로 스크롤하고 프록시 설정 변경을 클릭합니다.
  4. 고급 탭을 선택하십시오.
  5. 보안 범주까지 아래로 스크롤하고 TLS 1.1 사용 및 TLS 1.2 사용 옵션 상자를 수동으로 선택합니다.
  6. 확인을 클릭하십시오.
  7. 브라우저를 닫고 Opera를 다시 시작하십시오.
