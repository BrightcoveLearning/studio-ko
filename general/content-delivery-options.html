소개

Video Cloud 계정을 만든 후 가장 먼저 결정해야 할 사항 중 하나는 비디오 파일을 제공 할 계획입니다. Video Cloud는 다양한 장점과 한계가있는 다양한 선택을 제공합니다. 선택하는 전송 방법은 비디오 파일을 보호하는 방법, 시청자에게 도달하려는 방법, 선호하는 업로드 방법, 비디오 자산이 이미 다른 CDN (콘텐츠 전송 네트워크 또는 콘텐츠)에 저장되어 있는지 여부에 따라 다릅니다. 유통망) 등.

이 항목에 설명 된 콘텐츠 전송 옵션은 Video Cloud Enterprise 게시자 만 사용할 수 있습니다. Video Cloud Enterprise 및 Express 게시자는 Brightcove를 CDN으로 사용해야합니다.

CDN이란 무엇입니까?

게시자는 CDN을 사용하여 미디어를 온라인 시청자에게 널리 배포합니다. 위키 백과 CDN을 다음과 같이 정의합니다.

콘텐츠 전송 네트워크 또는 CDN (콘텐츠 배포 네트워크)은 최종 사용자에게 성능, 확장 성 및 비용 효율성을 개선하기 위해 콘텐츠를 가장 자주 전송하기 위해 투명하게 협력하는 인터넷을 통해 함께 네트워크로 연결된 컴퓨터 시스템입니다.

상업적으로 이용 가능한 많은 CDN이 있습니다. Video Cloud Enterprise 또는 Express 게시자 인 경우 Brightcove가 CDN 솔루션입니다. 비디오 클라우드 엔터프라이즈 퍼블리셔인 경우 브라이트코브 CDN 또는 비디오 클라우드가 지원하는모든 CDN을 통해 콘텐츠를 전송하도록 선택할 수있습니다.

CDN없이 내 콘텐츠를 전달할 수 있습니까?

적은 양의 트래픽이 예상되거나 비디오 트래픽을 완전히 제어 할 수있는 경우 비디오를 제공하는 데 CDN이 필요하지 않을 수 있습니다. 예를 들어, 일부 게시자는 내부 위키 또는 사설 네트워크에 콘텐츠를 배포하려고합니다. 시스템이 외부 CDN없이 미디어 전달을 처리 할 수 있습니다. Brightcove가이 설정을 지원할 수있는 방법에 대한 자세한 정보가 필요하면 Brightcove 고객 지원 또는 계정 관리자에게 문의하십시오.

내 미디어 콘텐츠는 어떻게 전달됩니까?

콘텐츠 전달 메커니즘에는 여러 유형이 있습니다.

  • 스트리밍
  • 점진적 다운로드

스트리밍 (HLS 또는 DASH라고도 함)

스트리밍 비디오는 뷰어의 컴퓨터 / 장치에 파일을 다운로드하지 않고 스트리밍 서버를 통해 전달됩니다. 시청자가 재생을 누르면 동영상이 재생되기 시작합니다. 사용자가 동영상의 다른 부분으로 이동하거나 건너 뛰기로 결정한 경우 즉시 수행 할 수 있으며 해당 지점부터 동영상이 계속 재생됩니다. 스트리밍 미디어의 장점 중 하나는 비디오의 시청 부분 만 전달되었으므로 시청자가 시청 한 비디오에만 대역폭이 사용된다는 것입니다. 클라이언트 측에는 아무것도 보관되지 않습니다. 모든 것이 서버 측에 있습니다.

HTML5 비디오 플레이어 시대에 모든 최신 장치와 브라우저는 스트리밍을 지원하며 가능한 경우 스트리밍을 사용합니다(프로그레시브 다운로드가 아닌). 시청자가 전체 비디오를 시청하지 않으면 버퍼링을 더 잘 관리하고 대역폭을 절약할 수 있으므로 좋은 방법입니다.

스트리밍은 다음을 원하거나 수행해야하는 상황에서 유용합니다.

  • 긴 파일 (10 분 이상) 또는 높은 비트 전송률 파일 (고화질) 전송
  • 대역폭 감지를 수행하고 사용 가능한 하드웨어에 대해 최상의 품질을 제공하는 Video Cloud 다중 비트 레이트 스트리밍 기능을 활용하십시오. 점진적 다운로드와 함께 다중 비트 전송률 스트리밍을 사용할 수 있지만 대역폭 감지는 재생 시작시에만 발생하며 플레이어는 시청자의 대역폭 변화에 적응하지 않습니다.
  • 더 적은 대역폭으로 더 많은 스트림 제공
RTMP 스트리밍에 대한 참조가 발생할 수 있습니다. 이것은 오래된 Flash 스트리밍 형식이며 지금은 거의 사용되지 않습니다.

iOS 기기로 스트리밍

Apple iOS 장치는 비디오 파일 전송을 위해 Apple HTTP Live Streaming (HLS) 및 HTTP (점진적 다운로드) 만 지원합니다. 비디오에 사용 가능한 HLS 변환이 하나 이상 있는지 확인해야합니다. App Store 규칙은 긴 형식의 비디오 콘텐츠 (5Mb 또는 10 분 이상)에 대해 Apple HTTP Live Streaming을 사용하도록 요구합니다.

스트리밍 핵심 포인트:

  • 비디오 콘텐츠는 임시 비디오 링크를 통해 보호됩니다.
  • 진행 영상이 가능합니다
  • Video Cloud의 동적 전달 기능을 효과적으로 활용
  • 적은 대역폭으로 더 많은 스팀을 제공 할 수 있습니다.
  • 사용중인 Apple HTTP Live Streaming (HLS) 또는 Real Time Messaging Protocol (RTMP) 프로토콜

실제로 위에서 설명한 것과 같이 고유 한 기능 중 일부를 찾지 않는 한 콘텐츠가 스트리밍되었는지 점진적으로 다운로드되었는지 거의 알 수 없습니다.

점진적 다운로드 (PD라고도 함)

아주 오래된 브라우저나 장치만 여전히 점진적 다운로드가 필요합니다. 이제 주요 용도는 사용자가 다운로드 및 저장할 수 있는 변환을 제공하는 것입니다.

점진적 다운로드를 사용하여 콘텐츠를 제공하면 일반 웹 페이지 또는 기타 다운로드 가능한 문서와 마찬가지로 HTTP 요청을 통해 표준 웹 서버에서 파일이 제공됩니다. 동영상이 재생되면 동영상 파일이 시청자의 컴퓨터 또는 장치에 다운로드되어 임시 디렉토리에 저장됩니다. 컴퓨터 / 장치에 충분한 파일이 다운로드되면 비디오가 재생되기 시작합니다. 시청자가 빨리 감기를 원하거나 동영상의 다른 부분으로 건너 뛰기를 원할 경우 동영상의 해당 부분이 이미 다운로드 및 저장된 경우에만 그렇게 할 수 있습니다. 스트리밍 비디오와 비교할 때 점진적 다운로드에는 단 하나의 일관된 이점이 있습니다. 비디오를 제공하는 데 스트리밍 서버가 필요하지 않습니다. 점진적 다운로드 비디오는 일반 웹 서버에서 제공 할 수 있습니다.

점진적 다운로드는 트래픽 요구 사항이 적은 애호가 또는 웹 사이트에 적합합니다. 콘텐츠가 시청자의 컴퓨터 / 장치에 캐시되어 있는지, 짧은 길이의 동영상 (10 분 미만) 만 제공해야하는지 신경 쓰지 마세요. 고급 기능이 필요하고 비디오 전송을 제어해야하는 게시자 및 / 또는 더 많은 잠재 고객 (수백 명 이상의 동시 시청자)에게 비디오를 표시해야하는 게시자, 비디오 사용 또는보기 통계를 추적 및보고해야하거나 제공하려는 게시자 최고의 대화 형 재생 환경을 제공하려면 동영상을 스트리밍해야합니다. 스트리밍 전송은 또한 시청 한 비디오의 일부만 실제로 전송되기 때문에 점진적 전송보다 적은 대역폭을 소비합니다.

프로그레시브 다운로드 요점:

  • 제한된 탐색 및 탐색 기능
  • 시청자는 콘텐츠에 액세스하고 용도를 변경할 수 있습니다.
  • Video Cloud의 동적 전달 기능을 효과적으로 활용하지 못함
  • 보안 수준이 낮고 콘텐츠를 쉽게 복사 할 수 있습니다.
  • 시청 한 부분뿐만 아니라 비디오의 모든 부분이 다운로드되므로 더 많은 대역폭이 사용됩니다.

이미지 자산은 어떻게 제공됩니까?

비디오 콘텐츠에 스트리밍 또는 프로그레시브 다운로드를 사용하든 관계없이 Video Cloud 플레이어에서 최종 사용자에게 전달된 이미지 (예: 썸네일 및 비디오 스틸) 는 HTTP 다운로드를 통해 전달됩니다.

콘텐츠 전송 비교: Brightcove 호스팅 비디오 자산 vs. BYO vs. 원격 자산

Video Cloud는 CDN 공급자 및 설정에 따라 몇 가지 기본 CDN 구성을 지원합니다. 선택 사항은 다음과 같습니다.

  • Brightcove CDN
  • BYO CDN
  • 원격 자산
  • 다중 선택

Brightcove CDN

사용하도록 선택할 수 있습니다. 브라이트코브CDN. Video Cloud는 Akamai, Fastly 및 기타 Tier 1 CDN을 활용하여 비디오 콘텐츠에 대한 스트리밍 및 점진적 다운로드 제공을 모두 제공합니다. 일반적으로 HLS 또는 DASH 스트리밍이 사용되며 프로그레시브 다운로드는 특별한 경우에만 사용되거나 비디오의 다운로드 가능한 변환을 제공합니다.

BYO CDN

원하는 CDN ( BYO CDN또는 “자체 대역폭 사용”) 을 사용하도록 선택할 수도 있습니다. 이 경우 CDN 제공 업체와의 계약에 따라 비디오는 프로그레시브 다운로드 (PD) 또는 스트리밍 메커니즘을 사용하는 Video Cloud 플레이어를 통해 원활하게 제공됩니다. BYO CDN을 선택하면 Video Cloud 풀 기반 수집을 사용하여 콘텐츠를 Video Cloud 계정으로 수집 한 다음 CDN 공급자로 이동하여 Video Cloud 플레이어를 통해 최종 사용자에게 전달할 수 있습니다.

원격 자산

최종 선택은 원격 자산. 이 경우 CDN에 이미 비디오 파일이 저장되어있을 수 있으며 새 콘텐츠를 추가하기 위해 Video Cloud 업로드 도구를 전혀 사용하고 싶지 않습니다. Video Cloud Media 모듈, Dynamic Ingest API 또는 CMS API CDN 공급자가 원격으로 저장한 기본 비디오 자산을 가리키는 비디오를 Video Cloud 계정에 생성합니다.

한계

원격 자산이있는 비디오를 생성 할 때 이미지 캡처 기능을 사용하여 원격 자산 비디오에 대한 비디오 스틸 및 썸네일 이미지를 생성 할 수 없습니다. 원격 자산 비디오를 위해 비디오 스틸 및 썸네일 이미지를 업로드 할 수 있습니다.

원격 자산 사용에 대한 자세한 내용은 원격 에셋 비디오작업을 참조하십시오 .

다중 선택

BYO CDN 및 원격 자산을 모두 사용하도록 Video Cloud 계정을 구성 할 수 있습니다. 예를 들어, 일부 비디오 콘텐츠가 CDN에서 이미 사용 가능한 경우 기존 콘텐츠에 원격 자산을 사용하여 다시 업로드하지 않도록하고 향후 새 업로드에 BYO CDN을 사용할 수 있습니다. 이 설정이 필요한 경우 당사에 문의하십시오. 또한 각각 다른 콘텐츠 전달 전략을 가진 여러 Video Cloud 계정을 만들 수도 있습니다.

BYO CDN 또는 원격 자산을 사용하도록 콘텐츠 전달 구성

다음은 BYO CDN 또는 원격 자산을 설정하는 주요 단계입니다.

  1. CDN으로 계정을 설정합니다 (현재 지원되는 계정을 보려면 다음 섹션을 참조하십시오).
  2. 마이그레이션해야하는 기존 비디오 콘텐츠 (Brightcove 또는 외부 CDN에서)가있는 경우 당사 서비스 팀에 문의하십시오.
  3. 설정을 완료하려면 Video Cloud 계정 관리자에게 문의하십시오.

Video Cloud는 BYO CDN에 대해 어떤 CDN을 지원합니까?

Video Cloud는 BYOB CDN 콘텐츠 전송을 위해 다음과 같은 주요 CDN을 지원합니다.

  • Akamai
  • CloudFront
  • 빠르게
  • JOCDN
  • 라임 라이트

다른 BYO CDN이 지원 될 수 있지만 기능 지원은 다양합니다. 자세한 내용은 영업 담당자또는 계정 관리자에게 문의하십시오.

BYO CDN에 지원되는 기능

BYO CDN에는 다음 기능이 지원됩니다.

  • HTTP / HTTPS
  • HTTPS 만
  • 서명 된 매니페스트 URL
  • 서명 된 콘텐츠 URL (위에 나열된 CDN에만 해당)
  • CDN 지역 제한 (CDN에서 지원하는 경우)
  • CDN IP 제한 (CDN에서 지원하는 경우)
  • 사용자 지정 TTL 설정 (CDN에서 지원하는 경우)
