SSO (Single Sign-On) 는 사용자가 하나의 로그인 자격 증명 (예: 사용자 이름 및 암호) 을 사용하여 여러 응용 프로그램에 액세스할 수 있도록 하는 세션 및 사용자 인증 서비스입니다. 이 서비스는 사용자에게 권한이 부여 된 모든 응용 프로그램에 대해 최종 사용자를 인증하고 사용자가 동일한 세션 동안 응용 프로그램을 전환 할 때 추가 프롬프트를 제거합니다.

게시자가 선택한 ID 공급자 (IdP)를 통해 SSO를 통해 Studio에 액세스 할 수있는 Video Cloud Studio에 SAML 기반 싱글 사인온 지원이 제공됩니다. 일부 기능은 다음과 같습니다.

  • Video Cloud Studio는 SAML 2.0 SSO 프로토콜을 지원합니다.
  • ID 매핑은 단일 이메일 도메인이 아닌 사용자 이름 (전체 이메일 주소)을 기반으로합니다. 이를 통해 일부 사용자는 SSO를 사용하도록 설정하고 다른 사용자는 일반 Video Cloud Studio 로그인을 사용하기를 원하는 게시자에게 더 많은 유연성을 제공합니다.
  • 계정 및 기존 사용자가 SSO에 대해 구성되면 Video Cloud Studio UI를 통해 계정에 추가 된 새 사용자는 추가하는 사용자의 SSO 설정을 상속합니다.
  • Video Cloud Studio는 SSO에 대한 서비스 공급자 시작 로그인을 지원합니다. 현재 시작된 직접 IdP는 지원하지 않지만 ID 공급자 포털에서 연결할 수있는 Video Cloud Studio 로그인 화면을 우회하는 URL을 제공 할 수 있습니다.

계정에서 SSO를 활성화하는 방법에 대한 자세한 내용은 계정 관리자에게 문의하십시오.

혜택

Video Cloud Studio 로그인에 SSO를 추가하면 얻을 수있는 몇 가지 이점은 다음과 같습니다.

  • 모든 애플리케이션에 대한 단일 로그인 -여러 사용자 이름과 암호를 기억하는 것은 관리하기 어렵고 IT 팀 (또는 도구의 계정 관리자)이 처리하는 데 골칫거리가됩니다. SSO를 활성화하고 여기에 Video Cloud Studio 로그인을 추가하면 Brightcove 계정 관리자는 더 이상 이러한 문제를 처리 할 필요가 없으며 도구 사용자는 Video Cloud Studio에 대한 별도의 사용자 이름 / 암호를 기억할 필요가 없습니다.
  • 보안 및 암호 요구 사항 -기업은 길이, 기간 및 복잡성과 같은 다양한 사용자 암호 요구 사항을 가지고 있습니다. SSO를 사용하면 Brightcove가 각 개별 요구 사항 집합을 수용하기 위해 Video Cloud Studio의 기본 제공 보안 관리를 변경하지 않고도 이러한 요구 사항을 지원할 수 있습니다.
  • 빠르고 쉽게 사용자 액세스 비활성화 -Single Sign-On을 통해 IT는 직원이 퇴사하는 즉시 사용자를 중앙에서 관리하고 사용자 액세스를 제거 할 수 있습니다. SSO는 불만을 품은 사용자가 Brightcove 미디어, 플레이어, 토큰 등을 수정하거나 삭제하는 것을 방지합니다.

한계

SAML에 대한 적시 사용자 프로비저닝은 현재 지원되지 않습니다. 사용자는 여전히 Studio를 통해 계정에 추가되어야하며 권한 / 모듈 액세스 제어는 Studio 사용자 관리 UI를 통해 구성됩니다. 새 사용자가 계정에 추가되면 추가하는 사용자의 SSO 설정을 상속합니다.

자주하는 질문

SAML 기반 SSO가 지원되지만 2 요소 (2FA) 또는 다중 요소 인증 은 어떻습니까? 지원됩니까?

아니요, 현재 일반 Studio 로그인에서 2FA를 지원할 계획이 없습니다. SAML 기반 SSO에 대한 로그인을 활성화하고 IdP 설정에 2FA를 추가 할 수 있지만 IT 팀이 자체 IdP에서 구성합니다.

CAS 또는 Kerberos와 같은 다른 SSO 프로토콜을 지원할 계획이 있습니까?

지금은 아닙니다.

계정에 SSO를 거치지 않아야하는 사용자와 SSO를 거쳐야하는 다른 사용자가 있습니다. 지원됩니까?

예. SSO는 사용자별로 활성화되므로 일부 사용자는 SSO 로그인에 대해 활성화되고 다른 사용자는 일반 Studio 로그인에 대해 활성화 될 수 있습니다. 한 가지주의 할 점은 Studio 사용자 관리 UI에서 새 사용자가 계정에 추가 될 때 사용자가 통과해야하는 인증 경로를 선택할 수있는 방법이 없다는 것입니다. 현재 사용자는 자신을 추가하는 관리자가 구성한 인증 경로를 상속합니다. 나중에 Studio에서이를 선택할 수있는 옵션을 공개 할 계획입니다. 혼합 인증 흐름에 대한 해결 방법은 하나의 관리자를 SSO 용으로 구성하고 다른 하나는 일반 Studio 로그인 용으로 구성하는 것입니다. 관리자는 설정해야하는 인증 방법에 따라 새 사용자를 추가 할 때 적절한 사용자 계정으로 로그인해야합니다.

다른 ID 공급자에 대해 서로 다른 사용자를 구성 할 수있는 유연성을 지원하는 동시에 단일 IdP와 모든 사용자가 항상 해당 단일 IdP에 대해 구성되기를 원하는 고객도 있습니다. 해당 구성도 사용할 수 있습니다.

단일 계정의 사용자에 대해 여러 ID 공급자를 설정하는 기능이 지원됩니까?

예. 특정 조직 (고객)에 대해 여러 ID 공급자를 설정할 수 있으며 사용자를 두 IdP에 할당 할 수 있습니다. 계정에 새 사용자를 추가 할 때 인증 경로 상속에 관한 위의 질문을 참조하십시오.

사용자에 대해 SAML SSO가 켜져있을 때 Brightcove에 직접 로컬 로그인 (Studio 인증 사용자 이름 / 암호)이 허용됩니까?

아니요. 사용자가 SSO를 사용하도록 설정하면 이것이 자신의 계정에 인증 할 수있는 유일한 방법입니다.

Brightcove는 SP에서 시작된 SSO로 SAML v2.0을 지원합니까?

예

  • SP 시작 로그인은 다음과 같은 특수 직접 도메인 로그인 URL을 통해 지원됩니다.
    https://signin.brightcove.com/login/ext/saml?behavior=xxxxxxxxx
  • 사용자가signin.brightcove.com페이지로 이동할 수도 있습니다. Brightcove 로그인 페이지에 사용자 이름 / 암호를 입력하면 로그인 할 수 있도록 IdP로 리디렉션됩니다 (아직 로그인하지 않은 경우). Brightcove 로그인 페이지에서 비밀번호 필드가 무시됩니다. 사용자가 SSO를 사용할 수 있는지 확인하기 위해 이메일 주소 만 검사합니다.
Brightcove는 SSO 후 비디오에 직접 액세스하기 위해 SAMLRequest / SAMLResponse에서 RelayState 매개 변수 사용을 지원합니까?

아니요. SSO를 통해 로그인하면 사용자는 항상 Studio 홈 대시 보드로 이동합니다.

SSO 통합이 싱글 사인 아웃을 지원합니까?

아니.

지원되는 ID 공급자는 무엇입니까?

Brightcove는 모든 SSO ID 공급자를 사용하여 테스트하지 않았지만 IdP가 SAML 2.0을 지원하는 한 문제가 없을 것이라고 확신합니다. 우리가 고객에게 이야기 한 몇 가지 일반적인 사항은 다음과 같습니다. Okta, Ping Federated, Ping ID, Microsoft Active Directory, OneLogin 및 Auth0.
