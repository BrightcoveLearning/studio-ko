Video Cloud API를 사용할 때 플레이어 서비스 설정에서 기본 인증을 활성화하지 않는 한 메서드 호출에 액세스 토큰이 필요합니다. 기본 인증 활성화에 대한 자세한 내용은 계정 설정관리를 참조하십시오 .

참고: 이 기능은 프로페셔널 계정과 엔터프라이즈 계정에서만 사용할 수 있습니다.

클라이언트 등록 생성

API 인증 페이지는 새 클라이언트 등록을 만드는 데 사용됩니다. 등록을 생성 할 때 등록으로 활성화 할 서비스와 역할을 선택할 수 있습니다.

계정에 대한 새 클라이언트 등록을 만들려면 다음과 같이 하십시오.

  1. Studio 헤더에서 ADMIN링크를 클릭합니다.
  2. API 인증링크를 클릭합니다. API 인증 페이지가 열리고 현재 클라이언트 등록이 표시됩니다.
    참고: 로그인한 계정에 대한 클라이언트 등록만 표시됩니다.
  3. 새 응용 프로그램등록을 클릭합니다 .
  4. 입력이름그리고설명클라이언트 등록을 위해.
  5. 승인 할 계정을 하나 이상 선택하십시오.
  6. 이 등록에 사용할 API 및 권한을 선택하십시오.
    노트:
  7. DFP/Google Ad Manager 통합에 필요한 API 권한은 다음과 같습니다. CMS 비디오 읽기 .
  8. 많은 수의 계정과 API를 선택하면 HTTP 헤더 크기가 커지고 무효화 될 수 있습니다. 서버가 허용하는 최대 헤더 크기는 8K입니다. 즉, 자격 증명을 적용하려면 30 개 이하의 계정을 선택해야합니다. 보안상의 이유로 자격 증명 집합에 대한 권한 범위를 필요한 가장 작은 계정 및 작업 집합으로 제한하는 것이 좋습니다.
  9. 저장을 클릭합니다 .
  10. 클라이언트 ID및클라이언트암호가 반환됩니다.
    참고: 클라이언트 암호가 다시 표시되지않으므로 클라이언트 암호를 복사하여 안전한 위치에 저장합니다.

클라이언트 등록 관리

API 인증 페이지를 사용하여 클라이언트 등록을보고 관리 할 수 있습니다. Studio 헤더에서 ADMIN링크를 클릭한 다음 API 인증을 클릭합니다 . 현재 등록 목록이 표시됩니다.

등록 세부 정보를 보려면 이름을 클릭하십시오. 여기에서 다음을 수행 할 수 있습니다.

  • 이름 수정
  • 클라이언트 ID 복사
  • 승인 할 계정 추가
  • 활성화 할 추가 API를 선택하세요.
  • 자격 증명 삭제

저장을클릭하여 변경 사항을 저장합니다.

여러 계정에 대한 자격 증명이 생성 된 경우 사용자는 모든 계정에 대한 액세스 권한이있는 경우에만 해당 자격 증명을 볼 수 있습니다. 예를 들어 사용자 1은 계정 A 및 B에 대한 자격 증명을 만듭니다. 사용자 2는 로그인하고 계정 A에만 액세스 할 수 있습니다. 사용자 2는 계정 B에 대한 액세스 권한이 없기 때문에 자격 증명을 볼 수 없습니다.

클라이언트 등록 삭제

클라이언트 등록을 삭제하려면 삭제 아이콘() 등록 옆에 있는 등록 또는 등록 편집을 클릭하고등록 삭제 . 등록을 삭제하면 영구적으로 비활성화되며 취소 할 수 없습니다.

DFP 작업

현재 DFP에서 콘텐츠 소스로 설정된 Video Cloud 계정이 여러 개인 경우 다음 두 가지 옵션이 있습니다.

  1. DFP에서 콘텐츠 소스로 설정된 각 Video Cloud 계정에 대해 고유 한 클라이언트 ID와 클라이언트 비밀번호를 만들 수 있습니다. 예를 들어 Video Cloud 계정이 10 개인 경우 Video Cloud의 API 인증 설정에 DFP라는 10 개의 애플리케이션이 있습니다.
  2. DFP에서 콘텐츠 소스로 설정된 모든 Video Cloud 계정에 대해 단일 클라이언트 ID와 클라이언트 비밀번호를 만들 수 있습니다. 예를 들어, 10 개의 Video Cloud 계정이있는 경우 Video Cloud의 API 인증 설정에 DFP라는 애플리케이션이 1 개 있습니다.

옵션 # 1 또는 # 2를 선택했는지 여부에 관계없이 DFP에서 설정 한 각 콘텐츠 소스에 대해 고유 한 계정 ID를 제공해야합니다. 따라서 현재 DFP에 설정된 콘텐츠 소스의 수를 줄이는 것이 아닙니다. 읽기 토큰을 클라이언트 ID 및 클라이언트 시크릿으로 바꾸는 것입니다. 이는 콘텐츠 소스간에 동일하거나 각각에 대해 고유 할 수 있습니다.

옵션 # 1은 더 많은 시간이 소요됩니다. 그러나 클라이언트 ID와 클라이언트 암호가 모든 콘텐츠 소스에서 동일한 경우 (옵션 # 2와 마찬가지로) 새 클라이언트 ID와 클라이언트 암호를 생성하고 업데이트해야하므로 새 콘텐츠 소스를 추가하는 것이 부담이됩니다. DFP의 모든 콘텐츠 소스입니다. 따라서 Brightcove는 옵션 # 1을 권장합니다.
