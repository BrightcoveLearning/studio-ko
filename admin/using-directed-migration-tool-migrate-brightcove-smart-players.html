Directed Migration Tool은 스마트 플레이어 대신 Brightcove 플레이어를로드하도록 지정하는 방법을 제공합니다. 이 도구는 기존 스마트 플레이어를 사용하는 대신 Brightcove 플레이어 내에서 비디오를 게시하는 쉬운 방법을 제공합니다. 기존 게시 코드를 변경할 필요가 없습니다.

이 항목에는 다음 섹션이 포함되어 있습니다.

  • 지정 마이그레이션 요구 사항
  • 매핑 생성
  • 플레이어 매핑 테스트
  • 플레이어 매핑 편집
  • 플레이어 매핑 삭제
참고: 스마트 플레이어와 브라이트코브 플레이어 매핑은 2017년 말에 스마트 플레이어의 수명이 끝난 후에도 계속 작동합니다.

지정 마이그레이션 요구 사항

Directed Migration Tool을 사용할 때 다음 요구 사항이 적용됩니다.

  • Directed Migration UI는 계정에 스마트 플레이어가 3 개 이상있는 경우에만 표시됩니다.
  • Brightcove Player와 호환되는 스타일, 사용자 지정 및 플러그인이있는 새 Brightcove Player를 만들어야합니다.
  • Directed Migration Tool은 재생 목록이있는 플레이어에 대해 지원되지 않습니다.
  • 플레이어가 교체되면 기존 페이지의 모든 Smart Player API 코드가 더 이상 작동하지 않습니다.

한계

Directed Migration Tool의 다음 제한 사항에 유의하십시오.

  • Directed Migration은 Smart Player 포함 게시 코드를 사용하거나 단축 또는 사용자 지정 URL (bcove.me 또는 link.brightcove.com)을 사용할 때 작동합니다.
  • 지정 마이그레이션은 Legacy Studio의 Flash 전용 소스 코드에서는 작동하지 않습니다.

매핑 생성

매핑은 Smart Player를 Brightcove Player로 교체해야 함을 지정하는 데 사용됩니다. 새 매핑을 생성하려면 다음 단계를 따르십시오.

  1. Studio 헤더에서 ADMIN링크를 클릭합니다.
  2. 플레이어 설정링크를 클릭합니다.
  3. 매핑생성을 클릭합니다 . 계정의 스마트 플레이어 목록이 표시됩니다.
  4. Brightcove Player로 교체 할 하나 이상의 Smart Player를 선택하십시오. 다음 을 클릭합니다 .
  5. Smart Player 대신 사용할 Brightcove Player를 선택하십시오. Brightcove Player를 선택하면 플레이어의 미리보기가 표시됩니다.
  6. 다음 을 클릭합니다 . 플레이어 매핑테스트지침이 표시됩니다.

    참고: 플레이어 매핑이 적용되는 데 최대 5 분이 걸릴 수 있습니다.

  7. 저장을 클릭합니다 . 현재 매핑 목록이 표시됩니다.

매핑을 생성한 후 상태는 Testing이 됩니다 . 매핑이테스트되고 커밋되면 상태가커밋됩니다 . [ 커밋됨] 상태로 표시된매핑은 스마트 플레이어 대신 브라이트코브 플레이어를 렌더링합니다.

플레이어 매핑 테스트

커밋되기 전에 생성 된 플레이어 매핑을 테스트해야합니다. 플레이어 매핑을 테스트하려면 다음 단계를 따르십시오.

  1. 기존 Smart Player를 표시하는 페이지를 찾습니다.
  2. 페이지 URL에enableMapping=true매개 변수를 추가합니다.
    • 페이지 URL에 URL 매개 변수가 없는 경우 URL 끝에 추가합니다?enableMapping=true . 예를 들면 다음과 같습니다.
      http://www.mysite.com/videos.html?enableMapping=true
    • 페이지 URL에 URL 매개 변수가 있는 경우 URL 끝에 추가합니다&enableMapping=true . 예를 들면 다음과 같습니다.
      http://www.mysite.com/videos.html?myparams=true&enableMapping=true

    새로 매핑 된 Brightcove Player가 표시되고 비디오가 재생됩니다.

    플레이어의 작동 방식에 만족하면 Video Cloud Studio로 돌아갑니다.

  3. 플레이어 매핑 옆의 확인란을 선택하고 [매핑커밋] 을 클릭하여 모든 사이트 방문자에게 새 브라이트코브 플레이어를 표시합니다.

플레이어 매핑 편집

플레이어 매핑을 편집하려면 다음 단계를 따르십시오.

  1. Studio 헤더에서 ADMIN링크를 클릭합니다.
  2. 플레이어 설정링크를 클릭합니다. 플레이어 매핑 목록이 표시됩니다.
  3. 편집 할 플레이어 매핑 옆의 확인란을 선택합니다. 매핑편집을 클릭합니다 .
  4. 매핑 할 새 Brightcove Player를 선택합니다. 다음, 저장을차례로 클릭하여 변경 사항을 저장합니다.

플레이어 매핑에 대한 변경 사항을테스트해야합니다.

플레이어 매핑 삭제

플레이어 매핑을 삭제하려면 다음 단계를 따르십시오.

  1. Studio 헤더에서 ADMIN링크를 클릭합니다.
  2. 플레이어 설정링크를 클릭합니다. 플레이어 매핑 목록이 표시됩니다.
  3. 삭제할 플레이어 매핑 옆의 확인란을 선택합니다. 매핑삭제를 클릭합니다 .
  4. 삭제를 클릭하여삭제를 확인합니다 .
