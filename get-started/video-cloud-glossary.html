ㅏ

액티브 미디어
계정의 활성 동영상 수입니다. 이 데이터는 실시간이 아니며 하루에 한 번 비디오 라이브러리에서 동기화됩니다. 또한 "실시간"분석 데이터 (3 일 미만)를보고있는 경우이 데이터가 정확하지 않을 수 있습니다.
Analytics API
개발자가 Video Cloud 비디오에 대한 분석 지표를 JSON 형식으로 직접 검색 할 수있는 REST 기반 애플리케이션 프로그래밍 인터페이스입니다. NS 애널리틱스 API Video Cloud Studio에서 Analytics UI에 대한 데이터를 제공합니다.
액세스 토큰
CMS API, Analytics API 및 Dynamic Ingest API와 같은 REST API에 대한 액세스를 허용하는 임시 OAuth2 토큰입니다.
제휴
Video Cloud 플레이어를 신디케이트하는 웹 게시자.
광고 요청
Video Cloud 플레이어에서 광고 서버로 보내는 요청 (호출)입니다. 광고 요청은 HTTP 호출이며 광고 서버가 컨텍스트에 적합한 광고를 타겟팅하는 데 도움이되는 정보를 전달하는 키 / 값 쌍을 포함합니다. 모든 광고 요청이 광고를 반환하는 것은 아닙니다.
Analytics API
NS 애널리틱스 API Video Cloud 계정 및 비디오에 대한 분석 데이터를 프로그래밍 방식으로 얻을 수 있습니다.
Apple HLS
주문형 비디오를 제공하기 위해 Apple에서 구현 한 프로토콜 인 Apple HTTP Live Streaming. Apple HLS는 비디오를 일련의 작은 파일 다운로드로 나누고 각 다운로드는 HTTP를 사용하여 비디오의 짧은 청크를로드합니다. Apple HTTP 라이브 스트리밍은라이브 스트리밍 비디오와는 아무런 관련이 없습니다 .
종횡비
비디오 프레임의 너비와 높이의 비율입니다. 표준 시스템의 경우 4:3, HDTV의 경우 16:9

비

대역폭
오디오 또는 비디오 장비가 인코딩하거나 디코딩 할 수있는 신호 주파수 범위입니다. 비디오는 오디오보다 더 높은 주파수를 사용하므로 더 넓은 대역폭이 필요합니다.
블로그
주기적으로 제공되는 웹 기반 출판물로, 종종 역순으로 표시됩니다.
브라이트코브 플레이어
Brightcove Player는 빠르게로드되고, 데스크톱 및 모바일 플랫폼에서 고품질 적응 형 비트 레이트 HLS 비디오를 제공하고, 선호하는 광고 및 분석 시스템과 통합되는 고성능 크로스 플랫폼 HTML5 우선 비디오 플레이어입니다.
Brightcove Player API
Brightcove Player API는 Brightcove Player의 모양과 동작을 프로그래밍 방식으로 제어하는 데 사용할 수 있습니다.
광대역 비디오 채널
인터넷을 통해 배포되는 관련 비디오 모음입니다.
범퍼
전체 길이 동영상 전 또는 플레이어가로드 될 때 표시 될 수있는 짧은 동영상 클립입니다.

씨

CMS API
NS CMS API비디오 메타데이터에 대한 캐시되지 않은 읽기 및 쓰기 액세스를 제공합니다.
코덱
아날로그 비디오 또는 오디오를 디지털 형식으로 캡처하는 데 사용되는 알고리즘입니다. 와 비교컨테이너 .
컨테이너
비디오 컨테이너는 비디오 및 오디오 데이터와 함께 비디오의 메타 데이터가 저장되는 방식을 정의하는 형식입니다. 데이터의 코딩 또는 압축을 정의하지 않습니다. 예를 들어 MP4는 H.264 파일의 컨테이너가 될 수 있습니다.
콘텐츠 전송 네트워크 (CDN)
콘텐츠 (특히 대용량 미디어 콘텐츠)를 최종 사용자에게 투명하게 전달하는 네트워크 서버를 제공하는 서비스입니다. 두 가지 주요 CDN은 Akamai와 Limelight입니다.
콘텐츠 타겟팅
콘텐츠 타겟은 해당 방문자에 대해 알려진 정보를 기반으로 다른 방문자에게 다른 콘텐츠를 표시하는 데 사용되는 프로세스입니다.
CSS
CSS 스타일 시트는 HTML 페이지의 디자인을 제어하기위한 표준 메커니즘입니다. CSS는 Brightcove 플레이어의 스타일을 지정하는 데 사용됩니다.
큐 포인트
미드 롤 광고 요청 또는 자막과 같은 외부 이벤트를 실행하는 데 사용할 수있는 동영상의 특정 지점에 삽입 된 보이지 않는 마커입니다. 미디어 모듈을 사용하여 큐 포인트를 설정할 수 있습니다.

디

대시 (MPEG-DASH)
HTTP를 통한 동적 적응 스트리밍 (MPEG-DASH라고도 함)은 기존 HTTP 웹 서버에서 제공되는 인터넷을 통해 미디어 콘텐츠의 고품질 스트리밍을 가능하게하는 적응 형 비트 전송률 스트리밍 기술입니다. Apple의 HTTP 라이브 스트리밍 (HLS)솔루션과 마찬가지로 MPEG-DASH는 콘텐츠를 일련의 작은 HTTP 기반 파일 세그먼트로 분할하여 작동합니다.
도메인 제한
특정 도메인을 제외하고 플레이어가 작동하지 못하도록하는 기본 형태의 콘텐츠 보안입니다.
DRM
디지털 권한 관리는 미디어 파일을 암호화하고 패키지화하여 정책 키를 기반으로 만 해독 할 수 있도록하는 방법입니다.
동적 전달
멀티비트레이트 스트리밍을 참조하십시오 .
Dynamic Ingest API
DI (Dynamic Ingest) API는 Video Cloud에 비디오 자산을 검색하고 이에 대한 변환을 생성하도록 지시하는 데 사용됩니다. NS 동적 수집 API캡션, 챕터 또는 자막에 대한 썸네일 및 포스터 이미지 또는 WebVTT 파일을 수집하는 데 사용할 수도 있습니다.

이자형

부호화
비디오 및 오디오 데이터를 플레이어에서 재생할 수있는 스트림으로 변환합니다.
암호화
승인 된 당사자 만 볼 수있는 방식으로 비디오를 인코딩하는 프로세스입니다.

에프

프레임 속도
비디오 클립이 표시되는 초당 프레임 수입니다.

지

지리적 필터링
종종 저작권 소유권을 이유로 동영상에 지리적 제한을 둡니다.

H

H.264 형식
MPEG-4 AVC (Advanced Video Coding)라고도합니다. 인터넷을 통해 비디오를 전달하는 데 사용되는 파일 형식입니다. 온라인 비디오 전송에는 H.264 형식이 권장됩니다. H.264 파일은 일반적으로 MP4 컨테이너를 사용하며.mp4 파일 확장명입니다. 이전 형식보다 훨씬 더 큰 압축을 제공합니다.
HLS
HTTP Live Streaming-클라이언트 측에서 스트리밍되는 짧은 세그먼트에서 비디오를 사용할 수 있도록 Apple에서 만든 비디오 형식입니다.
HTML
(HyperText Markup Language) 웹 브라우저에 표시 할 하이퍼 텍스트 및 기타 정보가 포함 된 웹 페이지를 생성하도록 설계된 마크 업 언어입니다.
HTML5
HTML5는 World Wide Web 콘텐츠를 구성하고 표시하는 데 사용되는 인터넷의 핵심 기술 마크 업 언어입니다. HTML5는 W3C (World Wide Web Consortium)의 HTML 표준의 다섯 번째 개정판입니다.

나는

수집 프로필
인제스트 프로필에는 업로드 모듈 또는 동적 인제스트 API를 사용하여 동영상을 업로드할 때 사용되는 트랜스코드 설정이 포함되어 있습니다.
Ingest Profiles API
Ingest Profiles API를 사용하면 Video Cloud 계정에 대한 수집 프로필을 생성, 검색, 업데이트 및 삭제할 수 있습니다. 인제 스트 프로파일은 Video Cloud에 비디오를 추가 할 때 생성되는 렌 디션을 정의합니다.

제이

자바 스크립트
다른 응용 프로그램에 포함 된 개체에 대한 스크립팅 액세스를 활성화하는데도 사용되지만 웹 사이트에서 사용하는 것으로 가장 잘 알려진 스크립팅 언어입니다.
JPEG (Joint Photographic Experts Group)
디지털 이미지에 사용되는 일반적으로 사용되는 표준 손실 압축 방법입니다. JPEG는 썸네일, 비디오 스틸 및 배경 이미지에 대해 Brightcove 서비스에서 사용되는 표준 이미지 형식입니다.

케이

키 / 값 쌍
플레이어가 광고 요청을 보낼 때 광고 서버에 광고 컨텍스트에 대한 정보를 제공하고 광고 타겟팅을 향상시키는 하나 이상의 키/값 쌍 (예: category=sports) 이 포함됩니다.

엘

우편함
이미지 위와 아래에 공백 (일반적으로 검은 색 막대)을 두어 가로 세로 비율이 다른 화면에 와이드 스크린 비디오를 표시하는 방법입니다.
심벌 마크
회사의 브랜드를 상징적으로 나타내는 독점 그래픽입니다.
로고 오버레이
비디오 재생 창에 표시되는 작은 투명 이미지입니다. 디지털 온 스크린 그래픽 (DOG), 버그 또는 워터 마크라고도합니다.
루프 / 루핑
재생 목록의 동영상이 완료되고 재생 목록이 처음부터 재생되기 시작합니다.
라이브 스트리밍 비디오
VOD를 위해 스트리밍을 위해 인코딩된 파일을 배포하는 것과는 반대로 캡처되고 인코딩된 스트림을 직접 푸시하여 이벤트 비디오를 스트리밍하는프로토콜입니다.

미디엄

미디어 라이브러리
Video Cloud 계정의 비디오, 재생 목록 및 기타 자산 모음입니다.
메타 데이터
메타 데이터는 동영상 정의에 제공하는 설명, 태그, 장르 및 기타 관련 정보를 의미합니다.
MPAA 필름 등급 시스템
미국 영화 협회 (Motion Picture Association of America)에서 제정 한 시스템으로 콘텐츠에 따라 영화를 평가합니다. MPAA 등급 시스템 (G, PG, R 등)은 잠재적으로 불쾌감을 줄 수있는 콘텐츠를 분류하는 데 가장 잘 알려진 시스템이지만 MPAA는 각 등급에 상표가 있으므로 일반적으로 영화 산업에서 사용하도록 제한됩니다.
다중 비트 레이트 스트리밍
여러 버전을 지원하는 기능 또는번역단일 비디오의 경우 시청자의 브라우저 크기 및 대역폭에 가장 적합한 변환을 제공할 수 있습니다. 이전에는이를 "동적 전달"이라고했습니다.
Muxing 및 Demuxing
다양한 오디오 및 비디오 스트림의 쓰기 및 읽기 컨테이너형식.

엔

네이티브 플레이어 SDK
Brightcove Native Player SDK는 iOS 및 Android 용 비디오 지원 앱을 빌드하는 데 필요한 도구를 제공합니다.

영형

OAuth
OAuth는 인증을위한 개방형 표준입니다. OAuth는 클라이언트 애플리케이션에 리소스 소유자를' 대신하여 서버 리소스에 대한 '보안 위임 액세스를 제공합니다.
OAuth API
OAuth API는 Brightcove의 다른 REST API에 액세스하기위한 인증 토큰을 얻을 수있는 RESTful API입니다.

피

재생
Video Cloud 플레이어에서 비디오 미디어 재생.
Playback API
Playback API는 런타임에 모바일 앱 또는 비디오 포털의 비디오 및 재생 목록 데이터에 액세스하는 데 사용할 수 있습니다.
Player Management API

플레이어 관리 API를 사용하여 다음을 수행 할 수 있습니다.

  • 플레이어 만들기
  • 플레이어 설정보기
  • 플레이어 설정 변경
  • 게시자와 관련된 모든 플레이어 나열
  • 소비자가 사용할 수 있도록 플레이어 게시
플레이어 게시 코드
웹 페이지에 Brightcove Player를 포함하는 데 사용되는 코드 "스 니펫"입니다.
재생 목록
Video Cloud 플레이어에서 재생하기 위해 특정 순서로 함께 그룹화 된 비디오 모음입니다.
재생 목록 API
Playlist API는 자바 스크립트를 사용하여 프로그래밍 방식으로 재생 목록을 제어하는 데 사용됩니다.
Policy API
정책 API는 Playback API에 액세스하는 데 사용할 수 있는 정책 키를 생성하거나 가져오는 데 사용됩니다 . Brightcove 플레이어를 위해 정책 키가 자동으로 생성됩니다.
포스터 이미지
비디오 재생 전후에 비디오 플레이어의 재생 영역에 표시되는 정적 이미지입니다.
px
CSS에서 사용되는 픽셀의 약자입니다.

큐

 
 

아르 자형

연주
다중 비트 전송률 스트리밍을 사용하면 각각 다른 비트 전송률과 크기를 가진 동일한 비디오의 여러 버전을 만들 수 있습니다. 이러한 각 버전을 변환이라고합니다. Video Cloud 다중 비트 전송률 스트리밍은 플레이어의 크기와 시청자의 연결 속도를 기반으로 최상의 변환을 선택합니다.
참조 ID
게시자가 동영상 또는 재생 목록에 할당 한 고유 ID입니다. 참조 ID (ref id)는 Video Cloud에서 생성 된 고유 ID 인 비디오 ID 또는 재생 목록 ID와 다릅니다. 참조 ID를 사용하면 Video Cloud와 게시자의 콘텐츠 관리 시스템과 같은 기타 서비스를 조정할 수 있습니다.
RSS 피드

토큰 기반 인증
토큰 기반 인증 시스템을 사용하면 사용자가 사용자 이름과 비밀번호없이 특정 리소스를 가져올 수있는 토큰을 얻기 위해 사용자 이름과 비밀번호를 입력 할 수 있습니다. 토큰을 획득하면 사용자는 주어진 기간 동안 특정 리소스에 대한 액세스를 제공하는 토큰을 리소스에 제공 할 수 있습니다.

비디오

Video Cloud 플레이어에서 재생할 단일 비디오 타이틀입니다. 비디오는 지원되는 여러 비디오 파일 형식 중 하나로 시작할 수있는 비디오 파일을 기반으로하지만 아직 이러한 형식 중 하나가 아닌 경우 H.264 파일 또는 FLV 파일로 트랜스 코딩됩니다. 또한 비디오에는 다음과 같은 메타 데이터가 포함될 수 있습니다.

  • 간단한 설명
  • 장황한 설명
  • 태그
  • 비디오 스틸 이미지
  • 포스터 이미지
  • 범퍼
  • 로고 오버레이
비디오 클라우드 스튜디오

Video Cloud 서비스에 액세스하고 제어하기위한 인터넷 애플리케이션입니다. Studio에는 다양한 작업을 완료하기위한 일련의 모듈이 포함되어 있습니다.

  • 업로드 -Video Cloud 계정에 동영상 업로드
  • 미디어 -비디오 속성 설정, 폴더를 사용하여 비디오 구성 및 비디오 게시
  • 플레이어 -플레이어 생성 및 스타일 지정 및 플레이어 용 게시 코드 생성
  • 해석학 -비디오 및 플레이어 성능에 대한 분석보기
  • 라이브 -라이브 이벤트 생성 및 스트리밍
  • 갤러리 -비디오 콘텐츠를위한 비디오 포털 생성
  • 청중 -Video Cloud의 비디오 분석을 마케팅 자동화 플랫폼에 연결
  • 사회적인 -YouTube, Facebook 및 Twitter에 비디오 배포
