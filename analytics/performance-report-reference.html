분석 데이터는 네이티브 플레이어 SDK에서 제공하는 데이터를 포함하여 Brightcove 플레이어에 의해 자동으로 전송됩니다. 당신이있는 경우 ~ 아니다 Brightcove Player를 사용하여 Video Cloud 비디오를 제공하려면 데이터 수집기로 데이터를 보내는 데 사용하는 플레이어를 계측해야 합니다. 개요를 참조하십시오. 자세한 내용은 데이터 수집 API주제를 참조하십시오.

실적 보고서를 보려면 분석 모듈을 열고성능왼쪽 탐색에서. 보고서에는 선택한 기간 동안 일일 총 동영상 조회수를 보여주는 그래프가 표시됩니다. 드롭 다운 목록을 사용하여 다른 측정 항목을 선택할 수 있습니다.

  • 비디오 조회수 -스트림이 시작될 때 녹화 된 비디오 재생 시작 횟수 (되감기 또는 다시 재생은 포함되지 않음) 개별 시청자의 척도가 아닙니다. 프리 롤 광고가있는 동영상의 경우 동영상 재생이 시작될 때까지 조회수가 기록되지 않습니다.
  • 비디오 노출 -동영상이로드되고 상호 작용할 준비가 된 횟수
  • 재생률 -로드 된 동영상 중 실제로 재생 된 비율 (동영상 조회수 / 동영상 노출 수)
  • 본 분 -동영상 시청 시간 (분)
  • 참여 점수 -동영상 조회 당 평균 시청률 (모든 시청률을 동영상 조회수로 나눈 값을 더하여 계산)
  • 일일 순 시청자 -시청자의 IP 주소와 사용자 에이전트를 사용하여 계산합니다. 브라이트코브는 브라우저에 의해 쿠키가 점점 차단되고 있기 때문에 타사 쿠키를 사용하지 않기로 결정했습니다. Daily Unique Viewers는 계정 또는 비디오 수준에서 사용할 수 있습니다. 이 측정항목은 동영상(또는 다른 항목)으로 필터링할 때 사용할 수 없습니다. Brightcove는 또한 IP 주소 대신 귀하가 제공하는 가치를 바탕으로 고유 한 시청자를 확보 할 수있는 기능을 제공합니다. 자세한 내용은 Analytics 이벤트 추적개발자 문서를 참조하십시오.
  • 비디오 대역폭 -비디오 전용 대역폭을 포함합니다. 대역폭 데이터는 3-5 일 지연됩니다.
  • 총 대역폭 -비디오, 이미지 및 플레이어로드를위한 대역폭을 포함합니다. 대역폭 데이터는 3-5 일 지연됩니다.

기본적으로 보고서는 모든 스트림 유형에 대한 성능 데이터를 표시합니다. 계정이 Brightcove Live에 대해 활성화 된 경우 모든 비디오, VOD 자산 또는 라이브 자산에 대한 성능 데이터를 보도록 선택할 수 있습니다. 스트림 유형필을 클릭하여 선택합니다.

메모:
  • 분석 보고서에는 계정에서 삭제 된 동영상에 대한 분석이 포함됩니다. 분석 모듈에서 삭제 된 동영상을 검색하여 삭제 된 특정 동영상에 대한 보고서를 볼 수 없습니다. 이러한 보고서를 받으려면 삭제된 동영상의 동영상 ID를 알고 Analytics API를 사용하여 보고서를생성해야 합니다.
  • 일일 순 시청자 수 및 비디오 대역폭 데이터를 사용할 수 있으려면 최대 3 일이 걸립니다.
  • 동영상 조회수는 참여 보고서의 조회수 x %와 동일하지 않습니다. 동영상 조회수는 동영상 재생이 처음 시작될 때 계산됩니다. 1 %의 조회수는 첫 번째 비율을 시청할 때 기록됩니다. 여기서 차이점은 첫 번째 백분율이 여러 번 시청되는 경우 (예를 들어 비디오가 자동 재생으로 설정된 경우) 개수는 증가하지만 비디오가 처음 재생되기 시작하면 비디오 보기가 계산되지만 재생에 계산되지 않는다는 것입니다 (사용자가 페이지를 다시 로드하거나 다른 비디오를 시청하지 않는 한재생 목록 등).
  • 동일한 계정 내에서 동시 보고서 요청이 연속적으로 실행됩니다. 보고서의 날짜 범위가 크고 필드를 많이 선택한 경우 모든 결과가 반환되기까지 약간의 지연이있을 수 있습니다. 좋은 대안은 큰 보고서를 예약하는 것입니다.
  • 인페이지환경에서는 동영상이 플레이어에 로드된 것처럼 보이더라도 시청자가 재생 버튼 오버레이를 클릭할 때까지 동영상이 로드되지 않습니다. 따라서 동영상 노출은 시청자가 동영상을 재생할 때까지 Analytics 모듈에 의해 기록되지 않습니다.

이전 기간과 비교

이전 기간과비교확인란을 선택하면 이전 기간의 데이터를 나타내는 오버레이 파선 그래프가 추가됩니다.

이전 기간은 보고서 날짜 범위에 대해 현재 선택된 항목을 기반으로합니다. 예를 들어 지난 7 일의 날짜 범위를 선택하면 이전 기간은 2 주 전이됩니다. 주별 추세를 비교할 때는 전체 주를 포함하는 기간을 선택하는 것이 가장 좋습니다. 예를 들어 월요일에서 시작하는 날짜를 선택하는 경우 종료 날짜를 일요일로 설정합니다. 시작 날짜가 금요일이면 종료 날짜를 목요일로 설정하십시오. 또한 막대와 파선 위에 마우스를 올려 날짜와 해당 값을 볼 수도 있습니다.

실적 보고서 세부 정보

보고서 본문에 표시되는 첫 번째 열은 선택한 보고서 측정 기준 (동영상 이름, 플레이어 이름, 국가, 기기 유형 등)에 따라 달라집니다. 모든 보고서에는 다음 데이터가 표시됩니다.

  • 비디오 조회수 -스트림이 시작될 때 녹화 된 비디오 재생 시작 횟수 (되감기 또는 다시 재생은 포함되지 않음) 개별 시청자의 척도가 아닙니다.
  • 비디오 노출 -동영상이 플레이어에로드되고 상호 작용할 준비가 된 횟수
  • 재생률 -동영상이로드 된 후 재생 한 사용자 비율 (동영상 조회수를 동영상 노출 수로 나눈 값)
  • 본 분 -동영상을 본 총 시간 (분)
  • 참여 점수 -동영상 조회 당 평균 시청률 (모든 시청률을 동영상 조회수로 나눈 값으로 계산)

동영상및요일차원에는 해당 기간 동안 평균일일 고유시청자인 일일 고유 뷰어도 표시됩니다.선택되었습니다. 이것은 시청자의 IP 주소와 사용자 에이전트를 사용하여 계산됩니다.

트래픽 소스

트래픽 소스 별 실적을 볼 때 가능한 트래픽 소스는 다음과 같습니다.

  • 추천 -사용자가 동영상 링크를 따라갔습니다.
  • 곧장 -사용자가 직접 링크를 입력했거나 리퍼러를 확인할 수 없음
  • 보안 검색 - Google 계정으로 로그인하고 검색을 수행한 후 결과를 클릭한 사용자 (유료 또는 유기적, 검색 정보는 이 경우 포함되지 않음)
  • 자연 검색 -사용자가 검색을 수행하고 결과를 클릭하여 비디오로 이동
  • 유료 검색 -사용자가 검색을 수행하고 유료 결과를 클릭하여 동영상으로 이동

비디오 분석보기

비디오차원을 선택하면 선택한 기간 동안 하루에 비디오 보기 수가 그래프에 표시됩니다. 보고서 본문에는 동영상 이름, 동영상 ID, 기간 및 각 동영상의 성능 세부 정보가 나열됩니다.

비디오 링크를 클릭하면 해당 비디오의 성능이 표시됩니다.

참고: Video Cloud 라이브러리에 없는 비디오가 Video Cloud 플레이어에서 재생되는 경우 (예: 원격 애셋 또는 플레이어 모듈의 미리보기 플레이어에 표시되는 자리 표시자 비디오) 관련 비디오 ID 없이기타/서드파티가표시됩니다.

플레이어 분석보기

플레이어차원을 선택하면 선택한 기간 동안 하루에 비디오 보기 수가 그래프에 표시됩니다. 보고서 본문에는 각 플레이어의 성능 세부 정보가 나열됩니다.

플레이어에 대한 링크를 클릭하면 해당 플레이어의 성능이 표시됩니다.

참고: 2016년 2월 이전에는 iOS 또는 Android용 Brightcove Player SDK를 사용하여 빌드된 앱에 사용된 플레이어가 플레이어 이름으로 표시됩니다. 사용자 지정 또는 기본 재생. 2016년 2월부터 앱에 지정된 소스 및 대상 값을 사용하여 이러한 플레이어를 식별할 수 있습니다. 자세한 내용은 iOS또는 Android SDK 설명서를 참조하십시오.

트래픽 분석 소스보기

Source of Traffic차원을 선택하면 선택한 기간 동안 하루에 비디오 보기 수가 그래프에 표시됩니다. 보고서 본문에는 각트래픽 소스에 대한 성능 데이터가 표시됩니다 . 선택한 기간 내의 데이터가있는 범주 만 표시됩니다.

트래픽 소스에 대한 링크를 클릭하면 선택한 트래픽 소스에 대한 트래픽 소스 실적이 표시됩니다.

디스플레이 도메인 분석보기

Display Domain차원을 선택하면 선택한 기간 동안 하루에 비디오 보기 수가 그래프에 표시됩니다. 보고서 본문에는 디스플레이 도메인에 대한 성능 데이터가 표시됩니다. 디스플레이 도메인은 시청자가 동영상을 볼 때 있었던 도메인입니다.

디스플레이 도메인에 대한 링크를 클릭하면 선택한 디스플레이 도메인에 대한 표시 URL이 표시됩니다.

참고: 플레이어가 iframe에 있으면 디스플레이 도메인이 iframe에 페이지의 URL로 기록됩니다.

필터 사용

필터를 사용하여 보고서에 표시되는 데이터를 제한 할 수 있습니다. 필터를 추가하려면 다음 단계를 따르십시오.

  1. 보고서 상단에 있는 + 필터 추가필을 클릭합니다.
  2. 데이터는 다음으로 필터링할 수 있습니다. 태그 , 사용자 정의 필드 , 국가그리고장치 유형 . 필터 패널을 사용하여 필터 기준을 구성하십시오. 다음 예제는 필터를 적용합니다.
    • 태그 = 주문형 교육및
    • 사용자 지정 필드제품 = 비디오 클라우드및
    • 국가 = 미국
  3. 필터적용을 클릭합니다 .

적용된 필터는 보고서 상단에 알약으로 표시됩니다. 각 필은 필터 유형과 최대 두 개의 값을 표시합니다. 최대 세 개의 알약이 표시됩니다.

추가 알약은 오버플로 알약으로 축소되며 마우스를 올리면 툴팁에 표시됩니다.

필터를 제거하려면 알약을 클릭하고 필터 패널에서 필터를 지우십시오.

필터 사용에 대한 참고 사항

  •  스트림 유형은 계정에서 Brightcove Live가 활성화 된 경우에만 사용할 수 있습니다.
  • 사용자 정의 필드에 대해 계정이 활성화되지 않은 경우 사용자 정의 필드 섹션이 비활성화됩니다.
  • 필터 유형 오른쪽에있는 더하기 / 빼기 버튼을 사용하여 필터를 추가 / 제거 할 수 있습니다.
  • 필터가 적용된 것처럼 보이지만 Analytics 모듈의 모든 섹션에 적용되지는 않습니다.
    • 특정 비디오 또는 플레이어로 드릴 한 경우 태그 및 사용자 정의 필드 필터가 적용되지 않습니다 (단, 비디오 별 플레이어 또는 일 차원은 예외 임).
    • 참여 보고서의 참여 그래프
  • 필터를 적용 할 때 분석 데이터를로드하는 데 시간이 조금 더 걸릴 수 있습니다.
  • 필터 유형
    • 모두 - 반드시 포함해야 함모두가치
    • 어느 - 다음을 포함할 수 있음어느가치
    • 없음 - 반드시 포함해야 함없음선택한 값의

보고서 데이터 내보내기

보고서 데이터는 쉼표로 구분 된 값과 Excel 형식으로 다운로드 할 수 있습니다. 내보내기 기능은 에서 직접 데이터를 저장합니다. 애널리틱스 API전화. 내 보낸 데이터는 UI 데이터 방식대로 조작되지 않으며 현재 UI에 표시되는 필드의 하위 집합뿐 아니라 보고서의 모든 기본 데이터를 포함합니다. UI에 표시되는 데이터와 내 보낸 데이터의 차이점은 다음과 같습니다.

  • 다른 단위 (분 대 초)
  • 숫자 반올림
  • 데이터 내보내기에는 사용자의 UI 위치에 관계없이 성과 및 참여 지표가 모두 포함됩니다.
  • 기타/타사항목은 연결된 비디오 ID 또는 비디오 이름 없이 내보내집니다.

API의 모든 데이터를 원래 형식으로 가져 오면 UI에서 제공하는 것과 다른 방식으로 데이터를 조작 할 수 있습니다.
