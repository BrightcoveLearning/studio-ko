참여 보고서를 보려면 분석 모듈을 열고약혼왼쪽 탐색에서 보고서에는 선택한 기간 동안의 총 동영상 참여를 보여주는 그래프가 표시됩니다. 차트 아래의 보고서 전환기를 사용하여 다른 보고서로 빠르게 전환 할 수 있습니다.

보고서 전환기 오른쪽의 드롭 다운 목록을 사용하여 보고서 측정 기준을 선택할 수 있습니다. 참여 보고서에서 다음 측정 기준을 사용할 수 있습니다.

  • 비디오
  • 플레이어
  • 일

페이지 본문에는 참여 점수별로 정렬 된 동영상 목록이 표시됩니다. 보고서 하단에는 표시 할 행 수를 선택하고 다른 페이지로 이동할 수있는 컨트롤이 있습니다.

비디오 참여 계산

참여도는 보기의 척도이며, ~ 아니다시청자. 참여 데이터를보고 할 때 동영상을 100 개의 균등 부분으로 나누고 매 100 분의 조회수를 계산합니다. 100번째 부분을 5번 시청하는 시청자와 5명의 시청자가 1회 시청하는 시청자 사이에는 구별이 없습니다.

측정 값은 표시를 위해 Analytics UI에서 반올림됩니다. 이 수치는 개인별로 조회수를 계산 한 다음 동영상 타임 라인의 비율에 따라 해당 조회수를 정규화 한 결과입니다. 내보내기 API는이 개수의 결과를 반환합니다. 이들은 계산 된 값이므로 특정 자릿수에 대한 값의 중요성을 추정하는 간단한 방법이 없습니다.

동영상의 길이가 고정되어 있지 않기 때문에 실시간 스트림에 대한 참여 통계는 신뢰할 수 없습니다.

참여 점수가 100보다 클 수 있습니다. 동영상을 스크러빙하고 다시 보는 것은 새로운 video_view 이벤트를 트리거하지 않지만 video_engagement를 추적합니다. 예를 들어 페이지 어딘가에 단일 동영상 플레이어에 동영상이 있다고 가정 해 보겠습니다. 다음 이벤트는 비디오 참여가 어떻게 크게 달라질 수 있는지 보여줍니다.

  1. 아직 시계가 없습니다. 참여 점수: 0.
  2. 시청자는 페이지를로드하고 동영상을 끝까지 봅니다. 참여 점수: 100 (100 % 조회 / 1 조회).
  3. 동일한 뷰어가 뒤로 스크럽하고 끝까지 다시 봅니다 (브라우저가 다시로드되지 않음). 참여 점수: 200 (200 % 조회 / 1 조회).
  4. 다른 시청자가 페이지를로드하고 전체 동영상을 봅니다. 참여 점수: 150 (300 % 조회 / 2 조회).

비디오 참여도를 계산하는 방법에 대한 자세한 내용은 비디오 참여계산 교육 비디오를 참조하십시오 .

최근 및 과거 데이터

32일 동안 브라이트코브는 비디오에 대한 자세한 분석 데이터를 보관합니다 (동영상 지속 시간의 100분의 1당 조회수). 그 후 분석 데이터는 기록 데이터로 이동되고 1 %, 25 %, 50 %, 75 % 및 100 % 포인트의보기 만 유지됩니다.

Analytics 모듈에서 내 보낸 보고서에는 지난 32 일 이내의 기간이라도 1 %, 25 %, 50 %, 75 % 및 100 % 포인트의보기 만 포함됩니다. 지난 32일 동안의 더 자세한 참여 데이터를 보려면애널리틱스 API사용되어야 합니다.

참여 보고서 세부 정보

보고서 본문에 표시되는 첫 번째 열은 선택한 보고서 측정 기준 (동영상 이름, 플레이어 이름 또는 요일)에 따라 달라집니다. 모든 보고서에는 다음 데이터가 표시됩니다.

  • 참여 점수 - 동영상의 각 100번째에 대해 기록된 조회수를 기반으로 한 계산
  • 조회수 1 % -동영상의 1 % 지점에서 기록 된 조회수
  • 25 % 조회수 -동영상의 25 % 지점에서 기록 된 조회수
  • 50 % 조회수 -동영상의 50 % 지점에서 기록 된 조회수
  • 75 %에서 조회 -동영상의 75 % 지점에서 기록 된 조회수
  • 100 % 조회수 -동영상의 100 % 지점에서 기록 된 조회수

모든 경우에 x %의 조회수는 동영상의이 세그먼트가 플레이어에서 재생되었음을 의미합니다.

x%의뷰는 실적 보고서의비디오 보기와동일하지 않습니다. 동영상 조회수는 동영상 재생이 처음 시작될 때 계산됩니다. 1 %의 조회수는 첫 번째 비율을 시청할 때 기록됩니다. 여기서 차이점은 첫 번째 백분율이 여러 번 시청되는 경우 (예를 들어 비디오가 자동 재생으로 설정된 경우) 개수는 증가하지만 비디오가 처음 재생되기 시작하면 비디오 보기가 계산되지만 재생에 계산되지 않는다는 것입니다 (사용자가 페이지를 다시 로드하거나 다른 비디오를 시청하지 않는 한재생 목록 등).

참고: 모든 분석 용어 및 정의의 전체 목록은애널리틱스 용어집을 참조하십시오 .
참고: 동일한 계정 내에서 동시 보고서 요청이 연속적으로 실행됩니다. 보고서의 날짜 범위가 크고 필드를 많이 선택한 경우 모든 결과가 반환되기까지 약간의 지연이있을 수 있습니다. 좋은 대안은 대규모 보고서를 예약하는 것입니다.

비디오 차원 분석보기

비디오차원을 선택하면 선택한 기간에 대한 전체 비디오 참여도가 그래프에 표시됩니다. 보고서 본문에는 동영상 이름, 동영상 ID, 기간 및 각 동영상의 참여 세부 정보가 나열됩니다.

동영상 링크를 클릭하면 해당 동영상에 대한 동영상 참여가 표시됩니다.

미리보기 플레이어는 참여 그래프와 동기화되며 참여 급증 또는 감소가 발생하는 비디오에서 볼 수있는 기능을 제공합니다. 참여도 그래프의 아무 지점이나 클릭하면 플레이어가 동영상에서 해당 지점을 재생합니다.

플레이어 차원 분석보기

플레이어차원을 선택하면 선택한 기간 동안의 전체 비디오 참여도가 그래프에 표시됩니다. 보고서 본문에는 각 플레이어의 참여 세부 정보가 나열됩니다.

플레이어에 대한 링크를 클릭하면 해당 플레이어에 대한 비디오 참여가 표시됩니다.

일 차원 분석보기

Day차원을 선택하면 선택한 기간에 대한 비디오 참여가 그래프에 표시됩니다. 보고서 본문에는 매일 참여 세부 정보가 나열됩니다. 보고서 본문에서 드롭 다운 목록을 사용하여 비디오 또는 플레이어별로 비디오 참여를 볼 수 있습니다.

필터 사용

필터를 사용하여 보고서에 표시되는 데이터를 제한 할 수 있습니다. 필터를 추가하려면 다음 단계를 따르십시오.

  1. 보고서 상단에 있는 + 필터 추가필을 클릭합니다.
  2. 데이터는 다음으로 필터링할 수 있습니다. 태그 , 사용자 정의 필드 , 국가및/또는장치 유형 . 필터 패널을 사용하여 필터 기준을 구성하십시오. 다음 예제는 필터를 적용합니다.
    • 태그 = 주문형 교육및
    • 사용자 지정 필드제품 = 비디오 클라우드및
    • 국가 = 미국
  3. 필터적용을 클릭합니다 .

적용된 필터는 보고서 상단에 알약으로 표시됩니다. 각 필은 필터 유형과 최대 두 개의 값을 표시합니다. 최대 세 개의 알약이 표시됩니다.

추가 알약은 오버플로 알약으로 축소되며 마우스를 올리면 툴팁에 표시됩니다.

필터를 제거하려면 알약을 클릭하고 필터 패널에서 필터를 지우십시오.

필터 사용에 대한 참고 사항

  •  스트림 유형은 계정에서 Brightcove Live가 활성화 된 경우에만 사용할 수 있습니다.
  • 사용자 정의 필드에 대해 계정이 활성화되지 않은 경우 사용자 정의 필드 섹션이 비활성화됩니다.
  • 필터 유형 오른쪽에있는 더하기 / 빼기 버튼을 사용하여 필터를 추가 / 제거 할 수 있습니다.
  • 필터가 적용된 것처럼 보이지만 Analytics 모듈의 모든 섹션에 적용되지는 않습니다.
    • 특정 비디오 또는 플레이어로 드릴 한 경우 태그 및 사용자 정의 필드 필터가 적용되지 않습니다 (단, 비디오 별 플레이어 또는 일 차원은 예외 임).
    • 참여 보고서의 참여 그래프
  • 필터를 적용 할 때 분석 데이터를로드하는 데 시간이 조금 더 걸릴 수 있습니다.
  • 필터 유형
    • 모두 - 반드시 포함해야 함모두가치
    • 어느 - 다음을 포함할 수 있음어느가치
    • 없음 - 반드시 포함해야 함없음선택한 값의

보고서 데이터 내보내기

보고서 데이터는 쉼표로 구분 된 값과 Excel 형식으로 다운로드 할 수 있습니다. 내보내기 기능은 에서 직접 데이터를 저장합니다. 애널리틱스 API전화. 내 보낸 데이터는 UI 데이터 방식대로 조작되지 않으며 현재 UI에 표시되는 필드의 하위 집합뿐 아니라 보고서의 모든 기본 데이터를 포함합니다. UI에 표시되는 데이터와 내 보낸 데이터의 차이점은 다음과 같습니다.

  • 다른 단위 (분 대 초)
  • 숫자 반올림
  • 데이터 내보내기에는 사용자의 UI 위치에 관계없이 성과 및 참여 지표가 모두 포함됩니다.

API의 모든 데이터를 원래 형식으로 가져 오면 UI에서 제공되는 것과 다른 방식으로 데이터를 조작 할 수 있습니다.
