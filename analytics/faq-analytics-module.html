1. Analytics 모듈이란 무엇입니까?

Video Cloud Studio Analytics 모듈은 비디오 및 플레이어에 대한 거의 실시간 분석 정보를 보는 데 사용할 수있는 사용자 인터페이스 모듈입니다. 분석 플랫폼은 빅 데이터 기술의 최신 혁신을 사용하며 대규모로 설계되었습니다. 분석 플랫폼은 분석 데이터에 대한 빠르고 실시간에 가까운 액세스를 제공하며 개발자가 Video Cloud Studio에 포함 된 것 외에도 사용자 지정 분석 솔루션 및 보고서를 만들 수 있도록하는 강력한 개방형 API를 포함합니다.

2. 분석 모듈에서 보고서 데이터를 내보낼 수 있습니까?

예. 분석 보고서에는 보고서 데이터를.csv 및.xlsx 파일로 다운로드하는 데 사용할 수 있는 링크가 있습니다.

3. 분석 모듈에서 데이터가 얼마나 자주 업데이트됩니까?

일반적으로 실시간에 가까운 데이터는 약 15 분 지연됩니다. 이 시간은 현재 데이터 수집 시스템에서 수신하는 데이터의 양에 따라 다를 수 있지만 경우에 따라 최대 3 시간까지 지연 될 수 있습니다.

Analytics 시스템은 여러 소스의 데이터를 컴파일하며 실시간에 가까운 데이터는 지금까지 수집 된 데이터를 기반으로 한 근사치입니다. 수치는 보고서마다 다를 수 있으며 정확한 수치가 아닌 추세로보아야합니다.

4. 보고 또는 API에 내부 동영상 ID를 포함하는 방법이 있습니까?

내부 ID를 Video Cloud의 비디오에 매핑하는 데 사용할 수있는 비디오 속성으로 사용할 수있는 참조 ID 필드가 있습니다. 참조 ID 필드는 사용자 정의 보고서를 만들 때 표시 될 수 있습니다. 사용자 지정 보고서에 대한 자세한 내용은 Analytics 모듈에서 사용자 지정 분석 보고서만들기를 참조하십시오 .

5. 동영상 앞에 프리 롤 광고가 있고 시청자가 몇 초 동안 광고를 본 후 나가면 어떻게 되나요? 시청자가 주요 콘텐츠를 전혀 보지 않았다면 동영상 조회수로 집계 되나요?

아니요. 동영상 조회수는 기본 콘텐츠 재생이 시작된 후에 만 계산됩니다.

6. 분석에서 IP 주소 범위를 필터링 할 수 있습니까?

현재 IP 주소 범위를 필터링하는 기능은 사용할 수 없습니다.

7. 분석은 라이브 이벤트 스트리밍으로 플레이어를 추적합니까?

Analytics는 실적 보고서의 일부로 조회 시간을 추적합니다. 사용자 지정 보고서를 만들 때 또는 결제 모듈을 통해 본 라이브 초를 표시 할 수 있습니다. 라이브 이벤트 대역폭이 표시되지 않습니다.

8. 분석 모듈을 통해 특정 비디오에 대한 정보를 찾으려면 어떻게합니까?

검색 기능을 사용하여 특정 비디오를 검색 할 수 있습니다. 비디오 이름, 간단한 설명, 자세한 설명, 태그, 비디오 ID, 참조 ID 및 사용자가 보유한 모든 사용자 지정 텍스트 필드가 검색됩니다. 결과를 클릭하면 해당 동영상에 대한 성과 또는 참여도가 표시됩니다. 지정된 비디오 ID에 대한 분석을 표시하기 위해 사용자 지정 보고서를 만들 수도 있습니다.

9. 분석 모듈에서 참조 ID를 사용하여 특정 비디오를 검색 할 수 있습니까?

예. 검색 기능을 사용하여 참조 ID 필드를 기반으로 비디오를 검색 할 수 있습니다. 비디오 이름, 간단한 설명, 자세한 설명, 태그, 비디오 ID 및 사용자가 가지고있는 모든 사용자 지정 텍스트 필드도 검색됩니다.

10. 고객 참여를보고하고 특정 비디오를 중단하려면 어떻게해야합니까?

참여 보고서는이 정보를 보여줍니다. 참여 보고서를 선택하고 동영상 측정 기준을 선택하면 계정에 대한 전반적인 참여를 보여주는 그래프가 표시됩니다. 비디오를 선택하면 시간 경과에 따른 해당 비디오에 대한 참여를 표시하는 그래프가 표시됩니다.

11. URL에 대해보고 할 수 있습니까? 예를 들어 A 페이지와 B 페이지에 동일한 동영상이 호스팅되어있는 경우 두 게재 위치 간의 실적을 비교할 수 있습니까?

예. 성능 보고서는 디스플레이 도메인별로 비디오 분석을 볼 수있는 기능을 제공합니다.

12. 동영상이 반복되는 경우 동영상이 재생 될 때마다 한 번의 동영상 조회 또는 여러 번의 조회수로 집계 되나요?

동영상 조회수는 스트림이 시작될 때 시청 세션 당 한 번만 계산됩니다. 참고로조회수 x%참여 보고서에서비디오 보기실적 보고서에서 동영상 조회수는 동영상 재생이 처음 시작될 때 계산됩니다. 1 %의 조회수는 첫 번째 비율을 시청할 때 기록됩니다. 여기서 차이점은 첫 번째 백분율이 여러 번 시청되는 경우 (예를 들어 비디오가 자동 재생으로 설정된 경우) 개수는 증가하지만 비디오가 처음 재생되기 시작하면 비디오 보기가 계산되지만 재생에 계산되지 않는다는 것입니다 (사용자가 페이지를 다시 로드하거나 다른 비디오를 시청하지 않는 한재생 목록 등).

13. 내 계정에 대한 순 시청자 수를 보여주는 보고서를 실행할 수 있습니까?

순 시청자 수는 실적 보고서에서 확인할 수 있습니다.

14. Analytics 모듈에서 사용할 수있는 표준 보고서를 사용자 지정할 수 있습니까?

성과 및 참여 보고서에 표시되는 데이터는 사용자 정의 할 수 없습니다. 그러나 분석 모듈은 표시 할 데이터를 선택할 수있는 사용자 정의 분석 보고서를 생성하는 기능을 제공합니다. 도 있다애널리틱스 API프로그래밍 방식으로 분석 데이터에 액세스하는 데 사용할 수 있습니다.

15. 일부 참여 점수가 100보다 큰 매우 높은 이유는 무엇입니까?

참여 점수가 100보다 클 수 있습니다. 동영상을 스크러빙하고 다시 보는 것은 새로운 video_view 이벤트를 트리거하지 않지만 video_engagement를 추적합니다. 예를 들어 페이지 어딘가에 단일 동영상 플레이어에 동영상이 있다고 가정 해 보겠습니다. 다음 이벤트는 비디오 참여가 어떻게 크게 달라질 수 있는지 보여줍니다.

  1. 아직 시계가 없습니다. 참여 점수: 0.
  2. 시청자는 페이지를로드하고 동영상을 끝까지 봅니다. 참여 점수: 100 (100 % 조회 / 1 조회).
  3. 동일한 뷰어가 뒤로 스크럽하고 끝까지 다시 봅니다 (브라우저가 다시로드되지 않음). 참여 점수: 200 (200 % 조회 / 1 조회).
  4. 다른 시청자가 페이지를로드하고 전체 동영상을 봅니다. 참여 점수: 150 (300 % 조회 / 2 조회).

비디오 참여도를 계산하는 방법에 대한 자세한 내용은 비디오 참여계산 교육 비디오를 참조하십시오 .

16. 조회수 통계가 조회수 1 %보다 25 % 높은 이유는 무엇입니까?

두 숫자에 불일치를 일으킬 수있는 몇 가지 요인이 있습니다.

  • 조회수 1 %와 조회수 x %는 서로 다른 두 가지 측정 항목입니다. 1 %의 조회수는 시청 세션별로 고유하며 되감기 및 다시 재생을 포함하지 않습니다. 시청자가 동영상의 이전 부분을 다시 재생하거나 되감 으면 % 포인트는 시청자 수를 증가 시키지만 동영상 조회수는 증가하지 않습니다.
  • %의 조회수 및 참여 곡선은 UTC 시간을 사용하여 일일 단위로 저장되므로 UTC와보고 시간대가 교차하는 경우 특정 날짜에 2 일 분량의 데이터가 포함될 수 있습니다.

17. 특정 날짜의 동영상 조회수가 시간에 따라 달라지는 이유는 무엇입니까?

분석 시스템은 매시간 수집 된 조정되지 않은 최근 지표를 제공하고 일일 수준에서보고 된 실제 지표와 자동으로 조정됩니다. 이 조정은 가능한 한 빨리, 일반적으로 3 ~ 5 일 내에 이루어집니다. 우리는 시간 단위로 수집 된 데이터를 조정하지 않습니다. 게시자는 시간별 데이터를 중요하게 생각하므로 시간별 미 조정 데이터를 32 일 동안 사용할 수 있습니다. 화해에 대한 자세한 내용은 다음과 같습니다.

  • 플레이어는 데이터 수집 API에 데이터를 보냅니다.
  •  데이터 수집 API는 실시간 데이터베이스에 쓰기 위해 최선을 다하고 이벤트를 로그 스토리지에 씁니다.
  • CDN 및 기타 소스에서 추가 데이터를 수집하면 분석 데이터가 업데이트됩니다. 이것이 화해 과정입니다
  • 야간 배치 시스템은 원시 로그 이벤트를보고 기록 데이터 가져 오기 파일을 작성합니다.
  • 데이터 가져 오기 파일이 기록 데이터베이스에로드됩니다. API 호출을 할 때 조정 된 데이터 만 볼지 (가장 정확한지) 또는 reconciled 매개 변수를 사용하여 조정 된 데이터와 조정되지 않은 데이터를 모두 볼지 선택할 수 있습니다. 자세한 내용은애널리틱스 API 개요를참조하십시오.
  • 모든 과거 데이터가 조정됩니다. 실시간 데이터 (32 일 동안 사용 가능)는 아직 조정되었거나 조정되지 않았을 수 있습니다. 데이터를받는 기간이 32 일 이전에 종료 된 경우에는 다음을 사용할 필요가 없습니다. reconciled data, since the data will all be historical.

18. 일부 메트릭이 null 값을 반환하거나 값이없는 이유는 무엇입니까?

이는 요청한 기간 동안 메트릭을 사용할 수없는 경우 발생합니다. 일부 측정 항목의 지연 시간은 2 ~ 3 일이므로, 예를 들어 지난 30 일 (오늘 포함) 동안이 데이터를 요청한 경우 요청 된 범위에 아직이 데이터가없는 날이 포함되므로 이러한 측정 항목은 null 일 수 있습니다. 데이터를 요청할 때이를 방지하려면 요청 된 범위가 대기 시간을 벗어 났는지 확인하십시오.

또한 시간이 지남에 따라 추가 측정 항목과 측정 기준 조합을 추가했습니다. 예를 들어 다년 보고서를 실행하는 경우 전체보고 기간에 대해 해당 측정 항목을 추적하지 않았기 때문에 일부 측정 항목이 비어 있거나 부분적으로 표시 될 수 있습니다. 유효한 날짜 및 차원 조합에 대한 자세한 내용은 Analytics API 개요문서를 참조하십시오.

19. 2013 년의 대상 도메인 데이터가 다른 성능 데이터와 일치하지 않는 이유는 무엇입니까?

대상 도메인 기능은 2014년 1월까지 켜지지 않았습니다. 2013년 3월부터 2014년 1월까지의 데이터는 부분적으로 채워졌지만 이 기간 동안 일부 대상 데이터만 사용할 수 있습니다. 2014년 1월 이후의 데이터가 완성되었습니다.

20. 삭제 된 동영상에 대한 분석이 포함 되나요?

분석 보고서에는 계정에서 삭제 된 동영상에 대한 분석이 포함됩니다. 그러나 분석 모듈에서 삭제 된 동영상을 검색하여 삭제 된 특정 동영상에 대한 보고서를 볼 수는 없습니다. 이러한 보고서를 얻으려면 삭제된 동영상의 동영상 ID를 알고 Analytics API를 사용하여 보고서를생성해야 합니다. 보고서 데이터를 내보내는 경우 삭제 된 동영상에 대한 데이터가 동영상 ID 또는 이름을 표시하지 않고 표시 될 수 있습니다.

21. 분석 보고서에서 직원보기를 제외 할 수 있습니까?

이를 수행하는 쉬운 방법은 없으며 트래픽이 많은 비디오의 경우 직원 조회수가 분석에 미치는 영향이 무시할 정도로 적을 것입니다. 그러나이 작업을 수행하는 것이 중요하다고 생각하는 트래픽이 적은 비디오의 경우 가장 간단한 솔루션은 프로덕션 플레이어를 복제하고 직원이 복사본에서 비디오를 테스트 /보기하도록하는 것입니다. 그런 다음 필터를 사용하여 Analytics 모듈 또는 Analytics API의 사용자 정의 보고서 기능을 사용하여 프로덕션 플레이어에 대한 보고서 만 만들 수 있습니다.

22. 참여를 위해 항상 정수가 표시되지 않는 이유는 무엇입니까?

참여 번호가 소수점으로 반환 될 수 있습니다. 그 이유는 참여가 정규화되어 'video_percent_viewed * (video_engagement_25 / video_engagement_sum)'비율이므로 검색어에 대해 선택한 시간 범위를 기준으로 정확히 나누지 않는 경우 부동 소수점 숫자를 볼 수 있기 때문입니다.

23. 일부 보고서에 동영상 ID 만 표시되는 이유 또는 일부 동영상이 전혀 표시되지 않는 이유는 무엇입니까?

일정 기간 동안 동영상에 대한 조회수가 없는 경우 보고서에 광범위한 데이터가 표시되지 않습니다 (예: 동영상 ID만 표시되고 그 밖에 없음). 데이터가 전혀 다운로드되지 않으면 (즉, 이미지 데이터도 아님) 비디오가 보고서에 표시되지 않습니다.

24. 분석 데이터를 얻을 수있는 가장 빠른 날짜는 언제입니까?

분석 데이터의 가장 빠른 날짜는 2010년 12월 28일입니다. 일부 데이터 항목은 나중에 수집되지 않았으므로 데이터를 볼 수있는 가장 빠른 날짜는보고있는 항목에 따라 달라집니다. 비디오 조회수는 2010년 12월 28일로 다시 볼 수 있습니다.

25. 분석 모듈과애널리틱스 API동일한 날짜 범위에 대해 동일한 데이터를 제공하시겠습니까?

예.하지만 상대적으로 최근 데이터를 비교하고 Studio와 API에서 서로 다른 시간에 데이터를 가져 오는 경우에는 다를 수 있습니다.

  1. 데이터가 3 일 미만입니다. 3 일 미만의 분석 데이터는 잠정적이며 언제든지 변경 될 수 있습니다.
  2. 동일한 기간의 데이터는 32 일 미만일 때와 32 일 후에 다시 조회됩니다. 32 일 후 데이터는 별도의 "이력"저장소로 이동되고이 시점에서 일부 데이터, 특히 세부 참여 데이터가 삭제됩니다. 이는 참여도 점수와 같은 계산 된 필드가 덜 세분화 된 데이터로 계산되기 때문에 약간 변경 될 수 있음을 의미합니다.
