플레이어 제작 및 퍼블리싱을 빠르게 시작하려면단계별 정보를 확인하세요. 플레이어 모듈을 사용하여 플레이어주제를 만들고 게시합니다.

플레이어 모듈 사용

플레이어 모듈에 액세스하려면 Studio에 로그인하고선수탐색 헤더에서. Video Cloud Studio는 아래와 같습니다. Brightcove Player Studio UI에는 약간의 차이가 있습니다.

페이지 본문은 다음과 같이 표시됩니다.

  • 선수 이름
  • 플레이어 ID
  • 업데이트 된 날짜 및 시간
  • 플레이어 상태
    • - 플레이어에 변경 사항이 적용되었지만 변경 사항이 게시되지 않았음을 나타냅니다.
    • - 플레이어가 활성화되고 플레이어에 대한 모든 변경 사항이 게시되었음을 나타냅니다.
    • - 플레이어가 비활성 상태임을 나타냅니다.

플레이어 모듈을 사용하여 다음을 수행 할 수 있습니다.

  • 새 플레이어 만들기
  • 플레이어 속성 편집
  • 플레이어 복제
  • 플레이어 삭제
  • 선수 검색
  • 플레이어 미리보기
  • 플레이어의 소스 코드 및 URL 생성

새 플레이어 만들기

Player 모듈을 사용하여 새 플레이어를 생성하려면

  1. [ 새 플레이어]버튼을 클릭합니다.
  2. 플레이어의 이름과 간단한 설명을 입력한 다음저장을 클릭합니다 .
  3. 새 플레이어가 플레이어 목록에 나타납니다.
참고: 최대 200 명의 플레이어를 만들 수 있습니다. 브라이트코브는 많은 수의 플레이어를 만드는 것을 권장하지 않지만, 필요한 경우 계정 관리자에게 연락하여 만들 수 있는 플레이어 수를 늘리도록 요청하십시오.

플레이어 모듈을 사용하여 새 플레이어를 만들면 플레이어는 Brightcove에서 게시 한 최신 플레이어 버전을 사용하여 빌드됩니다. 플레이어 버전은 플레이어 모듈 홈페이지에 표시됩니다. 모든 플레이어가 동일한 버전이 아닐 수 있습니다. Brightcove가 Brightcove 플레이어를 계속 개발하고 개선함에 따라 새로운 플레이어 버전이 출시 될 것입니다. 기본적으로 업데이트는 모든 플레이어에게 자동으로 적용됩니다. 플레이어의 자동 업데이트를 방지하려면업데이트 모드설정을수동 (플레이어 잠금이라고도 함) 으로 설정해야 합니다. 자세한 내용은 플레이어 업데이트 모드 및 버전설정을 참조하십시오 .

플레이어 속성 편집

플레이어의 모양과 느낌을 사용자 지정하도록 구성 할 수있는 다양한 플레이어 속성이 있습니다.

  • 선수 정보
  • 스타일링
  • 재생
  • 통제 수단
  • 소셜
  • 플러그인
  • 최종 화면
  • 광고하는
  • 콘텐츠 제한
  • 운동
  • JSON 편집기
  • 타사 분석

플레이어 속성을 편집하려면 왼쪽 탐색 메뉴에서 항목을 클릭합니다. 플레이어 속성 구성에 대한 자세한 내용은 플레이어 속성설정을 참조하십시오 .

참고: 플레이어에 대한 변경 사항은 게시된 플레이어에게 나타나는 데 최대 5분이 소요될 수 있습니다.

플레이어 복제

플레이어를 복제하면 기존 플레이어를 기반으로 새 플레이어를 만들 수 있습니다. 플레이어 모듈에서 플레이어를 복제하려면

  1. 복제하려는 플레이어 옆의 확인란을 선택합니다.
  2. 복제버튼을 클릭합니다.
  3. 플레이어의이름과간단한 설명을입력합니다.
  4. 저장을 클릭합니다 .

플레이어 삭제

플레이어를 삭제하면 계정에서 제거되며 취소 할 수 없습니다. 플레이어 모듈에서 플레이어를 삭제하려면 다음과 같이 하십시오.

  1. 삭제하려는 플레이어 옆의 확인란을 선택하십시오.
  2. 삭제버튼을 클릭합니다.
  3. 삭제를 클릭하여삭제를 확인합니다 .

선수 검색

검색 필드는 계정의 플레이어를 검색하는 데 사용할 수 있습니다. 검색 할 때 다음 필드가 검색됩니다.

  • 선수 이름
  • 플레이어 ID
  • 간단한 설명

플레이어 미리보기

플레이어를 미리 보려면 플레이어 목록에서 플레이어를 선택한 다음플레이어 미리 보기를 클릭합니다 . 플레이어가 미리보기 대화 상자에서 열립니다.

플레이어의 소스 코드 및 URL 생성

Embed Code & URL버튼은미리 보기 플레이어또는게시된 플레이어에 대한 플레이어 임베드 코드와 URL을 생성할 수있습니다. 일반적으로 미리보기 플레이어 소스 코드 및 URL은 플레이어의 변경 사항을 미리 보는 데 사용됩니다. 이를 통해 플레이어를 프로덕션 사이트에 게시하기 전에 변경 사항을 확인할 수 있습니다. 플레이어가 게시되면 해당 특정 플레이어의 모든 인스턴스에 변경 사항이 반영됩니다.

미리 보기 플레이어와 게시된 플레이어의 차이점과 플레이어의 내장 코드 생성에 대한 정보는 다음을 참조하십시오. 플레이어 임베드 생성 암호 .

키보드 단축키

? 를 누르면? Players 모듈의 키를 누르면 사용 가능한 단축키 목록이 열립니다.
