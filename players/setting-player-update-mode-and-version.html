Brightcove Player의 주요 장점은 사용자 지정 기능입니다. 사용자 지정을 통해 브랜드별 사용자 인터페이스 요소 및 색상, 대화형 요소, 타사 광고 또는 분석 시스템과의 통합 등을 수행할 수 있습니다. 그러나 사용자 지정은 정기적 인 품질 보증 및 릴리스 프로세스의 일부로 플레이어를 Brightcove 테스트 버전과 다르게 만들 수도 있습니다. 플레이어를 사용자 정의하고 새 버전이 플레이어 또는 사이트 동작을 방해하지 않도록하려는 고객을 위해 Brightcove는 고객에게 정기적 인 자동 버전 업데이트를 옵트 아웃 할 수있는 기능을 제공합니다.

플레이어 설정의 일부로업데이트 모드및플레이어 버전을 구성할수 있습니다. 업데이트 모드는 플레이어 업데이트 처리 방법을제어합니다. 플레이어버전은 플레이어 버전을 설정하는 데 사용됩니다. 업데이트 모드및플레이어 버전을 제어할 수 있으면 다음과 같은이점이 있습니다.

  • 게시자는 API를 사용하지 않고도 출시 전 플레이어 버전을 사용해 볼 수 있습니다.
  • 게시자는 플레이어 업데이트시기를 제어 할 수 있습니다.

플레이어 업데이트 프로세스

Brightcove는 고객에게 가능한 한 빨리 최신 기능과 버그 수정을 제공하기 위해 최선을 다하고 있습니다. 브라이트코브는 애자일 딜리버리 분야의 일환으로 기능 향상 및 버그 수정 사항을 2주마다 릴리스할 수 있는 증분 업데이트로 나누기 위해 노력하고 있습니다. 릴리스 프로세스는 단위 테스트, 자동화 된 통합 및 회귀 테스트, 그리고 페이지가 테스트 환경에서 로컬로 수정되는 고객 사이트의 대표적인 샘플에 대한 수동 테스트로 구성됩니다. 새 버전이 모든 테스트를 통과 한 후 Brightcove는 시험판으로 릴리스합니다. 이를 통해 고객은 플레이어 관리 API를 사용하거나 플레이어 모듈의 플레이어 버전 선택기 드롭 다운을 사용하여 명시 적으로이 버전을 선택할 수 있습니다. 이 시점에서 명시 적으로 새 버전을 선택한 플레이어를 제외하고는 어떤 플레이어도 업데이트되지 않습니다. 기본 자동 업데이트 모드를 사용 중이고 다음 글로벌 업데이트에서 사이트가 제대로 작동하는지 확인하려는 경우 새 버전으로 특수 테스트 플레이어를 만들고 해당 사이트의 테스트 또는 스테이징 버전에서 사용해 볼 수 있습니다. 출시 전 기간.

주요 버전 업데이트

주 버전 업데이트는 버전 식별자의 첫 번째 숫자에 대한 변경으로 표시됩니다 (예: 6.42.5의 주 버전은6입니다 . 주 버전이 변경되면 API 작동 방식이 크게 변경되어 기존 통합이 중단 될 수 있습니다. 이 때문에 사이트 문제를 방지하기 위해 주요 버전 업데이트에 대해자동 업데이트가 이루어지지 않습니다 . 즉, 자동 업데이트 모드를 선택한 경우에도 주 버전의 최종 릴리스에서 플레이어가 효과적으로 잠 깁니다.

관리 릴리스

Brightcove는 일반적으로 관리형 릴리스가 시작되기 전에 고객이 약 1주일 정도 시험해 볼 수 있습니다. 관리 릴리스는 프로덕션 웹 사이트에서 제어 된 A / B 테스트를 실행하는 프로세스입니다. 관리 형 출시 시스템을 사용하면 짧은 기간 (보통 몇 시간) 동안 적은 비율의 페이지로드로 새 버전의 플레이어를로드 할 수 있습니다. 이 기간 동안 Brightcove는 이전 버전과 새 버전의 성능을 비교할 수 있도록 데이터를 수집합니다. 오류 분포, 조회 시간 (분), 광고 시작 횟수 및 기타 여러 측정 항목이 모두 검사됩니다. Brightcove는 데이터의 분포 수와 요약뿐만 아니라 추적 된 지표에서 평균보다 큰 차이가있는 사이트 인 이상 점도 살펴 봅니다. 모든 데이터를주의 깊게 검토 한 후 업데이트가 커밋되거나 롤백됩니다. 롤백하기로 결정한 경우 식별 된 문제를 수정하기 위해 소규모 업데이트를 수행 한 다음 관리 릴리스가 다시 실행됩니다.

플레이어 업데이트 모드 설정

플레이어업데이트 모드를편집하려면 다음 단계를 따르십시오.

  1. 플레이어모듈을 엽니다.
  2. 플레이어를 클릭하여 플레이어 속성을 엽니 다.
  3. 왼쪽 탐색 메뉴에서플레이어 정보를 클릭합니다.
  4. 업데이트 모드를 선택합니다 . 자동및수동모드를 모두 사용할 수 있습니다.

    자동 플레이어 업데이트

    자동모드가 기본 모드입니다. 자동은 Brightcove가 플레이어 버전을 "현재 기본"상태로 승격 할 때마다 플레이어를 업데이트합니다. 자동을 선택하면플레이어 버전옵션은항상 최신 릴리스및항상 최신 시험판 버전입니다. .

    참고: 최신 시험판 플레이어 버전에 대한 자세한 내용은브라이트코브 플레이어 사전 출시문서를 참조하십시오.

    수동 플레이어 업데이트

    수동모드에서는 플레이어가 자동으로 업데이트되지 않습니다. 이를 플레이어 "잠금"이라고도합니다. 플레이어에 대해이 옵션을 선택하면 해당 플레이어는 관리 릴리스 프로세스의 일부가 아니며 선택한 버전으로 유지됩니다. 이 모드를 선택하면 더 이상 자동 버그 수정이나 기능 향상을받을 수 없으므로 신중하게 생각해야합니다. 어느 시점에서 이전 플레이어 버전은 더 이상 지원되지 않습니다. 수동 모드는 기존 버전의 플레이어와 최신 버전 간의 충돌을 해결하는 동안 임시 해결책으로 만 사용해야합니다. 언제설명서모드가 선택되면 특정 플레이어 버전을 사용하여 선택할 수 있습니다. 플레이어 버전드롭 다운 목록.

    수동 업데이트를 선택하는 이유

    • 고급 광고 구성 -IMA 또는 FreeWheel 플러그인을 사용 중이고 맞춤 구성 매개 변수 설정 또는 맞춤 플러그인 또는 페이지 로직에서 자바 스크립트를 사용하여 플러그인과 상호 작용하는 등 Studio UI를 통해 불가능한 광고 동작의 구성 또는 확장을 만든 경우, 이는 다음과 같을 수 있습니다. 플레이어를 잠그는 이유.
    • 타사 플러그인 - 타사에서 제공하는 애널리틱스 또는 기타 플러그인을 사용하는 경우, 이러한 플러그인은 브라이트코브에 의해 유지되지 않으며 플레이어의 JavaScript 이벤트 동작의 사소한 변경에 민감할 수 있음을 인식합니다. Brightcove는 이러한 플러그인 사용자가 플레이어를 잠글 것을 권장합니다.
    • 중요한 CSS 변경 -중요한 CSS 변경의 예로는 레이아웃 변경, 크기 조정, 요소 위치 지정 또는 업데이트 후 레이아웃, 크기 조정 또는 위치 변경을 견딜 수없는 변경이 있습니다. 안전한 CSS 변경의 예: 텍스트, 배경색 변경 또는 레이아웃에 영향을 주지 않는 방식으로 글꼴 변경
    참고: 플레이어가 버전에서 잠겨 있으면 해당 플레이어와 연결된 플러그인은 자동으로 업데이트되지 않으며 각 플러그인의 현재 버전에서도 잠깁니다.
  5. 저장을 클릭합니다 .

플레이어에 대한 변경 사항을 게시해야합니다.

플레이어 버전 설정

플레이어버전은 플레이어 버전을 설정하는 데 사용됩니다. 업데이트 모드가 수동이고 새 플레이어 버전을 사용할 수있는 경우 메시지가 표시됩니다.

플레이어 버전을편집하려면 다음 단계를 따르십시오.

  1. 플레이어모듈을 엽니다.
  2. 플레이어를 클릭하여 플레이어 속성을 엽니 다.
  3. 왼쪽 탐색 메뉴에서플레이어 정보를 클릭합니다.
  4. 업데이트 모드가자동으로설정된 경우플레이어 버전은 다음과 같을수 있습니다.
    • 항상 최신 릴리스 -이 옵션을 사용하면 새 릴리스 버전을 사용할 수있을 때 플레이어 버전이 최신 릴리스 버전으로 자동 업데이트됩니다.
    • 항상 최신 시험판 -이 옵션을 선택하면 새로운 시험판 버전을 사용할 수있을 때 플레이어 버전이 최신 시험판 버전 (또는 사용 가능한 시험판 버전이없는 경우 릴리스 버전)으로 자동 업데이트됩니다.
  5. 업데이트모드가수동으로 설정된 경우 특정플레이어 버전을 선택합니다 .
  6. 저장을 클릭합니다 .

플레이어에 대한 변경 사항을 게시해야합니다.

지원하다

Brightcove 기술 지원은 모든 버전의 Brightcove Player를 사용하여 현재 지원 계약을 맺은 모든 고객에게 지원 및 조언을 제공합니다. 그러나 버그 수정은 주요 버전의 최신 개정에만 적용됩니다. 주 버전이 출시 된 후 특정 버그 수정은 최대 6 개월 동안 이전 주 버전의 마지막 개정에 적용됩니다.

Brightcove는 이러한 옵션이 사이트에서 가능한 최상의 플레이어 경험을 보장하는 동시에 최대 사용자 지정 옵션을 제공하는 데 필요한 도구를 제공하기를 바랍니다.
