플레이어 모듈은 미리보기 및 게시 된 플레이어 모두에 대해 플레이어 소스 코드와 플레이어 URL을 생성하는 기능을 제공합니다. 일반적으로 런타임에 비디오를 플레이어에 동적으로 할당하려는 경우 플레이어 모듈을 사용하여 플레이어에 대한 소스 코드를 생성합니다. 플레이어에게 비디오를 동적으로 할당하는 방법에 대한 자세한 내용은 플레이어에동적으로 비디오 할당을 참조하십시오 .

재생 목록 또는 단일 비디오를 플레이어에 게시하려는 Video Cloud 고객 인 경우 Media 모듈 내의 게시 기능을 사용하십시오. 자세한 내용은 미디어 모듈을 사용하여 비디오 및 재생 목록게시를 참조하십시오 .

플레이어와 게시 된 플레이어 미리보기

변경 사항을 게시하기 전에 미리 볼 수 있도록 미리보기 플레이어가 사용됩니다. 변경 사항이 수용되면 플레이어를 게시 할 수 있으며 게시 된 모든 플레이어에 변경 사항이 반영됩니다. 미리보기 플레이어는 제작에 사용하지 않아야 함을 나타내는 Brightcove 로고 오버레이를 표시합니다.

게시 된 플레이어가 최적화되었으며 웹 페이지에 표시되는 플레이어입니다. 이 플레이어에는 플레이어 설정에 대한 모든 변경 사항이 포함되어 있습니다. 게시단추.

플레이어 모듈을 열고 플레이어를 선택합니다. 딸깍 하는 소리코드 및 URL 삽입클릭플레이어 미리보기또는게시된 플레이어 .

소스 코드 유형

생성 할 수있는 소스 코드에는 두 가지 유형이 있습니다.

  • 표준 (iframe) -대부분의 경우에 권장됩니다. 플레이어를<iframe>태그 안에 넣습니다. 플레이어 크기가 지정되면 iframe 크기가 플레이어 크기로 조정됩니다. 플레이어 크기를 지정하지 않으면 iframe을<div>태그 안에 넣을 수 있으며 플레이어의 반응형 특성은 플레이어의 전체 공간을 차지하도록 크기를 조정합니다. <div> .
  • 고급 (I N 페이지 ) - 플레이어가 iframe이 아닌 HTML 페이지에 직접 존재하도록 허용합니다. 이를 통해 개발자는 플레이어 및 관련 속성 및 이벤트에 쉽게 액세스 할 수 있습니다. 고급 임베드 코드를 사용하면 소셜 미디어 응용 프로그램 및 공유에 포함 된 단일 단위로 플레이어를 쉽게 사용할 수 없습니다. 이 옵션은 게시 된 플레이어에만 사용할 수 있습니다.

미리보기 플레이어 용 소스 코드 생성

미리 보기 플레이어에 대한 임베드 코드를 생성하려면

  1. 플레이어 모듈을 엽니 다.
  2. 코드를 생성하려는 플레이어의 행을 클릭하십시오. 플레이어 이름을 클릭하지 말고 속성 페이지를 열지 마십시오.
  3. 코드 및 URL 포함 을 클릭하고 플레이어미리 보기를 클릭합니다 .
  4. 적절한 소스 코드를 선택하십시오.

게시 된 플레이어에 대한 소스 코드 생성

게시된 플레이어에 대한 임베드 코드를 생성하려면 다음과 같이 하십시오.

  1. 플레이어 모듈을 엽니 다.
  2. 코드를 생성하려는 플레이어의 행을 클릭하십시오.
  3. 코드 및 URL 포함 을클릭하고게시된 플레이어를 클릭합니다 .
  4. 적절한 소스 코드를 선택하십시오.

게시된 플레이어 임베드 코드는 플레이어 이름을 클릭한 다음 게시 및 임베드 ( Publish & Embed) 를 클릭하여 복사할 수도있습니다.

임베드 코드에 애플리케이션 ID 추가

과거에는 게시자가 일반적으로 사이트의 다른 부분에서 사용하거나 다른 사이트에서 사용하기 위해 다른 이름을 가진 새 플레이어를 만들었습니다. 플레이어 분석을 검토 할 때 다른 플레이어 이름을 사용하면 어떤 플레이어가 가장 많은 조회수를 얻고 있는지 쉽게 확인할 수 있습니다. 임베드 코드에 애플리케이션 ID 매개 변수를 추가하면 동일한 플레이어를 여러 사이트에서 사용할 수 있으며 애플리케이션 ID를 사용하여 플레이어가 게시 된 위치를 식별 할 수 있습니다. 플레이어 애플리케이션 ID를 사용하면 생성 및 유지 관리해야하는 플레이어 수를 줄일 수 있습니다. 포함 코드에 응용 프로그램 ID를 추가하는 방법에 대한 자세한 내용은 플레이어 포함 코드에 응용 프로그램 ID 추가를 참조하십시오 .
