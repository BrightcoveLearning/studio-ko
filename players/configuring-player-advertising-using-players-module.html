광고 속성은 Brightcove Player에 대한 클라이언트 측 및 / 또는 서버 측 광고를 활성화하는 데 사용됩니다.

  • 클라이언트 측 광고는 HTML5 버전 3 용 Google의 IMA (Interactive Media Ads)를 지원합니다. 이를 통해 플레이어에 대한 VAST 광고를 요청하고 추적 할 수 있습니다. 구글 IMA에 대한 자세한 내용은 IMA HTML5 SDK 버전 3 사용문서를 참조하십시오.
  • 서버 측 광고는 광고를 비디오 콘텐츠 스트림에 바로 연결하므로 스트리밍 비디오를 재생할 수있는 모든 장치에서 작동합니다. SSAI가 없으면 VAST, VMAP 및 VPAID를 사용하는 IMA 또는 FreeWheel과 같은 클라이언트 측 광고는 브라우저에 광고 차단기가있는 경우 재생되지 않습니다. SSAI에 대한 자세한 내용은비디오 클라우드 SSAI 개요문서를 참조하십시오. SSAI에 대해 계정이 활성화되어 있지 않으면 서버 측 옵션이 표시되지 않습니다.

광고 서버 테스트 (클라이언트 측 광고)

클라이언트 측 광고 속성을 구성 할 때 광고 서버 URL이 필요합니다. Brightcove는 광고 서버 URL의 유효성을 확인할 것을 권장합니다. 재생되지 않는 광고와 관련된 대부분의 문제는 광고 태그 자체의 문제로 인한 것임을 확인했습니다. 광고 태그가 유효한지 확인하면 광고가 문제없이 재생됩니다. Google은 광고 태그를 확인하는 데 사용할 수있는 Video Suite Inspector를 제공합니다. 광고 태그를 확인하려면 다음 단계를 따르세요.

  1. 비디오 Suite 인스펙터페이지를 엽니다.
  2. 입력 유형 양식 입력 필드에 광고 태그 URL을 붙여 넣습니다.
  3. 테스트 광고 를 클릭합니다 .
  4. 광고가 표시되는지 확인하세요. 광고 태그가이 테스트 환경에서 작동하지 않으면 Brightcove Player에서 작동하지 않습니다.

광고 속성은 Brightcove Player를 HTML5 버전 3 용 Google의 IMA (Interactive Media Ads)와 통합하는 데 사용됩니다. 이를 통해 플레이어에 대한 VAST 광고를 요청하고 추적 할 수 있습니다. 구글 IMA에 대한 자세한 내용은 IMA HTML5 SDK 버전 3 사용문서를 참조하십시오.

클라이언트 측 광고 속성 구성

플레이어에 대한 클라이언트 측 광고를 구성하려면 다음 단계를 따르십시오.

  1. 딸깍 하는 소리선수탐색 헤더에서 플레이어 모듈을 엽니다.
  2. 구성하려는 플레이어의 링크를 클릭하십시오.
  3. 딸깍 하는 소리광고하는왼쪽 탐색 메뉴에서
  4. 클라이언트측 (IMA) 활성화 를선택합니다.
  5. 광고 태그를 입력합니다 .
  6. 광고요청설정을 선택합니다.
    • 로드 시 - 플레이어가 로드되면 즉시광고가 요청됩니다 (일반적으로 DFP/VPAYED에서 가장 좋은 경험입니다)
    • 재생중 - 재생이 시작될 때까지 첫 번째 광고 요청이 지연됩니다.
    • 주문형 -모든 광고 요청은player.ima3.adrequest()메서드를 사용하여 프로그래밍 방식으로 시작됩니다.
    • 큐 포인트 - 동영상에서광고 큐 포인트가 발견되면 광고 요청이 이루어집니다.
  7. VPADE 모드를 선택합니다 . VPAID 모드는 IMA 광고에서 VPAID 2 지원을 활성화하는 데 사용됩니다.
    • 사용 - 다른 도메인의 iframe에서 VPAID 광고재생
    • 안전하지 않은 - 동일한 도메인의 iframe에서 VPAID 광고재생
    • 비활성화 됨- VPAID 광고에서 오류가 발생 함
  8. 시간 초과값 설정 -재생하기 전에 광고가 초기화될 때까지 대기할 최대 시간 (밀리초) 입니다.
  9. 확인하다하드 타임아웃느린 로딩 광고가 표시되는 것을 방지합니다. 이 옵션을 선택 취소하면 광고로드 속도가 느려지고 동영상 재생이 중단 될 수 있습니다.
  10. 최대 리디렉션 수를 설정합니다 . 후속 리디렉션이 거부되고 광고로드가 중단되기 전 최대 리디렉션 수를 지정합니다. 리디렉션 수는 지연 시간과 사용자 경험에 직접적인 영향을 미칩니다.
  11. 사용할 IMA 플러그인버전을 선택합니다.
  12. 저장을클릭하여 변경 사항을 저장합니다.

IMA 플러그인은 UI에서 사용할 수없는 추가 속성을 지원합니다. 필요한 경우 필요한 다른 옵션을 포함하도록 IMA 플러그인 JSON을 수정합니다. IMA 플러그인별 옵션 및 광고 프레임워크 옵션에 대한 자세한 내용은 IMA3 플러그인을 사용한광고개발자 문서를 참조하십시오.

서버 측 광고 구성

서버 측 광고를 사용하는 경우 하나 이상의 광고 구성을 만들어야합니다. 광고 구성은 광고 호출, 비콘 및 기타 구성 옵션을 포함하여 SSAI 재생의 다양한 측면을 정의합니다. 계정은 여러 구성을 가질 수 있으며 현재 구성은 계정간에 공유 할 수 없습니다. Admin UI의 Server-Side Ad Setting 메뉴는 광고 구성을 만들고 관리하는 데 사용됩니다. 자세한 내용은 서버 측 광고 설정구성을 참조하십시오 .

플레이어에 대한 클라이언트 측 광고를 구성하려면 다음 단계를 따르십시오.

참고: 서버 측 광고 옵션은 계정이 서버 측 광고에 사용하도록 설정된 경우에만 나타납니다.
  1. 서버 측 (SSAI) 활성화 를선택합니다.
  2. 구성선택드롭다운을 사용하여 이 플레이어와 연결할 광고 구성을 선택합니다.
  3. 광고 위에 오버레이를 표시하려면 광고 정보 오버레이사용확인란을 선택합니다. 여기에는 '자세히 알아보기'및 광고 카운트 다운 오버레이가 포함됩니다.
  4. 저장을 클릭합니다 .
