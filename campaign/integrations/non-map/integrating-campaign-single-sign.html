캠페인 모듈은 시청자가 SSO(Single Sign-On)를 사용하여 시스템에 로그인했을 때 보기 활동을 추적하는 기능을 제공합니다. 이 시나리오에서 데이터보기는 마케팅 자동화 플랫폼으로 전송되지 않습니다. Campaign SSO 구성을 사용하면 뷰어를 식별하기 위해 SAML 속성이 Campaign에 전달됩니다. 이에 대한 일반적인 사용 사례 중 하나는 내부 교육 비디오입니다. 직원은 비디오를 시청하기 위해 로그인해야 할 수 있으며 시청 활동을 추적한 다음 캠페인 모듈 내에서 볼 수 있습니다. SSO와 함께 사용하도록 Campaign을 활성화하려면 다음 단계가 필요합니다.

  1. Single Sign-On 소프트웨어 구성
  2. SSO 속성에 액세스하도록 애플리케이션 구성
  3. 캠페인 SSO 연결 설정
  4. 통합 구성
  5. 캠페인 지원 플레이어 만들기
  6. 비디오 데이터보기

SSO 소프트웨어 구성

SSO 소프트웨어를 설치하고 구성하는 단계는이 항목의 범위를 벗어납니다. 사용자가 Video Cloud 비디오를보기 전에 자신을 인증해야하도록 SSO 소프트웨어를 구성해야합니다. SSO 소프트웨어는 외부 애플리케이션에서 액세스 할 수 있도록 일부 사용자 속성을 노출하도록 구성되어야합니다. 예를 들어, 이 Okta SSO 구성은 세 가지 속성FirstName , LastName및Email .

이 주제에서는Email속성은 캠페인 모듈에 대한 뷰어를 식별하는 데 사용됩니다.

참고: SSO 소프트웨어가 여러 속성을 노출하도록 구성할 수 있지만 Campaign은 단일 속성만 사용하여 최종 사용자를 추적합니다.

SSO 속성에 액세스하도록 애플리케이션 구성

Video Cloud 비디오를 보는 데 사용할 애플리케이션 또는 웹사이트는 JavaScript를 사용하여 Campaign에서 SSO 속성을 사용할 수 있도록 구성해야 합니다. 이 구성은 사용중인 응용 프로그램 또는 웹 사이트에 따라 다릅니다. 다음 예에서는 갤러리에 SSO 액세스 제어 프로필이 생성되었습니다. 구성의 일부로Email SSO 속성이 노출되고 있으며 갤러리 사이트 내에서 사용할 수 있습니다. 캠페인 모듈에 대한 뷰어를 식별하는 데 사용되는 값입니다.

참고: Gallery에서 액세스 제어 프로파일을 구성하는 방법에 대한 자세한 내용은 액세스 제어 프로파일구성을 참조하십시오 .
참고: 갤러리는 여러 속성을 노출하도록 구성할 수 있지만 Campaign은 단일 속성만 사용하여 시청자를 추적합니다.

연결 설정

다음 단계에 따라 Video Cloud와 SSO 공급자 간의 연결을 설정하십시오.

Video Cloud와 SSO 간의 연결 설정

  1. 캠페인 모듈을 열고데이터 연결왼쪽 탐색에서
  2. 새 연결추가를 클릭합니다 .
  3. 선택하다 SSO통합으로.
    참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
  4. (선택 사항) 연결 이름을 입력합니다 .
  5. 연결 을 클릭합니다 .

이제 SSO 연결이 설정되어 사용할 준비가되었습니다.

통합 구성

SSO 연결이 설정되면 통합을 구성 할 수 있습니다.

  1. 클릭empty링크 SSO 속성부분. 페이지에서 찾을 JavaScript 값의 이름을 입력하십시오. 이 예에서 Gallery는Email SSO 속성을 노출하도록 구성되었으므로 여기에서 사용되고 있습니다.

    이 주제에서는 캠페인 SSO 통합을 사용하여 보기 활동을 추적합니다. 저장을 클릭합니다 .

    참고: SSO 속성을비워 둔 다음추가 옵션 중 하나를 사용하여 뷰어를식별할 수도 있습니다.
  2. 기본적으로연결 상태는활성화됩니다 . 연결 상태에따라 Video Cloud 비디오 참여 데이터가 저장되는지 여부가 결정됩니다.
    • 활성화 됨 -데이터 수집
    • 장애인 -데이터가 수집되지 않음 (통합이 비활성화 됨)
  3. 데이터 보존 정책을 선택하십시오. 데이터 보존 정책은 보기 데이터 및 리드 양식 데이터(캠페인 리드 양식을 사용하는 경우)가 Video Cloud 계정에 보존되는 기간을 결정합니다. 개인 정보 보호 정책에 따라 보존 설정을 조정해야합니다.

통합을 제거하려면연결 끊기를 클릭합니다 . 페이지 하단에는 모든 캠페인 지원 플레이어, 연결된 연결 및 연결된 리드 양식이 나열됩니다.

캠페인 모듈은 마케팅 플랫폼에 대한 다중 연결을 지원합니다. 추가 통합을구성하려면 새 연결 추가를 클릭합니다.

시청자 식별을위한 추가 옵션

값을 지정하지 않으면 SSO 속성통합이 구성되면 뷰어를 식별할 책임이 있습니다. 캠페인 모듈에는 다음 정보가 필요합니다.

기술 URL 매개 변수 매개 변수 포함 값
GUID 란 무엇입니까 (보기 이벤트 추적에 필요) bcssoguid data-bc-sso-guid string

이 매개 변수는 URL 매개 변수 또는 플레이어 소스 코드를 통해 전달 될 수 있습니다. 시청자 추적이 발생하려면이 매개 변수가 동영상을 표시하는 페이지에서 URL 매개 변수로 전달되어야합니다.

참고: 플레이어 / 플러그인이 URL 매개 변수를 읽으려면 고급 (인 페이지) 소스 코드를 사용해야합니다. 플러그인이 iframe 내부에서 실행되므로 표준 (iframe) 포함 코드가 작동하지 않으므로 매개 변수가 있는 상위 페이지 URL에 액세스할 수 없습니다.

페이지에서 사용할 수 있는 JavaScript 값을 사용하여 캠페인 플레이어에 GUID를 전달할 수도 있습니다. 플레이어 구성을 사용하면 JavaScript 변수의 이름이 무엇인지 Campaign에 알릴 수 있으며 Campaign은 이를 사용자 정의 GUID로 사용하여 해당 변수를 읽으려고 시도합니다. 이렇게 하려면 캠페인 플레이어를 편집하고 캠페인 플러그인에 대한 플러그인 옵션을 구성합니다.

위의 예에서 플레이어 구성은 페이지에 JavaScript 개체가 있음을 Campaign에 알립니다. samlAttributes라는 속성이 있습니다. nameID . Campaign은 이를 읽고 사용자 지정 GUID에 해당 값을 사용하려고 시도합니다.

비디오 데이터보기

SSO 통합을 사용할 때 비디오 데이터를 보는 데 사용할 수있는 몇 가지 옵션이 있습니다.

  • 시청자 활동(이벤트) - 캠페인 모듈의 일부로 제공되는 이 옵션은 캠페인 모듈 내부의 비디오 데이터를 보는 데 사용할 수 있습니다. 자세한 내용은 다음을 참조하십시오. 캠페인 모듈에서 플레이어 이벤트 보기 .
  • 뷰어 프로필 - 캠페인 모듈의 일부로 제공되는 이 옵션은 캠페인 모듈 내부의 사용자 프로필 값을 기반으로 비디오 데이터를 보는 데 사용할 수 있습니다. 자세한 내용은 뷰어 프로필 정보보기를 참조하십시오 .
  • 데이터 내보내기 - 캠페인 모듈의 일부로 제공되는 이 옵션은 비디오 보기 및 리드 데이터를 내보내는 데 사용할 수 있습니다. 자세한 내용은 다음을 참조하십시오. 캠페인 모듈에서 캠페인 데이터 내보내기 .
  • Audience API -Audience API를 사용하여 비디오보기 및 리드 데이터를 프로그래밍 방식으로 검색 할 수 있습니다. 자세한 내용은개요를 참조하십시오. 오디언스 API .
