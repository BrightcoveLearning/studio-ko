캠페인 모듈을 사용하면 Video Cloud가 캠페인 지원에서 보기 데이터를 동기화할 수 있습니다.   Brightcove Player를 인기 있는 마케팅 자동화 플랫폼으로 전환합니다. CampaignCampaign 모듈은 사용자 정의 통합으로 구성할 수도 있습니다. 사용자 정의 통합을 사용하면보기 데이터가 Video Cloud에 로컬로 저장되고 Audience API는 프로그래밍 방식으로 데이터를 추출하는 데 사용됩니다.

캠페인 모듈 통합은 다음 두 가지 주요 구성 요소로 구성됩니다.

  • 비디오보기 데이터를 추적 할 수있는 Brightcove Player 플러그인
  • Audience API를 사용하여보기 데이터를 검색하는 사용자 지정 개발

요구 사항

캠페인 모듈을 사용하려면 다음을 수행해야 합니다.

  • Professional 또는 Enterprise Video Cloud 계정 보유
  • 캠페인 모듈을 열 때 나타나는 초기 시작 화면에서 필수 정보를 제출하여 계정에 대해 캠페인 모듈을 활성화하십시오.
  • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
  • 리드를 수집하려면리드 양식을구성하고 플레이어에 할당해야 합니다.
  • 플레이어를 게시 할 때 고급 (인 페이지) 소스 코드를 사용하십시오. 플러그인이 iframe 내에서 실행되므로 표준 (iframe) 소스 코드가 작동하지 않으므로 상위 페이지 URL 매개 변수에 액세스 할 수 없습니다.

연결 설정

사용자 정의 통합을 구성하려면 다음 단계를 따르십시오.

  1. 캠페인 모듈을 열고데이터 연결왼쪽 탐색에서
  2. 새 연결추가를 클릭합니다 .
  3. 선택하다관습통합으로.
    참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
  4. (선택 사항) 연결 이름을 입력합니다 .
  5. 연결 을 클릭합니다 .

이제 사용자 지정 연결이 설정되어 사용할 준비가되었습니다.

통합 구성

연결이 설정되면 통합을 구성 할 수 있습니다.

  1. 기본적으로연결 상태는활성화됩니다 . 연결 상태에따라 Video Cloud 비디오 참여 데이터가 저장되는지 여부가 결정됩니다.
    • 활성화 됨 -데이터가 수집되어 저장됩니다.
    • 장애인 -데이터가 수집되지 않음 (통합이 비활성화 됨)
  2. 데이터 보존 정책을 선택하십시오. 데이터 보존 정책은 보기 데이터 및 리드 양식 데이터(캠페인 리드 양식을 사용하는 경우)가 Video Cloud 계정에 보존되는 기간을 결정합니다.
    참고: 보존없음 을 선택하면 데이터가 저장되지 않습니다. API를 사용하여 데이터를 다운로드하면 아무 것도 없습니다.

추적 ID 전달

캠페인 모듈은 추적 ID를 전달하는 기능을 지원합니다. 예를 들어 추적 ID는 캠페인 ID를 추적하는 데 사용될 수 있습니다. 추적 ID는 캠페인 지원 플레이어를 표시하는 URL에 추가하거나 플레이어 내장 코드에 추가할 수 있습니다.

  • 예: URL에 추가됨
  http://www.myurl.com/video.html?
  • 예: 플레이어 소스 코드에 추가됨
<video data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js"  data-bc-trackingid="12345"통제 수단>< /동영상>< 스크립트 src="//players.brightcove.net/1486906377/default_default/index.min.js">< /스크립트>

시청자 식별

사용자 지정 통합을 사용하는 경우 뷰어를 식별해야합니다. 캠페인 플러그인에는 두 가지 주요 정보가 필요합니다.

기술 URL 매개 변수 매개 변수 포함 값
뷰어를 알고 있습니까 (리드 양식에 필요) bcknownuser data-bc-known-user 0또는1
GUID 란 무엇입니까 (보기 이벤트 추적에 필요) bccustomguid data-bc-custom-guid string
 

이러한 값은 URL 매개 변수 또는 플레이어 소스 코드를 통해 전달할 수 있습니다. 시청자 추적이 발생하려면 사용자 ID가 동영상을 표시하는 페이지에서 URL 매개 변수로 전달되어야합니다. 리드 양식에는 알려진 사용자 매개 변수가 필요합니다 (알 수없는 뷰어에게 조건부로 리드 양식을 표시하려는 경우).

참고: 플레이어 / 플러그인이 URL 매개 변수를 읽으려면 고급 (인 페이지) 소스 코드를 사용해야합니다. 플러그인이 iframe 내부에서 실행되므로 표준 (iframe) 포함 코드가 작동하지 않으므로 매개 변수가 있는 상위 페이지 URL에 액세스할 수 없습니다.

  • 시청자 활동(이벤트) - 캠페인 모듈의 일부로 제공되는 이 옵션은 캠페인 모듈 내부의 비디오 데이터를 보는 데 사용할 수 있습니다. 자세한 내용은 다음을 참조하십시오. 캠페인 모듈에서 플레이어 이벤트 보기 .
  • 데이터 내보내기 - 캠페인 모듈의 일부로 제공되는 이 옵션은 비디오 보기 및 리드 데이터를 내보내는 데 사용할 수 있습니다. 자세한 내용은 다음을 참조하십시오. 캠페인 모듈에서 캠페인 데이터 내보내기 .
  • Audience API -Audience API를 사용하여 비디오보기 및 리드 데이터를 프로그래밍 방식으로 검색 할 수 있습니다. 자세한 내용은개요를 참조하십시오. 오디언스 API .
