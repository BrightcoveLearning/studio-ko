캠페인 모듈을 사용하면 Video Cloud에서 Brightcove Player의 시청 데이터를 Marketo로 동기화할 수 있습니다. Marketo에서이 데이터는 세분화, 캠페인 로직, 보고서, 리드 스코어링 및 개인화 커뮤니케이션에 사용될 수 있습니다.

참고: Munchkin ID를 사용하여 Marketo 와 함께 사용하도록 캠페인 모듈을 구성하는 방법에 대한 자세한 내용은 Munchkin을 사용하여 Marketo 비디오 클라우드와통합 단원을 참조하십시오 .

Marketo와 캠페인 모듈 통합은 두 가지 주요 구성 요소로 구성됩니다.

  • 비디오보기 데이터를 추적 할 수있는 Brightcove Player 플러그인
  • Video Cloud에서 Marketo로 추적 데이터를 전송하는 동기화 프로세스

요구 사항

Marketo REST 통합과 함께 Campaign 모듈을 사용하려면 다음을 수행해야 합니다.

  • 모든 API 권한에 대한 액세스 권한이있는 Marketo에서 역할을 생성합니다.
  • LaunchPoint 서비스 생성
  • Professional 또는 Enterprise Video Cloud 계정 보유
  • 캠페인 모듈을 열 때 나타나는 초기 시작 화면에서 필수 정보를 제출하여 계정에 대해 캠페인 모듈을 활성화하십시오.
  • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
  • 플레이어를 게시 할 때 고급 포함 (페이지 내) 게시 코드를 사용하십시오. 플러그인이 iframe 내에서 실행되므로 표준 삽입 (iframe) 게시 코드가 작동하지 않으므로 상위 페이지 URL 매개 변수에 액세스 할 수없고 올바른 쿠키를 얻지 못할 수 있습니다.

API 전용 사용자 및 LaunchPoint 서비스 만들기

Marketo REST API를 호출하기 전에 모든 API에 대한 액세스 권한이있는 새 API 역할을 생성하고 API 전용 사용자를 생성 한 다음 클라이언트 애플리케이션을 고유하게 식별하는 서비스를 생성해야합니다. 이러한 작업을 완료하는 단계는 Marketo 문서에 간략하게 설명되어있습니다.

연결 설정

Marketo 측의 구성 단계가 완료되면 다음 단계에 따라 Video Cloud와 Marketo 간의 연결을 설정합니다.

Video Cloud와 Marketo 간의 연결 설정

  1. 캠페인 모듈을 열고데이터 연결왼쪽 탐색에서
  2. 새 연결추가를 클릭합니다 .
  3. 선택하다마케토통합으로.
    참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
  4. (선택 사항) 연결 이름을 입력합니다 .
  5. 연결 을 클릭합니다 .
  6. 표준 (REST API)을 선택한 다음저장을 클릭합니다 .
    참고: Munchkin 통합 사용에 대한 자세한 내용은 Munchkin을 사용하여 Marketo 비디오 클라우드와통합을 참조하십시오 .

    Marketo 계정에서 여러 값을 붙여 넣어야합니다. 다음 단계를 완료하려면 Marketo 관리자로 로그인해야합니다. 다음 단계의 경우 빈 링크를 클릭하고 적절한 값을 제공하십시오.

  7. 마케토 계정 ID - 이 값을 얻으려면 Marketo 클릭관리자 > 완성 > 먼치킨 .
  8. Marketo REST API 엔드포인트 URL - 이 값을 얻으려면 Marketo 클릭관리자 > 완성 > 웹 서비스 .
  9. Marketo REST API ID URL - 이 값을 얻으려면 Marketo 클릭관리자 > 완성 > 웹 서비스 .
  10. Marketo LaunchPoint 클라이언트 ID - 이 값을 얻으려면 Marketo 클릭관리자 > 완성 > 런치포인트그런 다음세부 정보보기이 설정의 일부로 이전에 생성한 LaunchPoint에 대한 링크입니다.
  11. Marketo LaunchPoint 클라이언트 시크릿 - 이 값을 얻으려면 Marketo 에서관리 > LaunchPoint 를클릭한 다음이 설정의 일부로 이전에 생성한 LaunchPoint에 대한 세부 정보 보기링크
  12. 설정 을 클릭합니다 .

이제 Marketo에 대한 Video Cloud 연결이 설정되어 사용할 준비가되었습니다.

참고: Marketo 사용자에게 올바른 권한이 할당되지 않은 경우 Studio에 오류가 표시됩니다. 권한을 확인한 다음 다시 시도하십시오.

통합 구성

Video Cloud와 Marketo 연결이 설정되면 통합을 구성 할 수 있습니다.

  1. 기본적으로연결 상태는활성화됩니다 . 연결 상태에따라 비디오 클라우드 비디오 참여 데이터가 Marketo 로 전송되는지 여부가 결정됩니다.
    • 활성화 됨 -데이터가 수집되어 Marketo로 전송됩니다.
    • 장애인 -데이터가 수집되지 않음 (통합이 비활성화 됨)
  2. 기본적으로 알 수 없는 뷰어에 대한 데이터추적은사용 안 함으로 설정됩니다 . 이 옵션을 활성화하면 알 수없는 사용자의 데이터를 추적합니다. 데이터는 선택한 데이터 보존 정책에 따라 보존됩니다. 데이터 보존이 보존없음으로 설정된경우 이 필드는 표시되지 않습니다.
  3. 데이터 보존 정책을 선택하십시오. 데이터 보존 정책은 캡처 한 리드 정보 데이터를 Marketo뿐만 아니라 Video Cloud 계정에 보존할지 여부와 그 기간을 결정합니다. Video Cloud에 데이터를 보관하면 중단 된 동기화에서 복구 할 수 있습니다. 하나, 개인 정보 보호 정책에 따라 보존 설정을 조정해야합니다. -Video Cloud에 사용자 데이터를 보관할 필요가 없습니다. 데이터는 데이터 보유 기간이 지나면 삭제됩니다.

Marketo 와의 통합을 제거하려면연결 끊기를 클릭합니다 .

시청자 식별

Marketo 통합이 있는 캠페인 지원 플레이어는 사용자의 브라우저에서 생성되는 Munchkin 쿠키(_mkto_trk)를 참조합니다. 이 값은 캠페인 모듈의 보기 데이터에서 캡처되며 보기 데이터를 Marketo와 동기화하는 데 사용됩니다.

사용자의 이메일 주소는 비디오를 표시하는 페이지의 bcemail URL 매개 변수로 전달할 수 있습니다. 자세한 내용과 예는 다음을 참조하십시오. 캠페인 모듈이 시청자를 식별하는 방법 .

통합 확인

통합이 구성되면 통합을 테스트하여 비디오보기 데이터가 Marketo로 전송되는지 확인합니다. 통합 확인에 대한 자세한 내용은 Marketo REST 통합 테스트 가이드를 참조하십시오 .
