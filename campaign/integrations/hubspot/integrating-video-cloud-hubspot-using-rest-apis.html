캠페인 모듈을 통해 Video Cloud는 시청 데이터를 Brightcove Player에서 HubSpot으로 동기화할 수 있습니다. HubSpot에 있으면이 데이터를 세분화, 캠페인 논리, 보고서, 리드 스코어링 및 개인화 커뮤니케이션에 사용할 수 있습니다.

Video Cloud를 HubSpot과 통합하기위한 두 가지 옵션이 있습니다.

  • 표준(REST API) 방법 사용 - 이 방법을 사용하면 Campaign은 알려진 시청자의 시청 데이터만 추적합니다. 기본적으로 리드 양식은 알 수없는 뷰어에게만 표시됩니다. 이 방법은이 항목에서 다룹니다.
  • 대용량 사용(클라이언트 측 API) 메서드 - 이 방법을 사용하면 Campaign은 모든 시청자의 시청 데이터를 추적합니다. 이러한 유형의통합에 대한 자세한 내용은 클라이언트측 API를 사용하여 HubSpot과 비디오 클라우드 통합 단원을 참조하십시오 .

HubSpot과 캠페인 모듈 통합은 두 가지 주요 구성 요소로 구성됩니다.

  • 비디오보기 데이터를 추적 할 수있는 Brightcove Player 플러그인
  • Video Cloud에서 HubSpot으로 추적 데이터를 전송하는 동기화 프로세스

요구 사항

HubSpot에서 캠페인 모듈을 사용하려면 다음을 수행해야 합니다.

  • HubSpot 계정 보유
  • Professional 또는 Enterprise Video Cloud 계정 보유
  • 캠페인 모듈을 열 때 나타나는 초기 시작 화면에서 필수 정보를 제출하여 계정에 대해 캠페인 모듈을 활성화하십시오.
  • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
  • 플레이어를 게시 할 때 고급 포함 (페이지 내) 게시 코드를 사용하십시오. 플러그인이 iframe 내에서 실행되므로 표준 삽입 (iframe) 게시 코드가 작동하지 않으므로 상위 페이지 URL 매개 변수에 액세스 할 수없고 올바른 쿠키를 얻지 못할 수 있습니다.

풍모

Video Cloud-HubSpot 통합의 주요 기능 중 일부는 다음과 같습니다.

  • 리드 양식 -시청자가 동영상을 볼 때 자동으로 표시되는 HubSpot 양식을 만듭니다. 자세한 내용은 HubSpot에 대한 사용자 지정 리드 양식생성을 참조하십시오 .
  • 데이터 형식 - 데이터 보기는 사용자 지정타임라인 이벤트 유형으로 제공됩니다 . HubSpot에 복제되는 데이터는 다음과 같습니다.
    • 비디오 이름
    • 비디오 ID
    • 계정 ID (비디오 클라우드)
    • 페이지 URL (참조 페이지의 URL)
    • 플레이어 ID (비디오 클라우드 플레이어)
    • 시청률 (25 %, 50 %, 75 %, 95 %)
  • 데이터 전송 -보기 데이터는 자동화 된 프로세스를 사용하여 HubSpot으로 전송됩니다. 그러나 HubSpot 플랫폼의 일부 보고서는 새로 고치는 데 다소 시간이 걸릴 수 있습니다.
  • 데이터 보존 -수집 된 데이터는 선택적으로 최대 6 개월 동안 Video Cloud에 보관할 수 있습니다.

연결 설정

다음 단계에 따라 Video Cloud와 HubSpot 간의 연결을 설정하십시오.

Video Cloud와 HubSpot 간의 연결 설정

  1. 캠페인 모듈을 열고데이터 연결왼쪽 탐색에서
  2. 새 연결추가를 클릭합니다 .
  3. HubSpot을통합으로 선택합니다.
    참고: 사용 가능한 통합은 Video Cloud 구독을 기반으로 합니다.
  4. (선택 사항) 연결 이름을 입력합니다 .
  5. 연결 을 클릭합니다 .
  6. 표준 (REST API)을 선택한 다음저장을 클릭합니다 .
  7. 허브스팟에연결 을 클릭합니다 .
  8. HubSpot에 로그인하라는 메시지가 표시됩니다. 이메일주소와비밀번호를 입력합니다 . 로그인을 클릭합니다 .
  9. 로그인에 성공하면 계정을 선택하라는 메시지가 표시됩니다. 딸깍 하는 소리계정 선택 .

이제 HubSpot에 대한 Video Cloud 연결이 설정되어 사용할 준비가되었습니다.

통합 구성

Video Cloud와 HubSpot이 연결되면 통합을 구성 할 수 있습니다. HubSpot 계정 정보를클릭하여 연결된 HubSpot 계정 정보를 표시합니다.

  1. 기본적으로연결 상태는활성화됩니다 . 연결 상태는 비디오 클라우드 비디오 참여 데이터가 HubSpot으로 전송되는지여부를 결정합니다.
    • 활성화 됨 -데이터가 수집되어 HubSpot으로 전송됩니다.
    • 장애인 -데이터가 수집되지 않음 (통합이 비활성화 됨)
  2. 기본적으로 알 수 없는 뷰어에 대한 데이터추적은사용 안 함으로 설정됩니다 . 이 옵션을 활성화하면 알 수없는 사용자의 데이터를 추적합니다. 데이터는 선택한 데이터 보존 정책에 따라 보존됩니다. 데이터 보존이 보존없음으로 설정된경우 이 필드는 표시되지 않습니다.
  3. 데이터 보존 정책을 선택하십시오. 데이터 보존 정책은 보기 데이터 및 리드 양식 데이터(캠페인 리드 양식을 사용하는 경우)가 Video Cloud 계정에 보존되는 기간을 결정합니다. Video Cloud에 데이터를 보관하면 중단 된 동기화에서 복구 할 수 있습니다. 또한 Campaign은 이벤트가 데이터 보존 기간을 지날 때까지 동기화에 실패한 이벤트를 계속해서 재시도합니다(일반적으로 시청자가 비디오를 본 시점에 알려진 연락처가 아니기 때문에). 개인 정보 보호 정책에 따라 보존 설정을 조정해야합니다. -Video Cloud에 사용자 데이터를 보관할 필요가 없습니다. 데이터는 데이터 보유 기간이 지나면 삭제됩니다.

HubSpot과의 통합을 제거하려면연결 끊기를 클릭합니다 . 캠페인 모듈은 마케팅 플랫폼에 대한 다중 연결을 지원합니다. 추가 통합을구성하려면 새 연결 추가를 클릭합니다.

통합 확인

통합이 구성되면 통합을 테스트하여 비디오보기 데이터가 HubSpot으로 전송되는지 확인합니다. 통합 확인에 대한 자세한 내용은 HubSpot REST API 통합 테스트 가이드를 참조하십시오 .

추적 ID 전달

캠페인 모듈은 추적 ID 값을 HubSpot에 전달하는 기능을 지원합니다. 예를 들어 추적 ID는 캠페인 ID를 추적하는 데 사용될 수 있습니다. 추적 ID는 캠페인 지원 플레이어를 표시하는 URL에 추가하거나 플레이어 내장 코드에 추가할 수 있습니다.

  • 예: URL에 추가됨
  http://www.myurl.com/video.html?
  • 예: 플레이어 소스 코드에 추가됨
  <video data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js"  data-bc-trackingid="12345"통제 수단>< /동영상>< 스크립트 src="//players.brightcove.net/1486906377/default_default/index.min.js">< /스크립트>

그런 다음 Campaign은 동기화된 다른 필드로 HubSpot에 값을 전달합니다.

시청자 식별

뷰어 추적이 발생하려면 다음 두 가지 방법 중 하나로 뷰어를 추적 할 수 있습니다.

  • URL 매개 변수 - 비디오를 표시하는 페이지의 bcemail URL 매개 변수로 사용자의 이메일 주소를 전달할 수 있습니다. 자세한 내용과 예는 다음을 참조하십시오. 캠페인 모듈이 시청자를 식별하는 방법 .
  • 허브스팟 쿠키 - 플레이어가 로드될 때 Campaign은 사용자가 HubSpot에 의해 이미 쿠키되었는지 확인합니다. 그렇다면 Campaign은 쿠키 값을 통해 사용자가 누구인지 알고 있는지 HubSpot에 묻습니다. HubSpot이 연락처를 반환하면 Campaign은 모든 보기 데이터를 해당 사용자와 연결합니다.

한 가지 가능한 시나리오는 추적 목적으로 고유하게 생성 된 URL로 비디오 랜딩 페이지를 만드는 것입니다. 그런 다음 캠페인 모듈은 URL 매개변수를 사용하여 사용자 서버 측을 조회하고 보기 데이터를 동기화합니다.

플레이어 / 플러그인이 URL 매개 변수를 읽으려면 고급 삽입 (인 페이지) 게시 코드를 사용해야합니다. 플러그인이 iframe 내부에서 실행되므로 표준 삽입 (iframe) 게시 코드가 작동하지 않으므로 매개 변수가있는 상위 페이지 URL에 액세스 할 수 없습니다.
