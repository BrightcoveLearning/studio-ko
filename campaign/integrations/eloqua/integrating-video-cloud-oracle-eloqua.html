캠페인 모듈을 사용하면 Video Cloud가 Brightcove Player에서 Oracle Eloqua로 보기 데이터를 동기화할 수 있습니다. Oracle Eloqua에서이 데이터는 세분화, 캠페인 로직, 보고서, 리드 스코어링 및 커뮤니케이션 개인화에 사용될 수 있습니다. Oracle Eloqua와의 캠페인 모듈 통합은 두 가지 주요 구성 요소로 구성됩니다.

  • 비디오보기 데이터를 추적 할 수있는 Brightcove Player 플러그인
  • Video Cloud에서 Oracle Eloqua로 추적 데이터를 전송하는 동기화 프로세스

요구 사항

Oracle Eloqua와 함께 캠페인 모듈을 사용하려면 다음을 수행해야 합니다.

  • Professional 또는 Enterprise Video Cloud 계정 보유
  • 캠페인 모듈을 열 때 나타나는 초기 시작 화면에서 필수 정보를 제출하여 계정에 대해 캠페인 모듈을 활성화하십시오.
  • 모듈을 처음 사용할 때 서비스 계약을 클릭하십시오.
  • 플레이어를 게시 할 때 고급 포함 (페이지 내) 게시 코드를 사용하십시오. 플러그인이 iframe 내에서 실행되므로 표준 삽입 (iframe) 게시 코드가 작동하지 않으므로 상위 페이지 URL 매개 변수에 액세스 할 수없고 올바른 쿠키를 얻지 못할 수 있습니다.
  • Oracle Eloqua 10 및 자격 증명이 있어야 로그인할 수 있습니다. https://login.eloqua.com
    메모: 이러한 자격 증명은 Oracle Eloqua 로그인 페이지에 입력되며 브라이트코브에 직접 제공되지 않습니다.
  • Oracle Eloqua에 로그인하는 데 사용되는 자격 증명은고급 사용자 - 마케팅그룹에 추가되어야 하며 다음과 같은 권한이 있어야 합니다.
    • API 사용
    • 클라우드 애플리케이션 설치
    • 마케팅 캠페인 설정

풍모

Video Cloud-Oracle Eloqua 통합의 주요 기능은 다음과 같습니다.

  • 리드 양식 -시청자가 동영상을 볼 때 자동으로 표시되는 Eloqua 양식을 만듭니다. 자세한 내용은 Oracle Eloqua용 사용자 정의 가망 고객 양식생성을 참조하십시오 .
  • 데이터 형식 -비디오 클라우드보기 데이터는 Oracle Eloqua의 사용자 정의 데이터 개체 (CDO)로 전달됩니다. Oracle Eloqua에 복제되는 데이터는 다음과 같습니다.
    • 비디오 이름
    • 비디오 ID
    • 계정 ID (비디오 클라우드)
    • 페이지 URL (참조 페이지의 URL)
    • 플레이어 ID (비디오 클라우드 플레이어)
    • % 보았다
  • 데이터 전송 -보기 데이터는 매시간 동기화되며 Oracle Eloqua에 표시되는 데 2-4 시간이 소요될 수 있습니다.
  • 데이터 보존 -수집 된 데이터는 선택적으로 최대 1 개월 동안 Video Cloud에 보관할 수 있습니다.

연결 설정

Eloqua와의 통합을 구성하기 전에 새 브라우저 창을 열고 Eloqua 계정에서 수동으로 로그 아웃하는 것이 좋습니다. 이렇게하면 통합을 구성 할 때 Eloqua 계정에 로그인하라는 메시지가 표시되고 올바른 계정을 사용하고 있는지 확인할 수 있습니다.

Video Cloud와 Oracle Eloqua 간의 연결을 설정하려면 다음 단계를 따르십시오.

  1. 캠페인 모듈을 열고데이터 연결왼쪽 탐색에서
  2. 새 연결추가를 클릭합니다 .
  3. 통합으로 Eloqua를 선택합니다.
    참고: 사용 가능한 데이터 연결은 Video Cloud 구독을 기반으로 합니다.
  4. (선택 사항) 연결 이름을 입력합니다 .
  5. 연결 을 클릭합니다 .
  6. Eloqua로연결을 클릭합니다 .
  7. Eloqua에 로그인하라는 메시지가 표시됩니다. Eloqua 회사 이름 , 사용자 이름및암호를 입력하십시오 . 로그인을 클릭합니다 .
  8. 로그인이 성공하면 Eloqua용브라이트코브 비디오 연결앱을 설치하라는 메시지가 표시됩니다. 수락 및 설치를 클릭합니다 .
  9. 계정에 대해 Eloqua를 활성화하라는 메시지가 표시됩니다. 활성화를 클릭합니다 .
  10. Eloqua용브라이트코브 비디오 커넥트가 Eloqua계정에 액세스할 수 있도록 허용하라는 메시지가 표시됩니다. 로그인을 클릭합니다 .
  11. 회사 이름과 사용자 이름을 확인하고수락을 클릭합니다 .
  12. Eloqua를 설정하라는 메시지가 표시됩니다. 설정 Eloqua 를 클릭합니다 .

통합 구성

Video Cloud와 Oracle Eloqua 연결이 설정되면 통합을 구성 할 수 있습니다.

  1. 뷰를 외부 활동으로동기화하려면 Eloqua와 외부활동으로 비디오 보기 동기화를 선택합니다. 외부 활동은 Eloqua에 정보를 넣는 또 다른 장소입니다. 게시자는 외부 활동을 사용하여 Eloqua에서보고를 수행하거나 작업을 트리거하도록 선택할 수 있습니다. 자세한 내용은 Eloqua 문서를 참조하십시오 .
  2. 기본적으로연결 상태는활성화됩니다 . 연결 상태에따라 비디오 클라우드 비디오 참여 데이터가 Oracle Eloqua로 전송되는지 여부가 결정됩니다.
    • 활성화 됨 -데이터가 수집되어 Eloqua로 전송됩니다.
    • 장애인 -데이터가 수집되지 않음 (통합이 비활성화 됨)
  3. 기본적으로 알 수 없는 뷰어에 대한 데이터추적은비활성화됩니다 . 이 설정은 뷰어를 알 수없는 경우 비디오 참여 데이터가 Video Cloud에 저장되는지 여부를 결정합니다. 알려진 뷰어에 대한 데이터 만 Eloqua로 전송됩니다.
    • 활성화 됨 -모든 시청자의 데이터를 수집하여 Video Cloud에 저장
    • 장애인 -알 수없는 시청자의 데이터는 Video Cloud에 저장되지 않습니다.
  4. 데이터 보존 정책을 선택하십시오. 데이터 보존 정책은 보기 데이터 및 리드 양식 데이터(캠페인 리드 양식을 사용하는 경우)가 Video Cloud 계정에 보존되는 기간을 결정합니다. Video Cloud에 데이터를 보관하면 중단 된 동기화에서 복구 할 수 있습니다. 또한 Campaign은 이벤트가 데이터 보존 기간을 지날 때까지 동기화에 실패한 이벤트를 계속해서 재시도합니다(일반적으로 시청자가 비디오를 본 시점에 알려진 연락처가 아니기 때문에). 개인 정보 보호 정책에 따라 보존 설정을 조정해야합니다. -Video Cloud에 사용자 데이터를 보관할 필요가 없습니다. 데이터는 데이터 보유 기간이 지나면 삭제됩니다.

Oracle Eloqua와의 통합을 제거하려면연결 끊기를 누릅니다 . 페이지 하단에는 모든 캠페인 지원 플레이어 및 관련 리드 양식이 나열됩니다.

캠페인 모듈은 마케팅 플랫폼에 대한 다중 연결을 지원합니다. 추가 통합을구성하려면 새 연결 추가를 클릭합니다.

통합 확인

통합이 구성되면 통합을 테스트하여 비디오보기 데이터가 Eloqua로 전송되는지 확인합니다. 통합 확인에 대한 자세한 내용은 Eloqua 통합 테스트 가이드를 참조하십시오 .

캠페인 지원 플레이어 만들기

동영상 참여 데이터는 캠페인 지원 동영상 플레이어를 사용할 때만 Eloqua로 전송됩니다. 캠페인 지원 Brightcove 플레이어 생성에 대한 정보는 다음을 참조하십시오. 캠페인 지원 플레이어 만들기 .

추적 ID 전달

캠페인 모듈은 추적 ID 값을 Eloqua에 전달하는 기능을 지원합니다. 예를 들어 추적 ID는 캠페인 ID를 추적하는 데 사용될 수 있습니다. 추적 ID는 캠페인 지원 플레이어를 표시하는 URL에 추가하거나 플레이어 내장 코드에 추가할 수 있습니다.

  • 예: URL에 추가됨
  http://www.myurl.com/video.html?
  • 예: 플레이어 소스 코드에 추가됨
  <video data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js"  data-bc-trackingid="12345"통제 수단>< /동영상>< 스크립트 src="//players.brightcove.net/1486906377/default_default/index.min.js">< /스크립트>

그런 다음 Campaign은 동기화되는 다른 필드로 값을 Eloqua에 전달합니다.

시청자 식별

Eloqua 통합이 있는 캠페인 지원 플레이어는 Eloqua의 API를 호출하여 Eloqua GUID라고 하는 방문자 ID를 얻습니다. 이 ID는 캠페인 모듈의 보기 데이터에서 캡처되며 보기 데이터를 Eloqua와 동기화하는 데 사용됩니다.

Eloqua는 자체 쿠키 세트를 사용하며 Brightcove Player가 Eloqua 통합으로 구성된 경우 Campaign은 Eloqua 쿠키의 GUID를 캡처하고 Eloqua 서버 측과 통신하여 사용자가 누구인지 알아내려고 시도합니다.

사용자의 이메일 주소를 전달하여 뷰어를 식별하는 데 사용할 수도 있습니다. 자세한 내용과 예는 다음을 참조하십시오. 캠페인 모듈이 시청자를 식별하는 방법 .
