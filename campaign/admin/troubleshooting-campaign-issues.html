리드 양식 문제

다음은 리드 양식에서 본 몇 가지 일반적인 문제와 질문입니다.

양식을 제출 한 후에도 Marketo Munchkin 통합에 대해 리드 양식이 계속 표시되는 이유는 무엇입니까?

Munchkin 통합에는 사용자가 알려 졌는지 여부를 묻는 데 사용할 수있는 API가 없습니다. 알려진 사용자인지 알 수 없기 때문에 항상 리드 양식을 표시합니다. Marketo REST 통합은 API를 제공하므로 알려진 사용자와 알려지지 않은 사용자를 확인할 수 있습니다. Marketo REST 통합으로 전환하는 것이 좋으며이 문제를 피할 수 있습니다.

내 맞춤 리드 양식이 제출시 리디렉션되도록 설정되어 있습니다. 양식을 제출한 후 1~2초 동안 리디렉션된 페이지가 표시되는 이유는 무엇입니까?

리드 양식을 닫기 전에 리디렉션 된 페이지가 완전히로드 될 때까지 기다려야합니다. 한 가지 해결책은 가벼운 페이지 (이미지 또는 JavaScript가 많지 않음) 또는 이 문제를 해결하기 위해 특별히 설정 한 빈 페이지로 리디렉션하는 것입니다. https://map.brightcove.com/lead/complete

확인 HTML로 리드 양식을 구성했습니다. 제출 후 몇 초 동안 검은 색 화면이 표시되는 이유는 무엇입니까?

HTML의 텍스트 색상을 변경해야합니다. 확인 HTML이 표시 될 수 있지만 기본 텍스트 색상이 검은 색이기 때문에 볼 수 없습니다.

제출 후 HubSpot 리드 양식이 닫히지 않는 이유는 무엇입니까?

HubSpot의 리드 양식이 로 리디렉션되도록 구성되어 있는지 확인합니다http://map.brightcove.com/lead/complete . 확인 메시지를 표시하려면 HubSpot이 아닌 Campaign 내에서 구성하십시오.

내 리드 양식이 표시되지 않는 이유는 무엇입니까?

리드 양식이 표시되지 않는 경우 먼저 리드 양식을 표시하기위한 모든 기준이 충족되었는지 확인하십시오. 시청중인 여러 동영상에 따라 달라지는 기준은 페이지 새로 고침시 작동하지 않습니다. 다음 동영상이 동일한 플레이어에 자동으로 대기하는 사용 사례를위한 것입니다.

  1. 캠페인 플러그인이 플레이어에 추가되었는지 확인합니다. 자세한 내용은 다음을 참조하십시오. 캠페인 지원 플레이어 만들기 .
  2. 통합이 올바르게 설정되었고 플러그인으로 전송되었는지 확인하십시오.
  3. 리드 양식이 해당 플레이어에 추가되었는지 확인합니다. 플레이어에 리드 양식이 추가 된 경우 플러그인이 구성에 대해 만드는 호출에 대한 응답에는 leadForm 속성 아래에 리드 양식 표시 정보가 포함되어야합니다.
  4. 시청자가 이미 리드가 아닌지 확인합니다. 리드 양식은 시청자가 마케팅 자동화 플랫폼(MAP)의 리드가 아니거나 리드 양식을 아직 작성하지 않은 것으로 캠페인에서 확인된 경우에만 표시됩니다. 플러그인은 뷰어가 리드인지 확인하기 위해 캠페인 서버에 별도의 JSONP 호출을 수행합니다. 뷰어가 리드 인 경우 서버는 상태 1 및보기 데이터를 MAP에 동기화하는 데 유용 할 수있는 기타 식별자로 응답합니다. 뷰어가 리드가 아닌 경우 서버는 상태 0으로 응답합니다.
    • 리드인이 아닌 시청자에 대한 응답 예: 
      /**/ JSON 콜백_2935650618 === '함수' 및 JSON 콜백_2935650618 ( {"status":0} );
    • 리드인 시청자에 대한 응답 예: 
      /**/ JSON 콜백_45936049288 === '함수' 및 JSON 콜백_45936049288 ( {"status":1,"eloqua":{"contactId":"5","internalId":"CTCPB000000000005"} }) 의 유형입니다.

동기화 문제

다음은 몇 가지 일반적인 동기화 문제입니다.

Marketo REST보기 이벤트가 활동 유형 ID '000000'을 찾을 수 없음 오류와 동기화하지 못함

이는 통합과 관련된 맞춤 Marketo 활동이 삭제되었을 때 발생합니다. Marketo REST 통합이 생성되면 활동을 가져 와서 해당 ID를 저장하고 해당 ID를 사용하여 이벤트를 동기화합니다. 솔루션 (새 Brightcove 비디오보기 활동이 생성되었다고 가정)은 통합을 연결 해제하고 다시 생성하여 새 활동의 ID를 선택하는 것입니다.

동기화되지 않은 이벤트보기

통합이 MAP으로 올바르게 설정되면 캠페인 지원 플레이어에서 데이터를 볼 때 데이터가 채워져야 합니다. 모든 최근 활동각 MAP의 알려진 리드에 해당하는 페이지 및 보기 이벤트는 결국 도구와 동기화되어야 합니다.

먼저 보기 데이터가모든 최근 활동페이지에 표시되는지 확인합니다.

보기 이벤트가 나열되지만 MAP에 없는 경우, 사용자는 API를 통해 보기 데이터를 MAP으로 가져올 수 있는 충분한 권한이 없거나 통합이 올바르게 구성되지 않은 사용자로서 MAP을 사용하여 Campaign을 승인했을 수 있습니다. 캠페인 홈페이지에는 최근 보기 이벤트 동기화 작업의 상태가 나열됩니다. 보기 이벤트가 MAP에 동기화되지 않는 이유를 나타내는 피드백을 사용할 수 있어야합니다.

캠페인에 보기 이벤트가 표시되는 경우상태열은동기화 . 이는 Campaign이 데이터를 MAP에 성공적으로 전달할 수 있음을 나타냅니다. unknown_contact상태가 표시되면 사용자를 알 수 없으며 MAP에서 수신할 수 없음을 의미합니다. 대부분의 MAP는 알려진 사용자에 대한 타사 데이터 만 허용합니다. 해결책은 MAP에서 생성 된 사용자 지정 리드 양식을 사용하여 실제로 알려진 사용자인지 확인하는 것입니다.

보기 이벤트가 나열되지 않으면 캠페인이 캠페인 활성화 플레이어로 설정되지 않았을 수 있습니다. 개발자 도구가 활성화된 브라우저를 사용하여 문제가 있는 비디오가 포함된 페이지를 방문하고 캠페인 플러그인에서 특정 HTTP 호출이 수행되는지 확인합니다.

플레이어가 로드되면 플러그인은 캠페인 구성 엔드포인트를 호출하여 통합을 기반으로 플러그인의 동작을 결정하는 매개변수를 가져와야 합니다.

이 호출이 네트워크 트래픽에서 누락 된 경우 플레이어에 통합이 할당되지 않았 음을 의미합니다. 이 작업을 수행하는 방법에 대한 정보는 다음을 참조하십시오. 캠페인 지원 플레이어 만들기 .

이 호출은 JSONP를 사용하여 이루어집니다. JSONP 응답이 빈 개체 인 경우 소스 코드가 올바르지 않고 잘못된 계정 ID를 지정했을 수 있습니다. 구성 요청에 계정 및 플레이어 URL 매개변수가 모두 포함되지 않거나 지정된 계정을 캠페인별 데이터베이스에서 검색할 수 없는 경우(대부분 URL/embed에 잘못된 계정이 지정되었을 가능성이 높은 경우) 이 끝점에서 빈 개체가 반환됩니다. 코드이며 데이터베이스에서 찾을 수 없음).

/**/ JSON 콜백_78754666587 === '함수' 및 JSON 콜백_78754666587 ( {} );

오브젝트에 ID 매개 변수가 있지만 통합 매개 변수가 누락 된 경우 사용자는 마케팅 자동화 도구와의 통합을 설정하지 않았으며보기 이벤트가 기록되지 않습니다.

캠페인 지원 플레이어를 게시할 때 표준 삽입(iframe) 코드를 사용한 경우에도 데이터 동기화 문제가 발생합니다. 플레이어를 게시 할 때 고급 삽입 (페이지 내) 코드를 사용하십시오. 캠페인 플러그인이 iframe 내부에서 실행되므로 표준 Embed(iframe) 게시 코드가 작동하지 않으므로 상위 페이지 쿠키 또는  URL 매개변수.

연결 문제

캠페인 모듈에 Eloqua 통합을 설정할 수 없다고 표시됨

Eloqua 요구 사항을 확인하십시오 . IP 화이트리스트 보안 기능을 사용하는 경우, Campaign - Eloqua 트래픽을 허용하기 위해 지정된 IP 주소를 화이트리스트에 등록하지 않는 한 Campaign은 Eloqua와 통신할 수 없습니다. 연락하다브라이트코브 지원자세한 내용은.

Campaign에서 내 Marketo REST 사용자 정의 활동에 필드가 누락되었다고 표시되지만 Marketo를 보면 모든 필드가 존재함을 알 수 있습니다.

확인해야 할 몇 가지 사항이 있습니다.

  1. 필드 이름은 정확히 일치해야합니다. "이벤트 1"은 "이벤트 1"과 동일하지 않습니다.
  2. 활동은 초안 상태가 아니라 승인되어야합니다.
  3. 사용자 지정 활동을 삭제하고 새 Marketo REST 통합을 설정하지 않고 다시 만든 경우 이전 통합을 연결 해제하고 새 통합을 만들어야합니다. 사용자 지정 활동을 조회하고 각 통합에 대한 ID 값을 저장합니다. 삭제하여 ID가 변경되는 경우 자동으로 새 ID를 선택할 수 없습니다.
  4. 최근에 새 필드가 활동에 추가 된 경우 변경 사항이 Marketo의 시스템에 적용되는 데 최대 24 시간이 걸릴 수 있습니다.
