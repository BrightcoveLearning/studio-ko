Cloud Playout 모듈을 사용하면 방송사와 미디어 게시자가 VOD 비디오 및 라이브 스트림 모음을 사용한 다음 선형 채널로 스트리밍 할 수 있습니다. 채널은 Brightcove Live, Amazon S3 또는 지정된 RTMP URL로 스트리밍 될 수 있습니다.

Cloud Playout 모듈에 액세스하려면 탐색 메뉴에서클라우드플레이아웃을 클릭합니다. 생성 된 모든 Cloud Playout 채널 목록이 표시됩니다 (있는 경우).

페이지에는 다음 정보가 표시됩니다.

  • 채널 이름 -채널 이름
  • 시작 시간 -채널 시작 시간
  • 종료 시간 -채널 종료 시간
  • 루핑 -프로그램이 반복되는지 여부를 나타냅니다.
  • 목적지 -채널의 대상 목적지
  • 생성 일자 -채널이 생성 된 날짜
  • 마지막 업데이트 -채널이 마지막으로 업데이트 된 날짜
  • 상태 -채널 상태
    • 초안 -채널이 생성되었지만 활성화되지 않았습니다.
    • 창조 -채널이 활성화되는 중입니다.
    • 예정 -채널이 활성화되어 스트리밍 할 준비가되었습니다.
    • 시작 -스트리밍 시작을위한 채널 시작 프로세스
    • 달리는 -채널이 활발히 스트리밍 중입니다.
    • 멎는 -채널 스트리밍이 중지됩니다.
    • 게으른 -채널이 성공적으로 실행되었고 중지되었습니다.
    • 삭제 -채널 삭제 중
    • 오류 -채널 생성 또는 스트리밍 중 오류가 발생했습니다.

채널 만들기

새 클라우드 플레이 아웃 채널을 만들려면 다음 단계를 따르세요.

  1. 딸깍 하는 소리클라우드 플레이아웃탐색 헤더에서.
  2. 딸깍 하는 소리 새로 추가 :
  3. 채널식별세부 정보를 입력합니다.
    • 이름 -채널 이름. 이름에는 영문자가 포함되어야 하며 다음과 같은 특수 문자가 허용됩니다. _ . : / = + - 
    • 기술 -채널 설명
    • 이미지 URL -채널 이미지 파일에 대한 URL 채널 메타 데이터의 일부로 검색 할 수있는 채널 이미지를 의미합니다.
  4. 채널일정세부 정보를 입력합니다.
    • 루프 재생 콘텐츠 - 켜져있으면 프로그램이 루프에서 재생되고종료 날짜/시간이활성화됩니다. 프로그램은 언제든지 수정할 수 있으며 변경 사항은 현재 프로그램과 후속 루프에 반영됩니다. 아래의 루핑 프로그램만들기섹션을 참조하십시오.
    • 광고 삽입 -켜져 있으면 채널 재생 중에 광고를 표시 할 수 있습니다. 이 옵션을 선택하면 라이브 입력이 허용되지 않으며브라이트코브 라이브 이벤트또는 Amazon S3 버킷으로만 출력할 수있습니다. 콘텐츠수익 창출을 참조하십시오 .
    • 시작 날짜 / 시간 -채널 재생이 시작되는 시간입니다. 모든 자산의 시작 시간은이 시작 시간에 기간을 더하여 계산됩니다. 이 시간은 채널이 실행 중 상태가 아닌 경우에만 업데이트 할 수 있습니다.
    • 종료 날짜/시간 - 값을 입력하지 않고루프 재생 콘텐츠를선택하지 않으면 재생이 완료되면 스트림이 종료됩니다. 루프 재생콘텐츠가 선택되어 있고종료 날짜/시간이없으면 채널이 중지될 때까지 재생이 스트리밍됩니다. 이 시간은 언제든지 업데이트 할 수 있습니다.
  5. 라이브 입력을 사용할 경우 다음을 설정하십시오. 라이브 입력기본 Brightcove MP4를 슬레이트에 사용하지 않는 한 Slate URL을 입력합니다. 인코더가 중지되거나 일부 기술적 인 문제로 인해 라이브 스트림이 차단되어 라이브 입력이 중지 될 때마다 슬레이트 MP4가 재생됩니다. 참고 Slate URL은 항상 사용할 수 있는 정적 URL이어야 합니다. (단기 CDN URL은 허용되지 않습니다.)
  6. 채널대상세부 정보를 입력합니다. Cloud Playout은 동시에 2 개의 출력을 지원합니다. 예를 들어 HLS를 S3 버킷으로 출력하고 RTMP 피드를 브라이트코브 라이브 채널 또는 타사 수집 지점에 출력할 수 있습니다. 브라이트코브 라이브이벤트와 RTMP를 동시대상으로 결합하는 것은 지원되지 않습니다.
    • 디스플레이 최적화 -출력에 대한 비트 전송률 및 크기를 제어합니다.
      • 모바일 및 웹 -체크시 HD 출력 (RTMP 또는 HLS 단일 비트 레이트)
      • 스마트 티비 -체크하면 풀 HD 출력 (RTMP 또는 HLS 단일 비트 레이트)
    • 출력 영역 -채널이 생성 될 지역. Brightcove는 대다수의 청중이 예상되는 지역과 지리적으로 가장 가까운 지역을 선택할 것을 권장합니다.
    • 출력 -채널 출력 방법 선택
      • Brightcove 라이브 이벤트 -이전에 만든 Brightcove 라이브 채널을 만들거나 사용합니다. 새 채널을 만들 때 선택한출력 영역에 따라 채널의영역이 최적화됩니다.
      • RTMP -RTMP 또는 RTP 수집 지점으로 스트림을 출력하려면 선택합니다. RTP는 타사 수집 지점에 사용할 수 없습니다. 광고 삽입및브라이트코브 라이브 이벤트가 활성화된경우에만 사용할 수 있습니다. 이 옵션을 선택하면 RTMP URL필드가 나타납니다. 라이브 스트리밍 서비스의 RTMP 입력 URL을 입력합니다.
      • Amazon S3 버킷 - HLS 출력을 Amazon S3 버킷으로 푸시하려면 선택합니다. 버킷 이름, 리전, 액세스 키 및 보안 키를 입력해야합니다. 버킷은 공개되고 쓰기 가능해야합니다.
      로 출력한다는 점에 유의하십시오. 브라이트코브 라이브 이벤트청구 가능한 계정에 대한 실시간 시간을 자동으로 생성합니다. Cloud Playout 사용에 대한 추가 요금은 없지만 라이브 스트리밍 시간에 대한 일반 요금이 적용됩니다. RTMP 또는 Amazon S3로 출력하면 청구 가능한 라이브 시간이 생성되지 않습니다.
  7. 딸깍 하는 소리 . DRAFT 상태로 새 채널이 생성됩니다 .

채널에 비디오 및 라이브 이벤트 추가

보다 Cloud Playout 채널에 콘텐츠 추가채널에 VOD 콘텐츠, 재생 목록 및 라이브 이벤트를 추가하는 방법에 대한 정보를 확인하세요.

또한 참조콘텐츠에 범퍼 추가 VOD 또는 라이브 이벤트 항목에 범퍼를 추가합니다.

동영상 및 라이브 이벤트가 채널에 추가된 후활성화를클릭하여 채널을 활성화합니다. 스트리밍을 시작하려면 먼저 채널을 활성화해야합니다. 자세한 내용은 채널 액션사용을 참조하십시오.

참고: 프로그램에서 비디오 순서를 변경하려면 비디오를 클릭하고 새 위치로 드래그하십시오.

비디오가 채널에 추가되면 시스템 생성 재생 목록이 생성되고 미디어 모듈에 표시됩니다.

Cloud Playout 모듈에서 채널을 변경하고 생성된 재생 목록을편집하지 않는 것이 좋습니다 .

RTMP 수집 지점 가져 오기

라이브 인코더의 경우 채널에 대한 RTMP 수집 지점이 필요합니다. 다음과 같이 채널설정에서 찾을 수 있습니다.

사용중인 인코더에 따라 RTMP 수집 지점과 스트림 이름을 모두 입력해야 할 수 있습니다. 채널의 스트림 이름은 인제스트 포인트 주소의 마지막 부분이기도 한채널 ID입니다.

OBS 소프트웨어 인코더를 사용하는 경우 다음을 지정해야 합니다. 관습...서비스를 입력해야 합니다. 섬기는 사람그리고스트림 키 . 서버는인제스트 포인트의 첫 번째 부분이 되고 , 아래 예와 같이 Steam Key는 나머지 부분이됩니다.

루핑 프로그램 만들기

반복되는 프로그램을 만들려면 채널을 만들 때루프 재생 콘텐츠설정을켜기로 설정합니다 . 프로그램에 동영상이 추가되면 프로그램 목록이 업데이트되어 전체 채널 예약 시간을 채우기 위해 연결된 시작 / 종료 시간과 함께 일련의 동영상이 표시됩니다. 라이브 이벤트의 기간을 변경하면 이미 지난 이벤트의 기간은 변경되지 않지만 향후 이벤트 재생에는 새로운 기간이 사용됩니다.
