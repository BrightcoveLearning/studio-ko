소개

Brightcove Cloud Playout은 라이브 피드와 결합 된 VOD 카탈로그의 항목 재생 예약을 프로그래밍하고 Brightcove Live의 안정성과 확장 성을 모두 갖춘 선형 채널로 스트리밍 할 수있는 Studio 모듈입니다.

Cloud Playout은 다음을 포함하여 예정된 프로그래밍 채널을위한 강력한 엔드 투 엔드 솔루션을 제공합니다.

  • 24x7 선형 채널
  • 팝업 / 가상 채널
  • 시뮬레이션 된 라이브 이벤트

Cloud Playout은 사전 녹화 된 콘텐츠를 활용하는 방송과 같은 라이브 TV 경험을 통해 청중 참여를 높이고 새로운 수익 기회를 창출하는 데 도움이됩니다. 기존 VOD 자산을 사용하여 예약 된 라이브 채널을 만듭니다.

풍모

Cloud Playout의 기능은 다음과 같습니다.

  • Video Cloud Media 라이브러리에서 VOD 자산을 가져옵니다.
  • VOD 자산과 예약 된 라이브 피드간에 전환합니다.
  • 쉽고 빠르고 안정적인 방식으로 마지막 순간 일정 변경에 실시간으로 대응합니다.
  • 서버 측 광고 삽입 (SSAI)을 통해 채널 콘텐츠로 수익을 창출하세요.
  • 프로그램을 반복하도록 채널을 구성하십시오.
  • 장치 별 플레이 아웃을 최적화합니다.
  • 드래그 앤 드롭하여 실행 중 상태에서도 언제든지 프로그램 일정을 다시 정렬 할 수 있습니다.
  • RTMP 출력을 통해 브라이트코브 라이브, Amazon S3 버킷 또는 타사 대상을 통해 소유 및 운영 속성으로 채널을 푸시합니다.
  • 채널에 대한 EPG (전자 프로그램 가이드)를 검색하는 API

한계

다음 가정 및 제한 사항이 적용됩니다.

  • 동영상은 Dynamic Delivery 프로필을 사용하여 수집하거나 다시 트랜스 코딩해야하며 MP4 변환이 있어야합니다.
  • 클라우드 재생에 지원되는 최대 입력 비트 전송률 및 해상도는 20MBPS / 1080p입니다. 이는 VOD 콘텐츠와 라이브 입력 모두에 적용됩니다.
  • Brightcove는 비디오와 전자 프로그램 가이드 정확도 간의 올바른 전환을 보장하기 위해 최소 2 분 길이의 비디오 사용을 권장합니다.
  • 현재 동영상에 추가 된 캡션 (텍스트 트랙)은 클라우드 플레이 아웃 채널에 표시되지 않습니다. 캡션 지원은 향후 릴리스에서 추가 될 예정입니다.
  • 라이브 이벤트의 최소 길이는 5 분입니다.
  • 클라우드 플레이아웃의 모든 라이브 이벤트는 비디오 클라우드 원격 자산과 1:1 관계를 공유합니다. 이 원격 자산은 다른 채널에서 재사용 할 수 없습니다. 라이브 이벤트가 채널에서 제거되면 연결된 원격 자산이 자동으로 삭제됩니다. 채널이 삭제되면 관련 원격 자산도 자동으로 삭제됩니다.
  • 비디오공유다른 계정에서 계정으로 현재 Cloud Playout 채널에 추가할 수 없습니다.
  • 라이브 이벤트가 현재 재생 중이지만 사용 가능한 활성 피드가 없으면 라이브 피드가 다시 시작되거나 이벤트가 끝날 때까지 슬레이트 콘텐츠가 반복 재생됩니다. 슬레이트 MP4 재생이 시작되기까지 약 30 초의 지연이 있습니다.
  • Video Cloud 계정이 지리적 제한을 위해 구성된 경우 채널출력 지역은 Video Cloud 지리적 제한 정책과 호환되는 리전에서생성되어야 합니다.
  • 최상의 경험을 위해 Brightcove는 Google Chrome 브라우저 사용을 권장합니다.
  • SSAI 지원 채널의 경우 대상을 광고 지원 RTP 라이브 이벤트로 설정해야 합니다.
