채널 액션 사용

채널 상태에 따라 채널 설정 페이지에서 다양한 채널 동작 버튼을 사용할 수 있습니다. 채널 세부 정보를 보려면 채널 이름을 클릭하여 채널 작업 버튼에 액세스하십시오. 다음 채널 작업 버튼을 사용할 수 있습니다.

  • - 채널을 활성화하면 스트리밍이 가능합니다. 채널에 비디오 및/또는 라이브 이벤트를 추가하고 클릭해야 합니다. 활성화 버튼이 나타나기 전에
  • - 클릭지금 플레이채널 스트리밍을 시작합니다. 이 옵션은 예정된 시작 시간이있는 채널에서만 사용할 수 있습니다. 예약 된 시작 시간에 채널이 자동으로 시작됩니다.
  • - 클릭지금 멈춰실행 중인 채널을 중지합니다. 채널이 중지되고 실시간 스트림이 종료됩니다. 채널이 루핑되고 종료 날짜 / 시간이있는 경우 해당 시간에 채널이 자동으로 중지됩니다. 이 옵션을 사용하면 조기에 중지 할 수 있습니다.

채널 중지시 리소스 할당 해제

채널을 중지하면 다음이 발생합니다.

  1. 클라우드 플레이아웃 채널이 중지되면 사용자가 채널 설정을 다시 수정할 수 있는 DRAFT상태로 들어갑니다.
  2. 또한 연결된 BC Live 이벤트가있는 경우이를 정리합니다.
  3. 채널은 다음과 같아야 합니다. 활성화됨다시 리소스가 할당되고 채널이예정상태.
  4. 라이브 스트림에 대해 Cloud Playout 채널이 활성화된 경우 다음이 표시됩니다. RTMP 수집을 위한 정적 진입점채널이 중지되고 다시 시작되는 횟수와 상관없이 채널의 수명 내내 정적 상태를 유지합니다.

    이 진입 점은 채널이 시작되고 실행되면 입력 수락을 시작할 수 있습니다.

  5. Cloud Playout이 내부 문제로 인해 정적 RTMP 수집 지점을 제공 할 수없는 경우 IP 주소 기반 수집 지점을 표시하려고합니다. 이 경우 채널의 RTMP 수집 지점이 변경되었는지 정기적으로 확인하는 것이 좋습니다.
  6. 참고: 채널을 활성화 한 후 Cloud Playout이 정적 수집 지점에 대한 새 IP 주소를 게시하는 데 평균 1 ~ 2 분이 걸립니다. 따라서 활성화 직후 1 ~ 2 분 이내에 채널이 시작되고 라이브 이벤트가 채널 재생 목록의 첫 번째 자산이되면 수집 지점을 확인할 수 없을 가능성이 있습니다. 정적 수집 지점에 대한 IP 주소가 게시되는 즉시 확인할 수 있습니다.
  7. 참고: 채널이 첫 번째 실행을 위해 라이브 입력으로 실행되고 중지 된 다음 채널에 대한 라이브 입력을 끄는 경우 기존 프로그램에 여전히 실패 할 라이브 이벤트가있을 수 있습니다. 채널을 중지한 후 라이브 스트림을 비활성화하기로 결정한 경우 채널을 다시 시작하기 전에 채널 콘텐츠에서 라이브 이벤트를 제거해야 합니다.
