업로드 모듈은 동적 수집 프로세스를 사용하여 비디오 콘텐츠를 검색하고 트랜스 코딩합니다. 업로드 할 파일을 선택하면 Video Cloud는 풀 기반 프로세스를 사용하여 저장 위치에서 콘텐츠를 가져온 다음 Zencoder 트랜스 코딩 서비스를 사용하여 콘텐츠를 트랜스 코딩합니다.

업로드 모듈에 액세스하려면업로드탐색 헤더에서.

지원되는 비디오 형식

비디오 형식 및 코덱 및 Video Cloud에서 지원하는 소스 비디오 형식에 대한 자세한 내용은지원되는 비디오 코덱 및 컨테이너를 참조하십시오 .

수집 프로필 선택

비디오를 업로드하기 전에 수집 프로파일을 선택하십시오. 수집 프로필에는 동영상이 트랜스 코딩 될 때 사용될 트랜스 코딩 설정이 포함되어 있습니다. 사용 가능한 수집 프로필 목록은 Video Cloud 계정이 구성된 방식에 따라 다릅니다. 다음 스크린샷에서는 둘 다컨텍스트 인식 인코딩그리고다이내믹 딜리버리프로필을 사용할 수 있습니다.

참고: 동적 전달에 대한 자세한 내용은 동적 전달개요를 참조하십시오 .
참고: HEVC (고효율 비디오 코딩) 프로필은 유료 추가 기능으로도 제공됩니다. HEVC에 대한 자세한 내용은 HEVC 비디오를 참조하십시오 .

생성되는 비디오 변환은 선택한 수집 프로파일을 기반으로합니다. Brightcove에서 제공하는 수집 프로필에 대한 자세한 내용은 다음을 참조하십시오. 동적 전달 및 컨텍스트 인식 인코딩을위한 표준 수집 프로필

포스터 및 축소판 이미지의 이미지 크기는 수집 프로필에 지정된 이미지 크기를 기반으로합니다.

참고: 추가 수집 프로필을 만들거나 계정의 기본 프로필을 설정하는 방법에 대한 자세한 내용은 수집 프로필관리를 참조하십시오 .

동영상 업로드

참고: 동영상을 업로드하기 전에 적절한 수집 프로필이 선택되었는지 확인합니다.

다음 방법 중 하나를 사용하여 비디오를 업로드 할 수 있습니다.

  • 끌어서 놓기 -파일 시스템에서 모듈 업로드 페이지로 파일을 드래그합니다. 탐색 헤더 아래의 전체 페이지가 드롭 영역이됩니다. 파일이 삭제되면 업로드 프로세스가 시작됩니다.
  • 파일 선택 대화 상자 - 비디오찾아보기를 클릭하여 파일 선택 대화 상자를 연 다음 업로드할 파일을 찾아서 선택합니다.
  • 원격 URL - URL 입력을 클릭하여 소스 파일의 위치를 지정합니다. 소스 파일은 공용 Dropbox 폴더 또는 Amazon S3 저장 위치에있을 수 있습니다. 딸깍 하는 소리프로세스 비디오완료되면.
메모:
  • 파일 크기 제한이 없습니다. Brightcove는 문제없이 최대 100GB의 업로드로 테스트했습니다.
  • 대용량 파일을 업로드할 때 Brightcove는업로드 대기열설정한 번에 하나의 파일 업로드 .
  • 대용량 파일을 업로드 할 때 Studio 시간 초과 오류가 발생하면 파일을 S3 또는 다른 원격 스토리지 위치에 업로드하고 Studio의 URL 업로드 옵션을 사용하십시오.
  • YouTube URL은 비디오 파일에 대한 직접 링크를 생성하지 않으므로 사용할 수 없습니다.
  • Google 드라이브에서 파일을 업로드하는 방법에 대한 자세한 내용은 원격 URL을 사용하여 동영상업로드를 참조하십시오 .
  • Dropbox에서 업로드 할 때 URL이 "? dl = 0"으로 끝나는 경우 파일을 업로드하기 전에 "? dl = 1"로 변경해야 할 수 있습니다.
  • 비디오 파일 이름 (확장명 포함) 은 단일 바이트 (60자 더블바이트) 120자를 초과해서는 안 됩니다. 이 경우 동영상은 성공적으로 처리되지만 나중에 다시 트랜스 코딩 할 수 없습니다.

파일을 선택하면 파일 업로드가 시작됩니다. 업로드를 취소하려면 업로드취소를 클릭합니다 . [ 닫기]를 클릭하여 페이지에서 파일 항목을 제거합니다. 업로드가 처리 상태에있는 동안에는 브라우저 창을 닫거나 멀리 이동하지 마십시오. 그렇게하면 업로드가 완료되지 않을 수 있습니다.

업로드가 완료되면처리 완료상태가 표시됩니다. 이는 처리가 완료되었으며 업로드 모듈에서 탐색하는 것이 안전함을 나타냅니다.

편집 (Edit) 을 클릭하여 인제스트 프로필과 비디오 속성의 하위 집합을 봅니다. 비디오 트랜스 코딩이 시작되지 않은 경우 수집 프로파일을 변경할 수 있습니다. 다음 비디오 속성을 편집 할 수 있습니다.

  • 상태 -비디오 상태
    • 유효한 -동영상 재생 가능
    • 비활성 -비디오를 재생할 수 없습니다

    기본적으로 동영상을 업로드 할 때 필수 필드가 누락되지 않는 한 상태는 활성으로 설정됩니다. 필수 필드가 누락 된 경우 필수 필드가 빨간색으로 강조 표시된 메시지가 표시됩니다. 필수 입력란이 누락 된 동영상은 계속 업로드 할 수 있지만 그대로 유지됩니다. 비활성모든 필수 필드에 값이있을 때까지. 필수 필드 작업에 대한 자세한 내용은 비디오의 필수 필드설정을 참조하십시오 .

    참고: 계정 레벨 설정이 있으며, 기본값은 비활성화 , 업로드된 모든 동영상을 비활성화하는 데 사용할 수 있습니다. 자세한 내용은 계정 설정관리를 참조하십시오 .
  • 이름 -동영상 제목
  • 간단한 설명 -동영상에 대한 간단한 설명
  • 태그 -동영상과 관련된 태그
  • 참조 ID -동영상 참조 ID
  • 비디오 프로젝션 - 360도 비디오가 360도 비디오임을나타내려면 360도 를 선택합니다. 360도 비디오 작업에 대한 자세한 내용은 360° 비디오작업을 참조하십시오 .
  • 오프라인 재생 활성화 -체크하면 장치 SDK를 사용하여 만든 응용 프로그램을 사용할 때 비디오를 로컬에 저장하고 오프라인으로 볼 수 있습니다. 이 옵션은 계정이 오프라인 재생을 위해 활성화 된 경우에만 나타납니다. 사용할 때만이 옵션을 설정하는 것이 좋습니다. 고급 업로드 (사용자가 업로드 처리를 수동으로 시작해야하는 경우). 동영상 업로드 중일 때 설정하면 문제가 발생할 수 있습니다.
  • 폴더로 이동 -비디오를 할당 할 폴더를 선택합니다. 폴더 작업에 대한 자세한 내용은 폴더를 사용하여 비디오구성을 참조하십시오 .
  • 사용자 정의 필드 -필수로 표시된 모든 사용자 정의 필드가 나타납니다.

비디오 속성에 대한 자세한 내용은 미디어 모듈을 사용하여 비디오 속성편집을 참조하십시오 .

저장을클릭하여 변경 사항을 저장하고 트랜스코딩 프로세스를 시작합니다.

추가 자산 업로드

추가 에셋업로드계정 설정을 사용하면 비디오 속성 패널에선택버튼도 표시되므로 포스터와 썸네일 이미지를 비디오와 함께 업로드할 수 있습니다. [선택]을 클릭하여 포스터 및 축소판 이미지를 선택한 다음 [ 프로세스 ] 를 클릭하여 코드 변환을 시작합니다.

포스터 및 썸네일 이미지의 이미지 크기는 수집 프로필에 지정된 이미지 크기를 기반으로합니다. 고급 업로드계정 설정 구성에 대한 자세한 내용은 계정 설정관리를 참조하십시오 .

기본 폴더 및 태그 설정

동영상을 업로드 할 때 업로드되는 동영상의 폴더와 태그를 지정할 수 있습니다. 폴더 드롭 다운 목록에는 미디어 모듈에서 생성 된 모든 폴더 목록이 표시됩니다. + 폴더 만들기를클릭하여 새 폴더를 만들고 비디오를 새 폴더에 할당합니다. 새 폴더를 만들 때:

  • 이름은 100 자로 제한됩니다.
  • 폴더 이름은 계정 내에서 고유해야합니다.
  • 잘못된 문자:/

폴더를 선택하면 업로드 된 모든 비디오가 해당 폴더에 추가됩니다. 미디어 모듈을 사용하여 비디오를 만들고 폴더에 할당하는 방법에 대한 자세한 내용은 폴더를 사용하여 비디오구성을 참조하십시오 .

동영상이 업로드 될 때 태그를 할당 할 수도 있습니다. 동영상에 할당 할 태그 목록을 쉼표로 구분하여 입력하세요.

폴더를 선택하거나 태그를 추가한 후 모두에적용을 클릭하여 변경 사항을 저장합니다. 업로드 과정이나 업로드가 완료되고 동영상이 업로드 모듈에 계속 표시됩니다 ( 닫기버튼을 클릭하지 않음).

모바일 장치에서 비디오 업로드

미디어 모듈은 전적으로 HTML5로 작성되었으며 태블릿 및 모바일 장치에서도 사용할 수 있습니다. 이를 통해 모바일 장치로 비디오를 촬영하고 Video Cloud 계정에 업로드 할 수 있습니다. 모바일 장치에서 콘텐츠를 업로드하려면 업로드 모듈을 열고 수집 프로필을 선택합니다. 비디오찾아보기를 클릭하면 장치의 사진 응용 프로그램이 열립니다. 업로드 할 동영상을 선택하고 완료를 클릭합니다. 모바일 장치의 비디오가 업로드되기 시작하고 진행률이 장치에 표시됩니다.

일반적인 업로드 오류

업로드 프로세스가 중단 된 것 같거나 진행되지 않습니다.

브라우저 캐시를 지운 다음 비디오를 다시 업로드하십시오. 브라우저 캐시를 지우면 동영상 업로드 문제를 해결하는 데 도움이됩니다.

요청 시간과 현재 시간의 차이가 너무 큽니다.

이 오류는 동영상을 업로드하는 기기의 시스템 시간이 올바르게 설정되지 않은 경우 발생합니다. 이것은 일반적으로 시간 변경 후에 발생합니다. 기기가 정확한 시간으로 설정되어 있는지 확인하십시오.
