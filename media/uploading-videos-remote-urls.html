업로드 모듈은 동적 수집 프로세스를 사용하여 비디오 콘텐츠를 검색하고 트랜스 코딩합니다. 업로드 할 파일을 선택하면 Video Cloud는 풀 기반 프로세스를 사용하여 저장 위치에서 콘텐츠를 가져온 다음 Zencoder 트랜스 코딩 서비스를 사용하여 콘텐츠를 트랜스 코딩합니다.

업로드 모듈은 다양한 동영상 업로드 방법을 지원합니다.

  • 끌어서 놓기 -이 방법을 사용하면 로컬 파일 시스템에서 모듈 업로드 페이지로 파일을 드래그 할 수 있습니다.
  • 파일 선택 대화 상자 -이 방법은 업로드 할 파일을 찾아보고 선택할 수있는 파일 선택 대화 상자를 엽니 다.
  • 원격 URL -이 방법을 사용하면 URL을 입력하여 소스 파일의 위치를 지정할 수 있습니다.

이 항목에서는 인기있는 클라우드 파일 호스팅 서비스의 원격 URL을 얻는 방법에 중점을 둡니다. 원격 URL 업로드 방법을 사용하면 조직 외부의 사용자가 편집 / 처리 한 대용량 파일에 유용합니다. 대용량 파일을 로컬 컴퓨터에 복사하는 대신 파일에 대한 원격 URL을 제공 할 수 있으며 Video Cloud는 해당 위치에서 파일을 가져올 수 있습니다. 모든 경우에 파일의 URL을 공개적으로 사용할 수 있는지 확인해야합니다. 그렇지 않으면 업로드가 실패합니다.

참고: 업로드 모듈 사용에 대한 자세한 내용은 업로드 모듈을 사용하여 비디오업로드를 참조하십시오 .
참고: 동영상을 업로드하기 전에 적절한 수집 프로필이 선택되었는지 확인합니다.

구글 드라이브

Google 드라이브에 저장된 동영상의 URL을 얻으려면 다음 단계를 따르세요.

  1. Google 드라이브 계정에 로그인하십시오.
  2. 업로드 할 비디오를 찾아 마우스 오른쪽 버튼으로 클릭합니다.
  3. 공유 가능 링크가져오기 를 클릭합니다 .
  4. 사용 권한의 경우 링크를 가진모든 사용자가 파일에 액세스할수 있도록 옵션을 선택해야 합니다. 권한이제한됨으로 설정된 경우 업로드가 실패합니다.
    참고: 회사 Google 계정이있는 경우 유일한 공유 옵션이 제한되거나 도메인에 공유 될 수 있습니다. 링크를가진모든 사람에게 파일을 공유할 수 없는 경우 Video Cloud에서 파일에 액세스할 수 없으며 업로드가 실패합니다.
  5. 링크복사를 클릭합니다 . 불행히도 이것은 비디오 파일에 대한 직접 링크를 생성하지 않습니다. 다른 사이트를 사용하여 공유 링크에서 비디오 파일에 대한 직접 링크를 생성 할 수 있습니다.
  6. https://sites.google.com/site/gdocs2direct/home 로 이동합니다 . Google 드라이브 파일에 대한 직접 링크를 생성하는 다른 사이트도 사용할 수 있습니다.
  7. 공유 URL을 붙여 넣습니다.
  8. 직접 링크만들기를 클릭합니다 .

출력 링크 필드에 반환 된 URL은 동영상을 업로드하기 위해 업로드 모듈에서 사용할 URL입니다.

참고: 동영상 파일에 대한 직접 링크가 생성되지 않으므로 YouTube URL을 사용할 수 없습니다.
참고: Google 드라이브에 대한 크기 경고가 표시되면(일반적으로 파일의 경우) 업로드가 작동하지 않습니다.> 100MB). URL을 수정한 후에도 크기 오류 페이지로 리디렉션되고 업로드가 실패합니다.

드롭 박스

Dropbox에 저장된 동영상의 URL을 얻으려면 다음 단계를 따르세요.

  1. Dropbox 계정에 로그인하세요.
  2. 업로드할 비디오를 찾은 다음공유링크를 클릭합니다.
  3. 링크만들기를 클릭하여 파일에 대한 링크를 만듭니다.
  4. 링크복사를클릭하여 링크를 파일에 복사합니다.

파일의 URL은 로 끝날 수 있습니다?dl=0 . 브라우저에서 링크 내용을 강제로 다운로드하려면 URL의 해당 부분을 로 변경해야 합니다?dl=1 .

상자

Box에 저장된 비디오의 URL을 얻으려면 다음 단계를 따르십시오.

참고: 파일에 대한 직접 링크를 얻을 수 있는 기능은 무료 개인 계정에서는 사용할 수 없는 Box 프리미엄 기능입니다.
  1. Box 계정에 로그인하십시오.
  2. 업로드할 비디오를 찾은 다음공유링크를 클릭합니다.
  3. 공유 링크만들기슬라이더를 클릭하여 파일에 대한 링크를 만듭니다. 그러면 파일에 대한 미리보기 링크가 생성됩니다.
  4. 링크 설정을 클릭합니다 . 파일에 직접 다운로드 링크를 할당해야합니다.
  5. 링크를 클립보드로복사하려면 복사를 클릭합니다.

기타 서비스

URL이공개 URL인 경우 다른 클라우드 플랫폼 (예: Amazon S3) 에서 호스팅되는 비디오 파일에 URL을 사용하는 데 문제가 없어야합니다.

참고: Amazon S3에서 비디오 파일을 수집 할 때 모든 사람이 사용할 수없는 비디오를 Video Cloud에서 액세스 할 수 있도록 버킷 정책을 생성 할 수 있습니다. 자세한 내용은 S3에서 동적 인제스트사용 개발자 문서를 참조하십시오 .
