비디오의 소스 파일을 바꾸고 싶을 때가있을 수 있습니다. 한 가지 일반적인 경우는 이미 게시 된 비디오를 편집해야하는 경우입니다. 비디오를 삭제하고 다시 업로드하면 모든 비디오 메타 데이터를 다시 입력해야하며 비디오와 관련된 모든 비디오 분석을 잃게됩니다.

새 변환이 생성되는 동안 비디오는 기존 변환을 사용하여 계속 재생할 수 있습니다.

참고: 계정에 공유된 동영상의 소스 파일을 대체할 수 없습니다.
참고: 비디오 파일 이름 (확장명 포함) 은 단일 바이트 (60자 더블바이트) 120자를 초과해서는 안 됩니다. 이 경우 동영상이 성공적으로 인제스트되지만 나중에 다시 트랜스코딩할 수 없습니다.

비디오 소스 파일을 교체하는 단계

비디오 소스 파일을 바꾸려면 다음 단계를 따르십시오.

  1. 딸깍 하는 소리미디어탐색 헤더에서 미디어 모듈을 엽니다.
  2. 소스 파일을 교체 할 비디오 링크를 클릭합니다.
  3. 비디오 파일섹션을 찾아 소스 파일바꾸기를 클릭합니다 .
  4. 업로드모듈이 다른 탭에서 열립니다. 업로드 모듈이 소스 파일을 바꾸고 있음을 표시하는지 확인하십시오.
  5. 사용할 수집 프로필을 선택합니다.
    참고: 처음에 Dynamic Delivery 또는 Context Aware 수집 프로필을 사용하여 트랜스 코딩 된 비디오는 이러한 수집 프로필 유형 중 하나를 사용해서 만 다시 트랜스 코딩 할 수 있습니다. 레거시 수집 프로필을 사용하여 트랜스코딩된 비디오는 모든 유형의 프로필로 재트랜스코딩할 수 있습니다.
  6. 업로드 모듈을 사용하여 새 소스 파일을 업로드합니다.
  7. 기존 이미지를유지하려면 기존 축소판 이미지 및 포스터 이미지 유지를 선택합니다. 이 상자를 선택 취소하면 선택한 수집 프로필의 이미지 설정에 따라 새 이미지가 생성됩니다.
    확인란을 선택하면 기존 이미지가 유지되지만, 이미지 URL이 변경됩니다 .
  8. 원본 소스 파일과 연결된 모든 오디오 트랙이 새 파일로 대체됩니다. 확인하다기존 오디오 트랙 유지수집된 별도의 오디오 트랙을 유지합니다.
  9. 프로세스를 클릭하여 트랜스코딩을 시작합니다. 처리 완료메시지가 표시됩니다. 업데이트 된 변환을 사용할 수 있으려면 지연이있을 수 있습니다. 미디어 모듈의비디오 파일섹션에서업로드 날짜값을 확인하여 새 변환을 사용할 수 있는지 확인합니다.
