소개

자막에 액세스하려는 개인에게 추가 또는 해석 정보를 제공하기 위해 비디오에 자막을 추가 할 수 있습니다. 청각 장애인이 비디오 프레젠테이션의 오디오 부분에 액세스 할 수 있도록 캡션이 필요합니다.

이러한 텍스트는 액세스 가능성, 오디오 트랙의 번역 또는 기타 목적을 제공하는 데 사용될 수 있지만 사용되는 기본 기술은 동일합니다.

캡션 파일 위치

WebVTT 캡션 파일을 호스팅 할 위치를 선택할 수 있습니다.

  1. 원격 URL (원격 자산)에서 -이 경우 WebVTT 파일은 서버에서 호스팅되며 파일의 URL을 비디오 메타 데이터에 추가합니다.
    참고: 자체 서버에서 캡션 파일을 호스팅하는 경우 파일이 Brightcove 플레이어에서 액세스 할 수있는 URL에 있는지 확인하십시오. 캡션 파일을 호스팅하는 서버에는 CORS (Cross-Origin Resource Sharing)도 활성화되어 있어야합니다. CORS에 대한 자세한 내용은여기에서 확인할 수있습니다.
  2. 비디오 클라우드를 사용하여 업로드 ( 수집 ) - 이 경우 WebVTT 파일이 비디오 클라우드에 업로드 (수집) 됩니다. 미디어 모듈 또는 API를 사용하여 수행 할 수 있습니다.
참고: Video Cloud에 인제스트하는 비디오 소스 파일에 608/708 캡션이 포함되어 있으면 비디오 처리 중에 무시되며 비디오 변환에는표시되지 않습니다 . 동영상에 캡션을 사용하려면미디어 모듈 , 동적 인제스트 API 또는 CMS API (원격으로 호스팅된 캡션의 경우) 를 통해 캡션을 추가해야 합니다.

캡션을 비디오와 연결

캡션 파일을 비디오와 연결하는 방법에는 여러 가지가 있습니다.

  1. Studio 사용
  2. 플레이어 게시 코드 수정
  3. API 사용

Studio에서 캡션 추가

캡션은 미디어 모듈을 사용하여 비디오와 연결할 수 있습니다. 자세한 내용은 미디어 모듈을 사용하여 비디오에 캡션추가를 참조하십시오 .

플레이어 게시 코드 수정

이 경우 플레이어 퍼블리싱 코드에<track>태그가 추가됩니다. 자세한 내용은 프로그래밍 방식으로 비디오에 캡션추가를 참조하십시오 .

API를 통해 캡션 추가

API를 통해 캡션을 추가하는 방법은 캡션 파일이 수집되는지 또는 원격 자산으로 사용되는지에 따라 다릅니다.

  • 원격 자산 캡션 -캡션을 직접 호스팅하려는 경우 (또는 캡션을 호스팅하는 캡션 제공자가있는 경우) CMS API를 사용하여 캡션을 비디오와 연결할 수 있습니다. 자세한 내용은 CMS API 사용을 참조하십시오. WebVTT 캡션을 추가합니다 .

  • Video Cloud에 캡션 추가 -Brightcove가 캡션을 호스팅하도록하려면 Dynamic Ingest API를 사용하여 WebVTT 파일을 수집 할 수 있습니다. 자세한 내용은 WebVTT 파일수집을 참조하십시오 .

참고: 비디오에 캡션 파일을 추가하는 데 사용되는 방법에 관계없이 1000개 이하의 텍스트 트랙을 추가할 수 없습니다.

캡션 제거

미디어 모듈을 사용하거나 CMS API를 통해 비디오에서 캡션을 제거 할 수 있습니다. Studio에서 미디어 모듈을 열고 비디오 속성을 편집합니다. 필요에 따라텍스트 트랙속성을 편집합니다.

캡션은 다음을 사용하여 제거할 수도 있습니다. CMS API함으로써비디오 업데이트요청 및 비어 있는 요청 본문 제출text_tracks배열(또는 해당 캡션 집합과 연결된 개체가 제거된 배열).
