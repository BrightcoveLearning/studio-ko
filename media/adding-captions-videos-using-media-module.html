비디오에 대한 캡션을 제공하도록 텍스트 트랙 속성을 구성 할 수 있습니다. 캡션은 액세스하려는 개인에게 추가 또는 해석 정보를 제공합니다. 청각 장애인이 비디오 프레젠테이션의 오디오 부분에 액세스 할 수 있도록 캡션이 필요합니다. Brightcove 플레이어에서 비디오에 대한 캡션을 표시하려면 다음 작업을 완료해야합니다.

  • 캡션 파일 만들기
  • 텍스트 트랙 속성 구성
  • 스타일 캡션

캡션 파일 만들기

비디오에 캡션을 추가하려면 먼저 캡션 파일을 만들어야합니다. Brightcove 플레이어는 WebVTT (Web Video Text Tracks Format), SRT, DFXP 및 SCC 형식을 지원합니다. WebVTT 파일 형식 (아래 참조)은 일반 텍스트이며 시간 텍스트 데이터와 원하는 캡션 또는 자막 텍스트를 포함합니다.

메모

  • SRT, SCC 및 DFXP 캡션은 Dynamic Delivery 계정의 WebVTT로 자동 변환됩니다.
  • 일본어 및 한국어와 같은 더블바이트 언어에 대해 올바르게 표시하려면 캡션 텍스트를 UTF-8 인코딩해야 합니다.
  • 캡션은 HTML로 표시되므로< >대괄호 안의 텍스트는 태그로 해석되며 표시되지 않습니다.
WEBVTT 00:00:00.000 --> 00:00:07.080 정렬:중간 선:90% 00:00:07.080 --> 00:00:10.280 align:middle line:84% 안녕하세요. 저는 Brightcove의 학습 전문가인 Bob Bailey입니다. 00:00:10.280 --> 00:00:12.780 align:middle line:84% 이 비디오에서는 Video Cloud Studio가 00:00:12.780 --> 00:00:14.940 align:middle line:84% 구성 및 간략한 개요 보기 00:00:14.940 --> 00:00:17.490 align:middle line:90% 당신은 그것을 사용하여 달성할 수 있습니다.
메모:
  • 캡션 파일 이름에는 문자 (대문자 또는 소문자), 숫자, 대시 (-) 및 밑줄 (_) 만 포함되어야합니다.
  • 타사에서 만든 캡션이 타임 스탬프로 캡션을 시작하는 경우를 보았습니다.

    01:00:00 .000

    이렇게 하면 캡션이한 시간동영상 속으로. 캡션이 비디오와 올바르게 동기화되도록 하려면 타임스탬프가 00:00:00 .000으로 시작하는지확인하고 01시간 오프셋에 대한 모든 타임스탬프를 확인합니다.

WebVTT 형식에 대한 자세한 내용은이 링크를 참조하십시오 . 지원해야하는 각 언어에 대해 별도의 캡션 파일이 있어야합니다. 캡션 파일을 만드는 데 도움이되는 많은 타사 도구가 있지만 Brightcove에는 필사, 캡션 및 번역 서비스를 제공하는 여러 파트너가 있습니다. 브라이트코브 파트너의 전체 목록은 당사 웹사이트의파트너섹션을 참조하십시오.

텍스트 트랙 속성 구성

캡션 파일이 생성되면 Brightcove에서 업로드 및 호스팅하거나 인터넷 액세스 URL에 상주 할 수 있습니다.

비디오의 텍스트 트랙 속성을 보고 편집하려면 미디어 모듈을 열고 비디오 제목 링크를 클릭한 다음텍스트 트랙섹션을 찾습니다. 텍스트 트랙 속성을 구성하려면 다음 단계를 따르십시오.

  1. 편집을클릭하거나 섹션을 두 번 클릭합니다.
  2. 텍스트 트랙 파일을 Video Cloud에 업로드할지 또는 외부 URL에서 호스팅할지 선택합니다.
    • 파일 업로드 - 찾아보기를클릭하고 파일 시스템에서 텍스트 트랙 파일을 선택합니다. 파일이 Video Cloud에 업로드됩니다.
    • URL 업로드 - 텍스트 트랙 파일이 있는 공용 URL을 입력한 다음업로드를 클릭합니다 . 파일이 Video Cloud에 업로드됩니다.
    • 원격 URL -텍스트 트랙 파일이있는 공개 URL을 입력합니다. URL은 250 자 미만이어야합니다. 캡션은 원격 URL에서로드됩니다.
      참고: 자체 서버에서 캡션 파일을 호스팅하는 경우 파일이 Brightcove 플레이어에서 액세스 할 수있는 URL에 있는지 확인하십시오. 캡션 파일을 호스팅하는 서버에는 CORS (Cross-Origin Resource Sharing)도 활성화되어 있어야합니다. CORS에 대한 자세한 내용은여기에서 확인할 수있습니다.
      참고: 괄호, 물음표, 따옴표, 공백 및 기타 특수 문자가있는 텍스트 트랙 파일 이름은 업로드 할 때 자동으로 실패합니다. 브라이트코브는 특수 문자 없이 파일 이름을 사용할 것을 권장합니다.
      참고: 계정에 공유된 동영상의 텍스트 트랙은 미디어 모듈에서 편집할 수 없습니다.
    참고: 기본 (자동 표시)  옵션은 현재 동적 배달 제목에 영향을 주지 않습니다.
  3. NS 언어필드에는 다음이 포함되어야 합니다. 두 글자 언어 코드트랙 텍스트 데이터의 언어를 지정합니다.
  4. 적절한 입력상표캡션 파일의 경우. 라벨은 플레이어 내부에 표시되며 시청자가 언어를 선택할 수 있도록합니다. 레이블이 제공되지 않으면 언어 코드가 표시됩니다.
  5. 텍스트종류 트랙을 선택합니다. 옵션은 다음과 같습니다.
    • 캡션 -자막과 유사하지만 음향 효과 및 기타 오디오 정보도 포함 할 수 있습니다.
    • 자막 -대화의 필사 또는 번역
    • 설명 -스크린 리더를 통해 동영상을 설명하는 별도의 텍스트 파일입니다.
    • 장 -사용자가 동영상을 탐색 할 수 있도록 돕기위한 것
    • 메타 데이터 -기본적으로 시청자에게 표시되지 않는 동영상에 대한 정보 및 콘텐츠 (JavaScript를 사용하여 표시 할 수 있음)
  6. 텍스트 트랙 파일의 MIME 유형을입력합니다. 기본값을 선택하면 비디오가재생될 때 캡션이 표시됩니다 (브라이트코브 플레이어 6.x 이상).
    SRT 또는 DFXP 캡션을 업로드하는 경우 MIME 유형을 편집할 수 없습니다.
  7. 기본 (자동 표시)을 선택하여 트랙을 기본값으로 설정합니다. 대부분의 브라우저는 기본 텍스트 트랙을 먼저 표시하려고하지만 언어 및 접근성 설정이 기본 설정을 재정의 할 수 있습니다.
    참고: 기본 설정은 IE11에서 작동하지 않습니다.
  8. 텍스트 트랙추가를 클릭하여 텍스트 트랙 정보를 저장합니다.
  9. 텍스트 트랙추가를 클릭하여 다른 텍스트 트랙을 추가하거나저장을클릭하여 텍스트 트랙을 저장합니다.
    참고: 텍스트 트랙 속성에 대한 변경 사항은 게시된 플레이어에게 나타나는 데 최대 15분이 소요될 수 있습니다.

텍스트 트랙이 기본값으로 지정된 경우굵게 표시됩니다 . 텍스트 트랙을 삭제하려면 삭제 아이콘() 텍스트 트랙 옆에 있습니다.

참고로default값은 트랙당이 아니지만 다음으로 설정됩니다. true동일한 언어의 모든 트랙에 대해( srclang )

연결된 캡션이 있는 플레이어가 게시되면 플레이어 크롬에 CC버튼이 나타납니다. CC버튼을 클릭하여 캡션 언어를 선택하고 캡션을 표시합니다.

캡션 속성 편집

기존 캡션의 속성을 편집하려면 [텍스트 트랙] 패널에서 [ 편집 ] 을 클릭한 다음 편집할 캡션 항목을 클릭합니다.

스타일링 캡션

캡션의 기본 모양과 느낌은 변경 될 수 있으므로 캡션의 스타일을 지정할 수 있습니다. 캡션 스타일 지정에 대한 자세한 내용은 캡션스타일 지정을 참조하십시오 .

한계

  • 캡션은 다음 장치 유형 및 운영 체제에서 지원됩니다.
    • iOS 10 이상
    • Android 7 이상
    • Internet Explorer 11 이상
    • 데스크톱의 최신 Chrome, Safari 및 Firefox 브라우저
  • 캡션 길이에는 기술적 인 제한이 없지만 전체 캡션이 표시 되려면 플레이어가 그 정도의 텍스트를 표시 할 수있을만큼 커야합니다.
  • Safari는 캡션 스타일을 지정하는 데 기본 기능을 사용하기 때문에 Safari 브라우저 (iOS 및 macOS 모두) 의 CC메뉴에서캡션 설정메뉴 옵션을 사용할 수 없습니다.
  • Studio에서 잘못된 WebVTT 파일을 수집하면 파일이 '처리'되지만 동영상에 추가되지는 않습니다. 해결 방법 : 이 온라인 유효성 검사기를 사용하여 WebVTT 파일의 유효성을 검사할 수있습니다.
