Brightcove Social을 열거나게시됨왼쪽 탐색 영역에 Brightcove Social을 사용하여 게시된 비디오 목록이 나타납니다.

자동 동기화를 사용 중이고 동기화 규칙이 소셜 플랫폼에서 비디오 메타 데이터를 업데이트하도록 구성된 경우 동일한 비디오에 대한 여러 항목이 목록에 나타날 수 있습니다. 예를 들어 비디오가 처음 동기화 될 때 항목이 표시되고 비디오 메타 데이터가 업데이트되고 소셜 플랫폼에 동기화되면 추가 항목이 표시됩니다.

게시 상태보기

페이지 본문에는 소셜 미디어 대상에 게시 된 모든 비디오가 나열됩니다. 각 비디오에 대해 다음이 표시됩니다.

  • 상태
  • 이벤트 시간
  • 비디오 섬네일
  • 비디오 이름 및 간단한 설명
  • 비디오가 게시 된 소셜 미디어 계정 및 사이트에 대한 링크

게시 된 목록 필터링

페이지 상단에는 게시 상태 또는 대상에 따라 목록을 필터링하는 옵션이 제공됩니다. 다음 게시 상태를 사용할 수 있습니다.

  • 모든 상태 유형 -모든 상태 유형을 표시합니다.
  • 삭제 -소셜 플랫폼에서 게시 및 삭제 된 동영상을 표시합니다.
  • 오류 -오류가 발생한 게시물 표시
  • 숨겨진 -동영상이 비공개, 미등록 또는 소셜 미디어에 만료 된 게시물을 표시합니다.
  • 가공 -소셜 플랫폼에 게시중인 동영상을 표시합니다.
  • 게시 됨 -소셜 플랫폼에 성공적으로 게시 된 동영상을 표시합니다.
  • 예정 -소셜 플랫폼에 게시 될 예정인 동영상을 표시합니다.
  • 동기화 됨 -자동 동기화로 동기화 된 비디오 표시
  • 포기 됨 -소셜 플랫폼에 자동 동기화 된 비디오를 표시하고 다음 중 하나를 수행합니다.
    • 비디오가 삭제 될 때 작업을 수행하지 않도록 구성된 자동 동기화로 인해 비디오가 소셜 플랫폼에 남아있는 비디오 클라우드에서 삭제됨
    • 태그가 지정되지 않아 더 이상 자동 동기화와 일치하지 않지만 비디오 메타 데이터가 변경 될 때 작업을 수행하지 않도록 구성된 자동 동기화로 인해 소셜 플랫폼에 남아 있습니다.

비디오 검색

검색 필드는 계정의 비디오를 검색하는 데 사용할 수 있습니다. 검색 할 때 다음 필드가 검색됩니다.

  • 비디오 ID
  • 이름
  • 간단한 설명
  • 장황한 설명
  • 태그
  • 참조 ID
  • 사용자 정의 필드

일반적인 게시 오류

다음은 발생할 수있는 일반적인 게시 오류 목록입니다.

  1. YouTube - 업로드 거부: 비디오가 중복되었습니다 . YouTube는 동일한 비디오 (Video Cloud 비디오 ID가 아닌 업로드 된 비디오 바이너리의 디지털 지문을 비교하여 결정됨)를 채널에 두 번 이상 업로드하는 것을 허용하지 않으며이 오류로 인해 첫 번째 이후 시도가 실패합니다.
  2. NO_변환_소셜_플랫폼_오류입니다. Social 모듈은 플랫폼의 비디오 요구 사항을 충족하는 Video Cloud 비디오 변환을 찾을 수 없습니다. 이것의 가장 일반적인 원인은 트위터가 140 초 (Twitter의 최대 허용 시간)보다 긴 동영상 태그를 자동으로 동기화하는 게시자입니다.
  3. YouTube - 요청 메타데이터는 유효하지 않거나 빈 동영상 제목을 지정합니다 (더 긴 오류 메시지의 일부로) . YouTube의 최대 제목 길이는 100 자이므로 자동 동기화를 사용할 때 YouTube 제목에 매핑되는 Video Cloud 비디오 메타 데이터가 너무 길면이 오류가 발생합니다.
