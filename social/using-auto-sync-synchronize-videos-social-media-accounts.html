Brightcove Social은 비디오와 연결된 태그를 기반으로 비디오를 소셜 미디어 채널에 자동으로 동기화하는 데 사용할 수있는 자동 동기화 기능을 제공합니다.

자동 동기화를 구성하기 전에 일부 게시 대상을 만들어야합니다. 게시 대상 작성에 대한 자세한 내용은 소셜 미디어 게시 대상만들기를 참조하십시오 .

자동 동기화를 사용하고 소셜 모듈을 사용하여 수동으로 비디오를 게시하는 것은 독립적 인 작업입니다. 자동 동기화는 소셜 모듈을 사용하여 수동으로 게시 된 비디오를 인식하지 못합니다. 자동 동기화가 소셜 대상 중 하나에 게시하도록하는 방식으로 비디오에 태그가 지정되면 해당 비디오가 이전에 소셜 모듈을 사용하여 수동으로 게시되었는지 여부에 관계없이 그렇게됩니다.

자동 동기화 구성

자동 동기화를 구성하려면 다음 단계를 따르십시오.

태그를 입력하면 이러한 태그가있는 동영상 수가 표시됩니다.

참고: 지정된 태그가있는 기존 비디오는 자동으로 동기화되지 않습니다. 이러한 태그가있는 모든 새 동영상은 자동으로 동기화됩니다. 기존 비디오를 동기화하려면 비디오 메타데이터를 변경하여 자동 동기화를 트리거합니다.
  1. Brightcove Social을 엽니 다.
  2. 딸깍 하는 소리자동 동기화왼쪽 탐색에서 생성 된 모든 자동 동기화 이름의 목록이 표시됩니다.
    참고: 동영상이 동기화되면 썸네일 이미지가 채워집니다.
  3. 자동 동기화설정을 클릭합니다 .
  4. 동기화의이름을입력합니다.
  5. 배포 할 동영상을 선택하세요.
    • 모든 태그가 포함된 동영상 - 나열된모든태그가 있는 동영상은 동기화되어야 합니다.
    • 이러한 태그가 포함된 동영상 - 나열된모든태그가 있는 동영상은 동기화되어야 합니다.
    • 이 계정의 모든 동영상 -계정의 모든 동영상이 동기화되어야합니다.
  6. 이 계정의모든 동영상을 선택하지 않은 경우 쉼표로 구분된 태그 목록을 입력합니다.
  7. 제외해야하는 동영상을 선택하세요.
    • 비활성 -비활성 비디오는 동기화되지 않습니다 (기본값).
    • 제한된 일정 -예약 된 동영상은 동기화되지 않습니다.
    • 지리적으로 제한됨 -지역 제한이있는 동영상은 동기화되지 않습니다.
  8. 다음 을 클릭합니다 .
  9. 비디오를 동기화 할 게시 대상을 선택하십시오. 대상관리를 클릭하여 새 대상을 생성하거나 기존 대상을 편집합니다. 목적지를 클릭하면 목적지 기본값이 표시됩니다.
    참고: 대상 등록 정보를 편집하여 대상 기본값을 변경할 수 있습니다.
  10. 다음 을 클릭합니다 .
  11. 자동 동기화 규칙을 구성합니다 . 사용 가능한 옵션은 선택한 소셜 플랫폼에 따라 변경됩니다.

자동 동기화 규칙 구성

자동 동기화를 구성 할 때 소셜 미디어 사이트의 비디오에 발생하는 작업을 처리하도록 규칙을 구성 할 수 있습니다. 여기에는 비디오가 목록에서 제거 될 때 수행 할 작업 (비디오가 Video Cloud에서 삭제되거나 태그가 변경됨) 또는 비디오 메타 데이터 업데이트 처리 방법이 포함됩니다. 규칙은 선택한 소셜 미디어 플랫폼을 기반으로합니다.

  • 유튜브
  • 페이스 북
  • 트위터
메모:
  • 자동 동기화를 사용하여 비디오가 소셜 플랫폼에 게시되면 Brightcove Social은 소셜 미디어 측에서 해당 비디오에 발생하는 작업을 추적하지 않습니다. 예를 들어 Brightcove Social은 비디오가 소셜 플랫폼에서 삭제 된 경우 알림을받지 않으며 해당 비디오가 삭제되었다는 사실은 Brightcove Social에 반영되지 않습니다. 이는 소셜 미디어 플랫폼이 저작권 보호 콘텐츠를 감지 한 다음 제거하는 경우에도 발생할 수 있습니다. 다른 동기화 이벤트를 발생시키는 Video Cloud 측의 비디오에 대한 후속 변경은 Brightcove Social에서 동기화 오류를 유발합니다. 이 경우 '재시도 가능한 오류'이며 재 시도하면 변경 사항이 적용된 동영상이 다시 업로드됩니다.
  • 사용 가능한 자동 동기화의 초기 제한은 20 개입니다. 더 필요한 경우 브라이트코브지원팀에 문의하십시오 .

YouTube 규칙 구성

YouTube에서 자동 동기화를 사용하는 경우:

  • YouTube는 제목에 대해 100 자 제한이 있으므로 소셜 모듈에서 비디오 제목을 100 자로 자릅니다.
  • YouTube에서 동영상 메타데이터업데이트 옵션을선택하면 동영상이 캡션 없이 업로드되고 캡션이 추가되면 자막이 YouTube로 전달됩니다.
  • YouTube에서 동영상 메타데이터업데이트 옵션을선택한 경우 동영상이 자막과 함께 업로드되는 경우 Video Cloud에서 캡션 정보를 변경하면 업데이트된 캡션이 YouTube에 전달됩니다.
  1. 비디오에 태그가 지정되지 않았거나 Video Cloud에서 삭제되면 어떻게되는지 선택하십시오.
    • YouTube에서 배포 된 비디오 삭제 -동영상이 YouTube에서 제거됩니다.
    • 배포된 동영상 만들기 - YouTube 동영상을비공개또는미등록 상태로 만들기
    • 조치 없음 -YouTube 동영상에 아무 변화가 없습니다.
  2. 비디오 메타 데이터가 변경되면 어떻게되는지 선택하십시오.
    • YouTube에서 비디오 메타 데이터 업데이트 -Video Cloud 측의 비디오 메타 데이터 변경 사항은 YouTube에 동기화됩니다.
    • 조치 없음 - 조치를 취하지 않음
      메모: Studio에서 비디오 소스 파일이 업데이트되면 업데이트 된 비디오가 YouTube에 동기화되지 않습니다. 이는 YouTube 제한 사항입니다 .
  3. 다음 을 클릭합니다 .
  4. 자동 동기화 설정을 확인한 다음마침을 클릭합니다 .

Facebook 규칙 구성

  1. 비디오에 태그가 지정되지 않았거나 Video Cloud에서 삭제되면 어떻게되는지 선택하십시오.
    • Facebook에서 배포 된 비디오 삭제 -동영상은 Facebook에서 삭제됩니다.
    • Facebook에 배포 된 비디오 만료 -동영상은 Facebook에서 만료됩니다.
    • 조치 없음 -Facebook 동영상에 아무 일도 일어나지 않습니다
  2. 비디오 메타 데이터가 변경되면 어떻게되는지 선택하십시오.
    • Facebook에서 비디오 메타 데이터 업데이트 -Video Cloud 측의 비디오 메타 데이터 변경 사항은 Facebook에 동기화됩니다.
    • 조치 없음 -조치가 취해지지 않음
  3. 다음 을 클릭합니다 .
  4. 자동 동기화 설정을 확인한 다음마침을 클릭합니다 .

Twitter 규칙 구성

  1. 비디오에 태그가 지정되지 않았거나 Video Cloud에서 삭제되면 어떻게되는지 선택하십시오.
    • Twitter에서 배포 된 비디오 삭제 -해당 동영상은 Twitter에서 삭제됩니다.
    • 조치 없음 -트위터 동영상에 아무 일도 일어나지 않습니다
  2. 다음 을 클릭합니다 .
  3. 자동 동기화 설정을 확인한 다음마침을 클릭합니다 .

자동 동기화 작업보기 및 편집

생성 된 자동 동기화 작업을보고 편집하려면 다음 단계를 따르십시오.

  1. Brightcove Social을 엽니 다.
  2. 딸깍 하는 소리자동 동기화왼쪽 탐색에서 생성 된 모든 자동 동기화 작업 목록이 표시됩니다.
    • [ 태그]열의 링크를 클릭하여 미디어 모듈을 열고 해당 태그를 기반으로 검색합니다.
    • 대상열을 클릭하여 게시 대상 설정을 엽니다.
    • 삭제를 클릭하여 동기화 작업을 삭제합니다.
  3. 자동 동기화 이름 및 ID열 이름에서 링크를 클릭하여 동기화 속성을 엽니다.
  4. STATUS 는다음과 같이 설정할 수 있습니다.
    • 유효한 -동영상이 동기화됩니다
    • PAUSED -동영상이 동기화되지 않습니다.
  5. SHOW목록을 사용하여 표시되는 내용을 필터링할 수 있습니다.
  6. 설정을클릭하여 동기화 설정을 편집합니다. 업데이트를 클릭하여 변경 사항을 저장합니다.
  7. 삭제를 클릭하여 동기화 작업을 삭제합니다.
