Brightcove Social을 사용하면 Video Cloud 계정의 비디오를 YouTube, Facebook 및 Twitter에 쉽게 게시 할 수 있습니다. 비디오를 게시하기 전에 비디오를 게시 할 소셜 계정에 대한 게시 대상을 만들어야합니다. 게시 대상 작성에 대한 자세한 내용은 소셜 미디어 게시 대상만들기를 참조하십시오 .

자동 동기화를 사용하고 소셜 모듈을 사용하여 수동으로 비디오를 게시하는 것은 독립적 인 작업입니다. 자동 동기화는 소셜 모듈을 사용하여 수동으로 게시 된 비디오를 인식하지 못합니다. 자동 동기화가 소셜 대상 중 하나에 게시하도록하는 방식으로 비디오에 태그가 지정되면 해당 비디오가 이전에 소셜 모듈을 사용하여 수동으로 게시되었는지 여부에 관계없이 그렇게됩니다.

게시 할 변환 결정

비디오를 소셜 플랫폼에 배포하려면 비디오에 MP4 변환 또는 비디오 마스터가 하나 이상 있어야합니다. Brightcove에서 제공하는 표준 수집 프로필 중 하나를 사용하여 업로드 된 비디오는 기본적으로 마스터를 저장합니다. 보관 마스터에 대한 자세한 내용은보관 마스터문서를 참조하십시오.

비디오를 소셜 플랫폼에 배포 할 때 Brightcove Social은 개별 소셜 플랫폼에서 적용하는 규칙을 충족하는 최고 품질의 렌 디션을 배포하려고 시도합니다. 소셜 플랫폼으로 전송 된 렌 디션이 거부되면 다른 렌 디션을 보내려는 시도가 없습니다.

동영상 순위 지정 규칙은 다음과 같습니다.

  • MP4 자산은 다른 비디오 컨테이너보다 낫습니다. 다른 유형의 비디오 컨테이너를 주문하려고 시도하지 않습니다.
  • 비디오 컨테이너의 품질이 동일한 경우 인코딩 속도가 더 높은 비디오가 더 나은 것으로 간주됩니다.
  • 인코딩 속도가 동일한 품질이면 해상도가 더 높은 비디오가 더 좋은 것으로 간주됩니다. 비디오의 모든 변환이 반드시 동일한 종횡비를 갖는 것은 아니므로 각 비디오의 높이와 너비의 곱을 사용하여 계산된 '컴파운드' 프록시를 기반으로 한 비디오를 해상도에 비교합니다.

비디오 자산 중 하나에 비교하는 데 필요한 위의 정보가 부족한 경우 해당 비디오는 필요한 모든 정보가 포함된' 동영상보다 '나빠진 것으로 간주됩니다.

소셜 미디어에 비디오 게시

게시 대상을만든 후에는 다음 단계에 따라 비디오를 소셜 미디어에 게시합니다.

  1. Brightcove Social을 엽니 다.
  2. 딸깍 하는 소리비디오왼쪽 탐색에서 Video Cloud 계정의 비디오 목록이 표시됩니다. 소셜 미디어 계정에 게시되지 않은 비디오에는상태 NS 게시되지 않음 . 게시된 동영상에는 상태확인이 표시됩니다 . 링크를 클릭하면 소셜 미디어 대상에서 비디오의 게시 상태를 볼 수 있습니다.
  3. 게시할 비디오를 클릭한 다음게시를 클릭합니다 .
    참고: DRM이 활성화 된 비디오는 비디오 섬네일에 잠금 아이콘이 표시됩니다. Dynamic Delivery 수집 프로필을 사용하여 수집 된 DRM 지원 비디오는 게시 할 수 있지만 다운로드하거나 클립을 만들 수는 없습니다. DRM 사용 비디오가 선택되면 이 버튼이 비활성화됩니다.
  4. 비디오를 게시 할 대상을 선택합니다. 새 대상추가를 클릭하여 새 대상을만들 수도있습니다.
    참고: 오류가 있는 대상에는 오류 아이콘() 이름 옆에 대상을 선택한 다음다시 연결을클릭하여 문제를 해결합니다.
  5. 각 대상에 대한 비디오 메타 데이터 속성을 검토합니다. 비디오 속성은 게시하는 소셜 미디어 플랫폼에 따라 다릅니다. 자세한 내용은 각 소셜 미디어 플랫폼에 대한 섹션을 참조하십시오.
    • 유튜브
    • 페이스 북
    • 트위터
  6. 대상에게시버튼을 클릭하여 비디오를 게시합니다. 클릭하여 배포 상태를 모니터링합니다. 게시됨왼쪽 탐색에서 게시 스트림의 상태 보기에 대한 자세한 내용은 소셜 미디어 게시 스트림보기를 참조하십시오 .

게시 버튼을 사용하여 비디오를 게시하는 것은 일회성 게시 작업입니다. 이 방법을 사용하여 게시 된 동영상은 초기 업로드 후 소셜 모듈에서 메타 데이터 업데이트를받지 않습니다.

YouTube 용 비디오 속성 설정

기본적으로 최대 15 분 길이의 동영상을 YouTube에 업로드 할 수 있습니다. 긴 동영상을 업로드하는 방법에 대한 자세한 내용은 YouTube에서 더 긴 동영상활성화하기를 참조하세요 . 동영상이 YouTube에 게시 된 후 Studio의 소스 동영상에 대한 업데이트는 YouTube 측에서 업데이트되지 않습니다.

YouTube 대상을 선택할 때 다음 비디오 속성을 구성 할 수 있습니다.

  • 썸네일 소스
  • 게시물 정보
  • 시간표
  • 캡션
  • 시계
  • 특허
노트:
  • YouTube의 기본 동영상 속성은 대상을 편집하여 설정할수 있습니다.
  • YouTube API는 현재 API를 통한 댓글 비활성화를 지원하지 않으므로 소셜 UI에서 댓글을 비활성화하는 옵션이 없습니다.
  • 비디오의 이름과 설명에 특수 문자를 사용하지 마십시오. "<"또는 ">"를 사용하면 오류가 발생하는 것으로 알려져 있지만 다른 특수 문자도 문제를 일으킬 수 있습니다.

썸네일 소스 구성

YouTube에 게시 할 때 비디오 썸네일 소스를 다음과 같이 설정할 수 있습니다.

  • 유튜브 -YouTube에서 미리보기 이미지를 선택합니다.
  • 비디오 클라우드 -Video Cloud 썸네일 이미지를 YouTube 썸네일로 사용
  • 커스텀 이미지 -사용할 사용자 지정 썸네일 이미지를 찾아 선택합니다.

확인 중모든 목적지에 사용현재 게시 중인 모든 대상에 대해 이 축소판을 사용합니다.

참고: Brightcove Social에서 YouTube로 사용자 지정 비디오 섬네일을 게시하려면 먼저 YouTube 계정을 확인해야합니다. YouTube 계정 확인에 대한 자세한 내용은이 링크를 참조하십시오 .
참고: Studio에서 썸네일 이미지를 업데이트해도 YouTube 측의 썸네일 이미지가 업데이트되지 않습니다.

게시물 정보 속성 구성

POST 정보 정보링크를 클릭합니다. 다음 YouTube 동영상 속성을 설정할 수 있습니다.

  • 표제 -비디오 제목 (기본값은 Video Cloud 제목). YouTube는 최대 100자를 지원합니다.
  • 기술 -비디오에 대한 설명 (기본값은 Video Cloud 간단한 설명). YouTube는 최대 5000자를 지원합니다.
  • YouTube 카테고리 -동영상 콘텐츠 카테고리
  • 태그 -비디오 용 태그 (기본값은 Video Cloud 태그). 특정 태그 수 제한은 없습니다. 태그 당 최대 30자를 사용할 수 있으며 텍스트 필드에는 최대 500자를 사용할 수 있습니다.
태그의 글자수 제한에 대해 주의할 몇 가지 사항은 다음과 같습니다.
  1. 속성 값은 목록이며 목록에있는 항목 사이의 쉼표는 500 자 제한에 포함됩니다.
  2. 태그에 공백이 포함 된 경우 API 서버는 따옴표로 묶인 것처럼 태그 값을 처리하고 따옴표는 문자 제한에 포함됩니다. 따라서 와 같은late news태그가 있으면 이 태그가 11자 (따옴표 포함) 로 처리되고 계산됩니다. "late news"
  3. 태그의 문자 제한을 초과하면 소셜 모듈에 오류가보고되지 않습니다. 동영상이 성공적으로 공유되었는지 확인하려면게시됨왼쪽 탐색에서 YouTube를 확인하거나소셜 API .

비디오 다운로드

Brightcove Social은 비디오 URL을 복사하거나 비디오 소스 파일을 다운로드하는 기능을 제공합니다. 다운로드 할 때 최고 품질의 MP4 변환이 다운로드됩니다. MP4 변환이 없으면다운로드링크가 비활성화됩니다.

변환 및 이미지에 대한 URL이 고정되어 있지 않음을 이해하는 것이 중요합니다. Brightcove는 수시로 미디어 자산의 저장소를 재구성하며 이러한 상황이 발생하면 특정 자산의 URL이 변경됩니다. 이러한 자산에 대해 하드 코딩 된 URL을 사용하는 경우 링크가 어느 시점에서 끊어 질 수 있습니다. 미디어 링크가 끊어지지 않도록 하는 가장 좋은 방법은 런타임에 CMS API를 사용하여 Video Cloud에서 미디어를 검색하는 것입니다.

참고: DRM이 활성화 된 비디오는 다운로드 할 수 없습니다 (비디오 섬네일에 잠금이 나타남). DRM 사용 비디오를 선택하면다운로드버튼이 비활성화됩니다.

비디오를 다운로드하려면 다음 단계를 따르십시오.

  1. Brightcove Social을 엽니 다.
  2. 딸깍 하는 소리비디오왼쪽 탐색에서 Video Cloud 계정의 비디오 목록이 표시됩니다.
  3. 다운로드할 비디오를 클릭한 다음다운로드를 클릭합니다 .
  4. 비디오 URL을 복사하거나 링크를 클릭하여 브라우저에서 비디오를 엽니 다. 비디오를 마우스 오른쪽 버튼으로 클릭하여 소스 파일을 로컬에 저장합니다.
  5. 완료를 클릭합니다 .

다운로드 링크의 목적은 파일을 다운로드하거나 URL을 소셜 플랫폼의 업 로더 모듈 (URL에서 업로드를 지원하는 플랫폼의 경우)에 제공 할 수있는 값으로 사용하는 것입니다. URL은 공개 소셜 플랫폼 게시물에 붙여 넣을 수있는 오래 지속되고 공개적으로 액세스 할 수있는 URL이 아닙니다.

360 ° 비디오 게시

YouTube, Facebook 및 Brightcove Player는 모두 360 ° 비디오 재생을 지원합니다. 360 ° 비디오가 Brightcove Social을 사용하여 YouTube 또는 Facebook에 게시되면 비디오가 360 ° 모드로 재생됩니다. 360 ° 효과가 처리되는 동안 지연이있을 수 있습니다. 360 ° 비디오를 Video Cloud에 업로드하기 전에해야 할 일이 몇 가지 있습니다. 360° 비디오 작업에 대한 자세한 내용은 360° 비디오작업을 참조하십시오 . 동영상이 업로드되면 360 ° 동영상을 게시하는 프로세스는 일반 동영상과 동일합니다.

참고: 360° 비디오 재생은 특정 브라우저로 제한됩니다.
참고: Social 모듈을 사용하여 360° 비디오를 동기화하는 경우 비디오를 인제스트하는 데 사용된 수집 프로필을 마스터 파일을 저장하도록 구성해야 합니다. 그렇지 않으면 비디오가 360에서 동기화되지 않습니다.
