미디어 모듈을 사용하여 비디오를 게시 할 때 플레이어 URL을 사용하여 비디오를 미리 보거나 iframe 또는 인 페이지 포함 코드를 복사하여 웹 페이지 또는 응용 프로그램에 붙여 넣을 수 있습니다.

참고: 미리 보기 URL 옆에 있는단축버튼을 클릭하여 단축된 미리 보기 URL을 생성합니다.

표준링크에는 iframe 포함 코드가 표시되고고급링크에는 페이지 내 포함 코드가 표시됩니다. 플레이어 관리 API 관점에서 embed_in_page구현이라고 하는고급플레이어 코드도 볼 수 있습니다.

참고: 반응형 플레이어 작업에 대한 자세한 내용은 브라이트코브 플레이어개발자 문서를 위한반응형 크기 조정 을 참조하십시오.

iframe 소스 코드 (표준)

일반적인 iframe 소스 코드는 다음과 같이 표시됩니다.

<iframe src="https://players.brightcove.net/1507807800001/BkTGbgSq_default/index.html?videoId=5076962725001"
allowfullscreen
webkitallowfullscreen
mozallowfullscreen></iframe>

iframe 임베드 코드를 사용할 때의 몇 가지 이점

  • 기존 JavaScript 및 / 또는 CSS와의 충돌 없음
  • 자동으로 반응
  • iframe은 소셜 미디어 앱에서 또는 비디오를 외부 환경에서 사용해야 할 때마다 쉽게 사용할 수 있습니다. 예를 들어 Facebook은 iframe이 사용될 때 보안 위협이 훨씬 적다는 것을 인식합니다. <video>사이트의 태그 및 관련 JavaScript (페이스북은 허용하지 않음)

인 페이지 소스 코드 (고급)

일반적인 인 페이지 소스 코드는 다음과 같이 표시됩니다.

<video data-video-id="5076962725001" 
data-account="1486906377" 
data-player="default" 
data-embed="default" 
data-application-id 
class="video-js" 
controls></video>
<script src="//players.brightcove.net/1486906377/default_default/index.min.js"></script>

인 페이지 게시 코드를 통합하는 것은 더 복잡 할 수 있지만 플레이어가 포함 된 페이지가 플레이어와 통신해야하는 경우 인 페이지 코드를 사용하는 것이 가장 좋습니다. 인 페이지 소스 코드를 사용하는 경우의 몇 가지 예는 다음과 같습니다.

  • 포함 페이지의 코드는 플레이어 이벤트를 수신하고 조치를 취해야합니다.
  • 플레이어는 상위 페이지의 스타일을 사용합니다.
  • iframe 코드는 부모 페이지에서 리디렉션하는 것처럼 응용 프로그램 로직이 실패합니다.

최종 구현에서 iframe 삽입을 사용하지 않더라도 코드의 경우 JavaScript 용 플러그인과 CSS 용 별도 파일과 함께 인 페이지 코드를 계속 사용할 수 있습니다. 이것은 여러 플레이어에서 쉽게 사용할 수 있도록 논리를 캡슐화합니다.

참고: Studio에서 게시 할 때 iframe 구현을 선택한 경우 옵션이 표시됩니다. allowfullscreen webkitallowfullscreen mozallowfullscreen iframe에 추가되었습니다. 인 페이지 임베드 또는 iframe 구현 인 Brightcove Player를 iframe에 수동으로 배치하는 경우 적절한 전체 화면 기능을 위해 이러한 옵션도 포함해야합니다. 어떤 이유로 플레이어를 여러 iframe에 중첩하면 모든 iframe에 옵션이 추가되어야 합니다.

Chrome 및 "깨진"HTML 비디오 아이콘

Chrome 버전 67+에서는<video ...>태그가 브라이트코브 플레이어로 변환될 때까지 끊어진 비디오 아이콘이 잠시 표시됩니다. 아이콘은 다음과 같이 나타납니다.

브라이트코브 플레이어 버전 6.11 이상을 사용하는 경우video태그를 로<video-js ...>변경하면 아이콘이 표시되지 않습니다.

추천

일부 애플리케이션 로직에서 인 페이지 코드를 사용해야하는 경우가 아니면 iframe (표준) 구현을 사용하는 것이 모범 사례로 간주됩니다. 캠페인 모듈을 사용하여 시청자 참여를 추적하는 경우 인페이지(고급) 포함 코드를 사용해야 합니다.
