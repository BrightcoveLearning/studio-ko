이 항목에는 비디오 및 재생 목록을 웹에 게시하는 방법에 대한 정보가 포함되어 있습니다. 다른 게시 옵션에 대한 자세한 내용은 다음을 참조하십시오.

  • 이메일에 비디오 게시
  • 소셜 미디어 플랫폼에 비디오 게시
  • 인 페이지 환경에 비디오 게시

비디오 콘텐츠를 플레이어에 동적으로 할당하려는 경우플레이어용 임베드 코드사용 가능한 기술 중 하나를 사용하여비디오를 플레이어에 동적으로 할당 .

 
 
참고: 재생 목록 생성에 대한 자세한 내용은 재생 목록만들기 및 관리를 참조하십시오 .

비디오 및 재생 목록 게시

비디오 또는 재생 목록을 게시하려면 다음 단계를 따르십시오.

  1. 딸깍 하는 소리미디어탐색 헤더에서 미디어 모듈을 엽니다.
  2. 단일 비디오를 게시하려면 측면 탐색 패널에서 [ 모든 비디오] 를 클릭한 후 다음 중 하나를 수행합니다.
    • 비디오 옆에 있는 확인란을 클릭한 다음게시 및 포함 > 웹으로 게시를 클릭합니다.
    • 비디오 이름을 클릭한 다음 비디오 속성 페이지에서게시 및 포함 >웹에 게시를 클릭합니다.
  3. 재생 목록을 게시하려면 측면 탐색 패널에서모든 재생 목록을클릭한 후 다음 중 하나를 수행합니다.
    • 재생 목록 옆에 있는 확인란을 클릭한 다음게시 및 포함 > 웹으로 게시를 클릭합니다.
    • 재생 목록 이름을 클릭한 다음 재생 목록 동영상 페이지에서 재생목록게시를 클릭합니다.
  4. 게시 대화 상자가 열리는 지 확인합니다. 게시중인 비디오 또는 재생 목록의 이름이 게시 대화 상자 제목에 나타납니다.

    알려진 문제: 미리보기 플레이어는 Studio 내에서 HTTPS를 통해로드되므로 콘텐츠가 HTTP를 통해 제공되는 경우 미리보기 플레이어는 콘텐츠를 재생하지 않으며 브라우저는 안전하지 않은 스크립트 메시지를 표시합니다. 이것은 Studio 버그가 아니라 브라우저 보안 제한 사항입니다.

  5. 사용할 플레이어를 선택하십시오. 재생 목록을 게시할 때 재생 목록의모양과 동작을 제어하도록 설정할 수 있는 재생 목록 관련 플레이어 속성이있습니다. 자세한 내용은 아래의 재생 목록 속성설정섹션을 참조하십시오.

    미리보기가 업데이트되어 선택한 플레이어를 표시합니다.

  6. (선택 사항) 미리보기 플레이어 아래의 미리보기 링크를 사용하여 다른 브라우저 탭에서 플레이어를 미리 볼 수 있습니다. 단축버튼을 클릭하여 단축된 URL을 생성합니다.
  7. 플레이어 크기 조정 옵션을 선택하십시오.
    • 반응 형 -플레이어 크기는 브라우저 크기에 따라 자동으로 조정됩니다.
    • 결정된 -플레이어 크기는 너비 및 높이 매개 변수를 기반으로합니다.
  8. 플레이어의 화면 비율을 선택하십시오.
    • 16:9또는 4:3 - 플레이어 너비를입력하십시오 (높이가 자동으로 결정됨)
    • 커스텀 -플레이어 너비와 높이를 수동으로 입력하려면
  9. 플레이어폭과플레이어 높이를 입력합니다 .
  10. 측정단위를선택합니다.
  11. 표준또는고급을 클릭하여 복사할 게시 코드 유형을 선택합니다 .
    • 표준-iframe 소스 코드-대부분의 경우 권장됩니다. 이 유형은 플레이어를<iframe>태그 안에 배치합니다. 플레이어 크기가 지정되면 iframe 크기가 플레이어 크기로 조정됩니다. 플레이어 크기를 지정하지 않으면 iframe을<div>태그 안에 넣을 수 있으며 플레이어의 반응형 특성은 플레이어의 전체 공간을 차지하도록 크기를 조정합니다. <div> .
      참고: 재생 목록이 나타나려면 iframe 너비가 640px보다 커야 합니다.
    • 고급-인 페이지 소스 코드 -플레이어가 iframe이 아닌 HTML 페이지에 직접 존재하도록 허용합니다. 이를 통해 개발자는 플레이어 및 관련 속성 및 이벤트에 쉽게 액세스 할 수 있습니다. 인 페이지 소스 코드를 사용하면 플레이어를 소셜 미디어 애플리케이션 및 공유에 포함 된 단일 단위로 쉽게 사용할 수 없게됩니다. 이 옵션은 게시 된 플레이어에만 사용할 수 있습니다.
    게시 코드 유형에 대한 자세한 내용은 다음을 참조하십시오. 올바른 삽입 선택 암호 .
  12. 게시 코드를 복사합니다. 완료되면닫기를 클릭합니다 .

임베드에 애플리케이션 ID 추가 암호

과거에는 게시자가 일반적으로 사이트의 다른 부분에서 사용하거나 다른 사이트에서 사용하기 위해 다른 이름을 가진 새 플레이어를 만들었습니다. 플레이어 분석을 검토 할 때 다른 플레이어 이름을 사용하면 어떤 플레이어가 가장 많은 조회수를 얻고 있는지 쉽게 확인할 수 있습니다. 임베드에 애플리케이션 ID 매개 변수를 추가하여 코드에서 동일한 플레이어를 여러 사이트에서 사용할 수 있으며 애플리케이션 ID를 사용하여 플레이어가 게시 된 위치를 식별 할 수 있습니다. 플레이어 애플리케이션 ID를 사용하면 생성 및 유지 관리해야하는 플레이어 수를 줄일 수 있습니다. Embed에 애플리케이션 ID 추가에 대한 정보 코드, 참조플레이어 임베드에 애플리케이션 ID 추가 암호 .

재생 목록 속성 설정

재생 목록으로 작업 할 때 구성 할 수있는 재생 목록 관련 플레이어 속성이 있습니다. 재생 목록 관련 플레이어 프로퍼티를 구성하려면다음 단계를 따르십시오.

  1. 플레이어모듈을 엽니다.
  2. 플레이어 이름을 클릭하여 플레이어 속성을 엽니 다.
  3. 플레이어 정보속성의 일부로플레이어유형의재생 목록을 선택합니다 . 저장을 클릭합니다 .
  4. 딸깍 하는 소리스타일링왼쪽 탐색 메뉴에서
  5. 재생 목록 유형 선택 :
    • 재생 목록 (세로) -플레이어 오른쪽에 재생 목록이 표시됩니다.
    • 재생 목록 (가로) -플레이어 하단에 재생 목록이 표시됩니다.
    • 재생 목록 (숨김) -재생 목록이 표시되지 않습니다.
  6. 언제선수 유형 NS 재생 목록의 일부로 선택됩니다. 선수 정보속성, 추가재생속성을 구성할 수 있습니다.
    • 플레이 순서 무작위 화 -동영상은 무작위로 재생됩니다.
    • 선택시 비디오 재생 -재생 목록 메뉴에서 미리보기 이미지를 클릭하면 플레이어의 이전 동영상이 일시 중지 된 경우에도로드시 동영상이 재생됩니다.
    • 연속 재생 모드 -동영상이 자동으로 진행되고 무제한 반복됩니다.
    • 비디오 카운트다운 - 연속 재생 모드가 선택된 경우에만활성화됩니다.
      • 다음 카드 위로 (자동 진행) -플레이어에 다음 카드가 표시됩니다.
      • 다음 최종 화면 위로 (카운트 다운) -최종 화면 카운트 다운이 플레이어에 표시됩니다.
  7. 저장을클릭하여 변경 사항을 저장합니다. 변경 사항이 게시 된 플레이어에 표시되도록 플레이어를 게시해야합니다.

표준 (iframe)포함 코드를 사용하는 경우 재생 목록에 표시/숨기기 버튼이 있으며 자동으로 크기가 조정되고 위치가 지정됩니다. 고급 (페이지 내)포함 코드를 사용하는 경우 재생 목록을 HTML <div>요소로 삽입하고 자체 CSS를 작성하여 재생 목록의 위치를 지정하고 크기를 조정해야 합니다. 이렇게하면 재생 목록의 표시를 완전히 제어 할 수 있습니다. 후자의 경우 재생 목록이 원하지 않으면 플레이어와 인접 할 필요조차 없습니다. NS 고급(인페이지 임베드) 플레이어섹션재생 목록 구현 문서에는 재생 목록의 형식 지정 및 스타일 지정에 대한 상당한 콘텐츠가 있습니다.
